## Why：为什么要学习游戏引擎原理？

现代游戏引擎是**复杂实时系统**的典范：

| 挑战 | 引擎解决方案 | 学习价值 |
|------|-------------|---------|
| **16ms 帧时间限制** | 多线程、Job System、GPU Driven | 实时系统优化思维 |
| **海量对象管理** | ECS、对象池、内存池 | 内存与性能优化 |
| **复杂渲染管线** | 多 Pass、延迟渲染、Cluster-Based | 图形编程高级技巧 |
| **跨平台适配** | 抽象层、渲染后端（DX/Vulkan/Metal）| 架构设计能力 |
| **动态资源流送** | 异步加载、LOD、纹理流送 | 资源管理系统设计 |

> 理解引擎原理，不仅能更好使用引擎，更能培养解决复杂工程问题的能力。

## 🎯 关键概念速查

### 渲染管线对比

```
传统管线：                    GPU Driven 管线：
                                              
CPU: 剔除/LOD/排序    →      CPU: 仅提交命令
      ↓                            ↓
GPU: 接收并渲染            GPU: 剔除/LOD/排序 + 渲染
```

### 可见性剔除层次

| 剔除级别 | 方法 | 说明 |
|---------|------|------|
| **物体级** | 视锥剔除（Frustum Culling） | 用包围盒快速剔除视锥外物体 |
| **簇级** | Cluster Culling | Nanite 技术，64 三角形为一簇 |
| **像素级** | Early-Z / Hi-Z | GPU 硬件剔除被遮挡像素 |

### 纹理压缩格式

| 平台 | 推荐格式 | 特点 |
|------|---------|------|
| PC/主机 | BC7 | 高质量，128 bits/块 |
| 移动端 | ASTC | 灵活，可变比特率 |

> [!quote] 游戏引擎开发信条
> - **Do Nothing**：用剔除算法让引擎绘制尽可能少的内容
> - **数据与实例分离**：资源池 + 句柄复用
> - **按材质排序**：减少 GPU 状态切换
> - **GPU Driven**：将计算从 CPU 转移到 GPU

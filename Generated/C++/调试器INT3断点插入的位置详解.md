# 调试器 INT3 断点插入的位置详解

> 关于软件断点的基本工作原理，见 [[调试器核心概念与原理#4.1 软件断点（Software Breakpoint）]]。

## 问题：INT3 到底插入到哪里？

很多初学者容易混淆：**调试器设置断点时，是修改了磁盘上的可执行文件，还是修改了内存中的进程？**

## 答案：插入到内存中的代码段

**断点只修改内存中的进程实例，不修改磁盘上的可执行文件。**

```
┌─────────────────────┐         ┌─────────────────────────┐
│  磁盘上的可执行文件   │         │      内存中的进程        │
│      main.exe       │   →     │     代码段(可执行)       │
├─────────────────────┤         ├─────────────────────────┤
│  0x0000: mov eax,1  │         │  0x401120: mov eax, 1   │
│  0x0005: add eax,2  │         │  0x401121: add eax, 2   │
│  0x0008: ret        │         │  0x401124: ret          │
└─────────────────────┘         └─────────────────────────┘
                                          ↑
                                    调试器修改这里！
```

## 核心区别

| 特性 | 磁盘文件 | 内存进程 |
|------|----------|----------|
| 可写性 | 通常只读 | 调试器有特权修改 |
| 生命周期 | 编译后固定 | 运行时动态 |
| 影响范围 | 所有后续运行 | 仅当前调试会话 |
| 修改方式 | 文件IO | `ptrace` / `WriteProcessMemory` |

## GDB 实践指令

> 📁 **实践工作区**：`workspace/gdb_learning/`
> 实际动手操作见 [[workspace/C++/gdb_learning/README|GDB 学习工作区]]

### 基本流程

| 步骤 | 指令 | 说明 |
|------|------|------|
| 启动 | `gdb ./program` | 加载程序，**尚未修改任何内容** |
| 设断点 | `break main` | 在内存中插入 `0xCC`（INT3）|
| 运行 | `run` | 执行到断点触发 `SIGTRAP` |
| 查看 | `info breakpoints` | 查看当前断点列表 |
| 继续 | `continue` | 恢复执行，调试器自动处理指令恢复 |

### 观察断点行为的关键指令

```gdb
# 查看断点处的内存内容
x/4bx 0x401120          # 查看断点处的4个字节（可能看到 0xCC）

# 查看断点信息
info breakpoints        # 显示所有断点状态

# 禁用/启用断点
disable 1               # 禁用断点1
enable 1                # 启用断点1

# 删除断点
delete 1                # 删除断点1（恢复原始指令）
```

## 为什么需要恢复原始指令？

当断点命中时，内存中的状态是这样的：

```
0x401120: 0xCC (INT3)      ← 只有第1字节被替换
0x401121: 0x01             ← 原指令的第2字节
0x401122: 0x00             ← 原指令的第3字节
...
```

调试器必须：
1. **恢复**原来的完整指令
2. **回退指令指针** (RIP/EIP)
3. 让用户看到正确的代码
4. **单步执行**后继续运行时，**重新插入** `0xCC`

## 常见误区

| 误区 | 正解 |
|------|------|
| ❌ 断点修改了磁盘文件 | ✅ 只修改内存，磁盘文件始终是模板 |
| ❌ 断点永久破坏程序 | ✅ 调试器会恢复原始指令 |
| ❌ 可以预先在二进制中插入断点 | ✅ 现代调试器不采用这种方式 |

## 总结

```
┌─────────────────────────────────────────────────────────┐
│                      核心要点                            │
├─────────────────────────────────────────────────────────┤
│  • 断点修改的是「内存」，不是「磁盘文件」                  │
│  • 通过 ptrace 等特权 API 实现                            │
│  • 命中时恢复原始指令，继续前重新插入                      │
│  • 对程序的正常执行没有永久影响                           │
└─────────────────────────────────────────────────────────┘
```

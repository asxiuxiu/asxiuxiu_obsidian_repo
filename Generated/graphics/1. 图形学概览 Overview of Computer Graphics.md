## Why：为什么要学习图形学？

计算机图形学的目标是**用计算机生成图像**，应用遍及：

| 领域 | 应用示例 |
|------|---------|
| **游戏** | 实时渲染、物理模拟、角色动画 |
| **电影** | 特效制作、虚拟场景、角色建模 |
| **设计** | CAD、建筑设计、产品设计 |
| **可视化** | 医学影像、科学计算可视化、数据可视化 |
| **VR/AR** | 虚拟现实、增强现实、混合现实 |

### 核心挑战

1. **真实感 vs 实时性** — 电影可以离线渲染数小时，游戏必须在 16ms 内完成
2. **复杂性** — 光线、材质、运动、物理的精确模拟
3. **交互性** — 用户输入到画面反馈的延迟控制

> **实时定义**：每秒生成 30+ 张图片（30fps），游戏通常追求 60fps 甚至 120fps。

## What：图形学的四大领域

```

```

### 1. 光栅化（Rasterization）

> **实时渲染的金标准（Gold Standard）**

**核心流程**：
```
3D 三角形 → 投影到 2D 屏幕 → 分解为像素（Fragment）→ 着色计算 → 输出
```

**特点**：
- ✅ **速度快** — GPU 高度优化，适合实时应用
- ✅ **硬件支持** — 现代显卡专为光栅化设计
- ❌ **近似计算** — 光照效果需要各种 trick 模拟

**应用场景**：所有实时 3D 游戏、VR 应用

### 2. 曲线与几何（Curves and Meshes）

**核心问题**：
- 如何表示一条光滑的曲线？
- 如何表示复杂曲面？
- 形状变化时如何保持拓扑结构？

**表示方法**：
| 类型 | 示例 | 用途 |
|------|------|------|
| **隐式表示** | $x^2 + y^2 + z^2 = 1$（球面）| 碰撞检测、CSG 建模 |
| **显式表示** | 三角形网格（OBJ/FBX）| 渲染、游戏资产 |
| **参数曲线** | Bézier 曲线、B-Spline | 字体、路径动画 |
| **细分曲面** | Catmull-Clark | 角色建模 |

![[Assets/graphics/Pasted image 20231210113346.png]]
*几何建模：如何用数学表示复杂形状*

### 3. 光线追踪（Ray Tracing）

> **离线渲染的金标准**

**核心思想**：
```
从相机发射光线 → 穿过每个像素 → 与场景求交 → 计算着色 → 反射/折射 → 遇到光源停止
```

**光线追踪 vs 光栅化**：

| 特性 | 光栅化 | 光线追踪 |
|------|--------|---------|
| **速度** | 极快（GPU 硬件加速）| 慢（需大量计算）|
| **反射/折射** | 需要近似 trick | 自然支持 |
| **全局光照** | 难以实现 | 自动获得 |
| **软阴影** | 需要特殊处理 | 自然支持 |
| **应用场景** | 游戏、实时应用 | 电影、建筑可视化 |

![[Assets/graphics/Pasted image 20231210114150.png]]
*光线追踪：从相机发射光线，模拟真实光路*

> [!note] 实时光线追踪
> NVIDIA RTX 系列显卡支持 **实时光线追踪（RTX）**，通过专用硬件（RT Core）加速，正在改变游戏渲染。

### 4. 动画与模拟（Animation / Simulation）

**关键帧动画（Key Frame Animation）**：
- 艺术家设定关键姿态
- 计算机插值生成中间帧

**物理模拟（Physics Simulation）**：
- 质点弹簧系统（Mass-spring System）
- 布料模拟、流体模拟、刚体动力学

## How：学习路径

### 知识依赖

```
数学基础
├── 线性代数（向量、矩阵、变换）
├── 微积分（导数、积分）
└── 概率统计（蒙特卡洛方法）
    ↓
图形学基础
├── 变换（Transformation）
├── 光栅化（Rasterization）
├── 着色（Shading）
└── 纹理映射（Texture Mapping）
    ↓
进阶主题
├── 光线追踪（Ray Tracing）
├── 全局光照（Global Illumination）
├── 几何处理（Geometry Processing）
└── 实时渲染技术（Real-time Rendering）
```

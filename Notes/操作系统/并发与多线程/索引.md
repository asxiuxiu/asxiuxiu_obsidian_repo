## 推荐学习顺序

**第一阶段：基础（笔记 1-4）**
- 理解进程与线程的区别
- 学会创建线程
- 亲眼见证数据竞争的 bug
- 认识编译器优化带来的隐蔽问题

**第二阶段：同步工具（笔记 5-8）**
- 用 `std::atomic` 保护单个变量
- 用 `std::mutex` 保护临界区
- 用 Spinlock 处理短临界区
- 用 Condition Variable 避免忙等

**第三阶段：深入原理（笔记 9-10）**
- 理解编译器和 CPU 的重排
- 掌握 Release-Acquire 模式

**第四阶段：综合实战（笔记 11）**
- 实现无锁 SPSC 队列

## 阶段目标

1. **理解数据竞争**：能解释为什么 `++counter` 会丢数据，画出"读-改-写"交错时序图。
2. **掌握基础同步工具**：能根据场景选择 atomic / mutex / spinlock / condition variable。
3. **理解内存序**：能解释为什么 `release`/`acquire` 能建立 Happens-Before，为什么 x86 "碰巧正确"的代码在 ARM 上会崩。
4. **实现无锁队列**：能独立写出生产级 SPSC Ring Buffer，正确放置 `release`/`acquire`。

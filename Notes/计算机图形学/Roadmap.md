# Graphics Journey 路线图

> 本路线与 [[Notes/SelfGameEngine/0_RoadMap|SelfGameEngine 构建手册]] **深度融合**。你的图形学知识不再是一条孤立的"理论线"，而是直接服务于引擎构建的"能力线"。
>
> **核心原则**：不在图形学里闭关修炼，而是在关键节点**切入引擎开发**，用工程需求反向驱动深度学习。
>
> **构建哲学**：先建立像素级直觉（软渲染器），再理解GPU如何替代你的for循环，最后把这些知识组织成引擎的渲染管线。

### 关键认知：这不是"先学后做"

传统思路：先花3个月把图形学学完，再开始写引擎渲染。

**本路线的思路**：
- 阶段一：建立像素级直觉（帧缓冲是什么）
- 阶段一.二：建立完整概念地图——软渲染器、GPU编程、引擎封装的关系
- 阶段二：建立3D数学基础（软渲染器实现，纯CPU模拟GPU算法）
- 阶段三后：**切入引擎阶段1.2**——你有足够知识把"最简图形后端"从清屏升级到画三角形
- 阶段五后：**切入引擎阶段5.1和5.3**——你有能力参与RHI和材质系统的开发
- 阶段六后：**切入引擎阶段5.2和5.4**——你理解纹理加载、GPU上传、渲染生命周期
- 阶段七后：与引擎阶段5.6完全对应，直接并行开发后处理
- 阶段八：不再是"学图形学"，而是"在引擎里设计渲染架构"

## 如何避免沉迷图形学？

> 这是本路线最重要的设计约束。

### 红线规则

| 规则 | 说明 |
|------|------|
| **阶段三后不闭关** | 完成第一个三角形后，必须开始回探引擎阶段1.2，把清屏升级为画三角形。不准"我再学学纹理再切入" |
| **阶段五后必须切** | 完成Blinn-Phong后，必须开始参与引擎阶段5.1和5.3的开发。此时你对Shader和Uniform有基础理解 |
| **阶段六后必须切** | 完成纹理系统后，必须正式参与引擎阶段5.2和5.4的开发。此时你对资源管理和渲染生命周期都有基础理解 |
| **阶段七-八不分开** | 帧缓冲/后处理/引擎渲染架构这三个阶段**必须在引擎代码里学**。不要在独立Demo里写三遍 |
| **阶段九按需深入** | PBR/阴影/Deferred可以按需深入。如果引擎暂时不需要，先了解概念，等需要时再实现 |

### "沉迷"的预警信号

如果你发现自己：
- 在独立Demo里实现了5种不同的高斯模糊优化，但引擎里还没有后处理管线 → **停止！回引擎！**
- 为了"完整理解"Vulkan，花两周写了一个能清屏的Vulkan程序，但引擎的RHI还是空的 → **停止！用OpenGL先跑通引擎！**
- 读了10篇PBR论文，但引擎里连一个带纹理的立方体都跑不起来 → **停止！先让立方体跑起来！**

### 正确的并行节奏

```
周1-2：    阶段三（GPU编程基础：第一个三角形）
周3：      回探引擎阶段1.2 → 把最简图形后端从清屏升级到画三角形
周4-5：    阶段四（顶点数据与索引）+ 引擎阶段5.2网格导入
周6：      阶段五（Shader与光照）+ 引擎阶段5.1/5.3 RHI/材质
周7-10：   阶段六~七 + 引擎阶段5.4/5.6 渲染生命周期/后处理
周11+：    阶段八~九 + 引擎阶段5全模块深化
```

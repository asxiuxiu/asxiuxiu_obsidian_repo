# 3D 图形学概览

> [!info] 前置知识
> 在阅读本文之前，建议先完成 [[从像素开始|从像素开始]] 和 [[光栅化 - 从直线到三角形|2D 光栅化]] 的学习。
>
> 本文作为**从 2D 到 3D 的过渡**，解释：我们已经能画 2D 三角形了，但游戏世界是 3D 的，这中间发生了什么？

---

## Why：从 2D 到 3D

在 [[光栅化 - 从直线到三角形|上一节]] 中，我们已经实现了：
- ✅ 用 Bresenham 算法画直线
- ✅ 用边缘函数填充三角形
- ✅ 用重心坐标做颜色插值

但这些都是在 **2D 屏幕空间** 进行的。真正的游戏场景是这样的：

```
3D 游戏世界                    2D 屏幕
┌─────────────┐              ┌─────────────┐
│  🏰 城堡    │   ──────→    │  ▲ 三角形   │
│  🌲 树木    │   变换+投影   │  ■ 像素     │
│  ☁️ 云朵    │              │  ● 颜色     │
└─────────────┘              └─────────────┘
```

### 核心问题

**如何把 3D 世界变成 2D 像素？**

这是计算机图形学要解决的核心问题。从 2D 光栅化到 3D 渲染，需要跨越三个关键门槛：

| 门槛 | 2D 光栅化 | 3D 图形学 |
|------|----------|----------|
| **坐标表示** | 屏幕坐标 (x, y) | 3D 空间坐标 (x, y, z) |
| **几何变换** | 直接指定像素位置 | 需要平移/旋转/缩放/投影 |
| **深度处理** | 无（直接覆盖）| Z-Buffer 处理遮挡 |

## 3D 世界到底是什么？

在讲数学变换之前，我们必须先回答一个更基础的问题：**3D 世界由什么构成？**

如果你刚从 2D 过来，你对"3D 世界"的想象可能是一堆复杂的数学公式。但其实，3D 世界和现实世界非常像——它是由**物体**、**相机**和**光源**组成的。

### 用"拍照"来理解 3D 渲染

想象你要给一桌美食拍一张照片：

```
┌─────────────────────────────────────────┐
│           现实世界（3D 场景）            │
│                                         │
│    🍰 蛋糕        🍷 红酒               │
│         🍝 意面                         │
│              📷 相机                     │
│                                         │
│    💡 灯光 ←──────────────→ 💡 灯光      │
└─────────────────────────────────────────┘
                    │
                    │ 拍照
                    ↓
┌─────────────────────────────────────────┐
│           照片（2D 图像）                │
│         ┌─────────────┐                 │
│         │  🍰  🍝  🍷  │                 │
│         │  （2D 像素）  │                 │
│         └─────────────┘                 │
└─────────────────────────────────────────┘
```

3D 渲染本质上就是**用虚拟相机给虚拟世界拍一张照片**。照片上的每一个像素，都是计算机计算出来的。

### 3D 场景的四大要素

| 要素 | 现实世界的类比 | 在图形学中的作用 |
|------|--------------|----------------|
| **模型 (Model)** | 桌子上的蛋糕、红酒 | 3D 世界的"演员"，由几何形状定义 |
| **材质 (Material)** | 蛋糕的奶油质感、红酒的玻璃反光 | 决定模型表面的颜色、光泽、透明度 |
| **光源 (Light)** | 餐厅的灯光、窗外的阳光 | 照亮场景，产生明暗和阴影 |
| **相机 (Camera)** | 你的手机/相机 | 决定从哪个角度看世界，最终生成 2D 图像 |

> [!note] 关键直觉
> 3D 渲染 = 摆放好物体 → 打好灯光 → 架好相机 → 按下快门（计算像素颜色）

## 3D 模型是如何表示的？

这是初学者最容易困惑的地方。一个 3D 角色、一辆汽车、一座城堡，在计算机里到底是什么？

### 模型 = 几何 + 外观

一个 3D 模型文件（如 `.obj`、`.fbx`）通常包含两部分数据：

```
┌─────────────────────────────────────────┐
│              3D 模型文件                 │
├─────────────────┬───────────────────────┤
│    几何数据      │       外观数据         │
├─────────────────┼───────────────────────┤
│  • 顶点位置      │  • 材质颜色            │
│  • 面片索引      │  • 纹理图片            │
│  • 法线方向      │  • 反光参数            │
│  • 纹理坐标      │  • 透明度              │
└─────────────────┴───────────────────────┘
```

### 几何数据：用三角形拼凑出形状

计算机无法直接画"一个球"或"一个人"。它只能画**三角形**。所以所有 3D 模型，本质上都是一堆三角形的集合：

```
    球体                  角色头部
   /\  /\                /\    /\
  /  \/  \              /  \  /  \
 /   /\   \            /____\/____\
/\  /  \  /\           \    /\    /
  \/    \/              \  /  \  /
   \    /                \/    \/
    \  /                  \    /
     \/                    \  /
                             \/
```

这些概念你可能听说过，但不太清楚具体含义：

| 概念                    | 通俗解释           | 作用                       |
| --------------------- | -------------- | ------------------------ |
| **顶点 (Vertex)**       | 三角形的三个角点       | 记录 3D 空间中的位置 `(x, y, z)` |
| **面片 (Face/Polygon)** | 一个三角形          | 三个顶点连成的面，是 3D 模型的最小砖块    |
| **面片索引 (Index)**      | 每个三角形对应的三个顶点编号 | 用顶点数组的下标定义面片，避免重复存储顶点    |
| **网格 (Mesh)**         | 所有面片的集合        | 构成模型的完整几何形状              |
| **法线 (Normal)**       | 垂直于面片朝外的方向     | 告诉光源"这个面朝哪个方向"，用于计算明暗    |

> [!example] 一个立方体有多少个面片？
> 6 个面 × 2 个三角形 = **12 个三角形面片**，8 个顶点。
> 一个精细的游戏角色可能有 **5万~50万个** 三角形。

> [!note] 基础渲染 vs. 高级拓扑
> 对于**画出来**这件事，**顶点 + 面片索引** 就是最小必要集合。
>
> 但某些高级算法（如网格细分、平滑着色、布料物理模拟、边缘检测）确实需要知道**面与面之间的邻接关系**（比如哪两个三角形共享一条边）。这种邻接信息**通常不会直接存储在 `.obj`、`.fbx` 等原始模型文件里**，而是由游戏引擎或应用程序在加载时，根据顶点和索引**自行推导并预计算**的。
>
> 所以：原始模型文件里的是「面片索引」，真正的「面片连接关系」是按需后算的。

### 外观数据：让三角形有颜色、有质感

如果只有几何数据，模型就是一堆白色的线框。我们需要**外观数据**来让它看起来像真实物体。

| 概念 | 通俗解释 | 作用 |
|------|---------|------|
| **纹理 (Texture)** | 一张贴在模型表面的 2D 图片 | 给模型添加细节，比如皮肤、木纹、砖墙 |
| **贴图 (Map)** | 纹理的另一种说法，或特指某种用途的纹理 | 颜色贴图、法线贴图、粗糙度贴图等 |
| **材质 (Material)** | 一组"表面属性"的参数包 | 定义颜色、反光程度、透明度、金属感等 |
| **纹理坐标 (UV)** | 2D 图片上的坐标 `(u, v)` | 告诉计算机"三角形的这个角对应图片的哪个像素" |

```
        3D 模型表面                    2D 纹理图片
    ┌─────────────┐                ┌─────────────┐
    │  A        B │                │  A───────B  │
    │   \      /  │      UV 映射    │   │ 皮肤  │  │
    │    \    /   │  ───────────→  │   │ 纹理  │  │
    │     \  /    │                │   │ 图片  │  │
    │      C      │                │  C─────────  │
    └─────────────┘                └─────────────┘
```

> [!note] 核心问题：三角形怎么贴矩形纹理？
> 每个顶点存储 **UV 坐标** $(u,v)$，GPU 通过[[Notes/数学基础/线性组合与张成空间与基|重心坐标]]在三角形内部插值，实现任意形状贴矩形纹理。
> 
> **UV 映射详解**（生成方法、缝合边、美术工作流程）：[[纹理映射与UV坐标]]

> [!tip] 记忆口诀
> - **顶点**定形状（几何）
> - **面片**拼模型（几何）
> - **纹理**加细节（外观）
> - **材质**定质感（外观）

## 相机、视角与投影

现在我们知道了 3D 世界由模型、材质、光源组成。但为什么从不同角度看，物体的形状不一样？为什么远处的物体看起来更小？

### 相机决定"看什么"

3D 场景里有无数物体，但你的屏幕只有那么大。相机的作用就是：
1. **确定位置**：我站在场景的哪里？
2. **确定朝向**：我朝哪个方向看？
3. **确定视野**：我能看多宽、看多深？

```
        3D 场景
    ┌─────────────────┐
    │    🏠 房子      │
    │      🌳         │    📷 相机
    │    🚗 汽车  ←───┼────┐
    │                 │    │ 朝向
    └─────────────────┘    ↓
                      ┌─────────┐
                      │ 2D 图像  │
                      │  🚗 🏠  │
                      │  （汽车近大，房子远小）│
                      └─────────┘
```

### 投影：把 3D"拍扁"成 2D

相机拍出的照片是 2D 的，这个"拍扁"的过程在图形学中叫做**投影 (Projection)**。

生活中有两种常见的投影：

| 投影类型 | 现实类比 | 特点 | 应用场景 |
|---------|---------|------|---------|
| **透视投影** | 人眼、普通相机 | 近大远小，有纵深感 | 游戏、电影、写实渲染 |
| **正交投影** | 建筑蓝图、CAD | 平行线保持平行，无近大远小 | 工程制图、策略游戏小地图 |

```
透视投影：                    正交投影：
    /\                        |───|
   /  \                       |   |
  /____\                      |___|\n  近大远小                    大小不变
```

> [!note] 关键直觉
> 平移、旋转、缩放 = **把物体摆到正确的位置和姿态**
> 视图变换 = **把相机架到正确的位置和角度**
> 投影变换 = **把 3D 场景拍成 2D 照片**

## What：渲染管线的三大核心

现在你已经有了"3D 世界 = 场景 + 相机 + 光源"的直觉。接下来，计算机图形学（专注渲染方向）可以划分为三个核心领域：

```
┌─────────────────────────────────────┐
│         3D 渲染管线核心               │
├──────────┬──────────┬───────────────┤
│ 几何变换 │ 光栅化   │ 着色计算       │
│Transform │Rasterize │    Shading    │
│ (坐标系)  │ (像素化)  │    (颜色)      │
└──────────┴──────────┴───────────────┘
```

> [!note] 关于动画与物理
> 动画系统（骨骼动画、关键帧插值）和物理模拟（布料、刚体、流体）虽然在视觉表现上与渲染紧密相关，但在现代引擎架构中属于独立的子系统。本系列专注于**渲染管线**本身，动画与物理相关内容将在独立目录中讨论。

### 1. 几何变换（Transformation）

**核心问题**：如何描述物体在 3D 空间中的位置、方向、大小？

```
模型坐标 → 世界坐标 → 相机坐标 → 屏幕坐标
   (局部)    (全局)     (视角)      (像素)
```

你需要学习：
- **线性代数基础**：向量、矩阵运算（详见 [[Notes/数学基础/索引|线性代数系列]]）
- **齐次坐标**：用 4×4 矩阵统一表示平移、旋转、缩放
- **MVP 变换**：Model（模型）→ View（视图）→ Projection（投影）

> [!note] 与 2D 的区别
> 2D 变换可以直接写死坐标，3D 变换必须用矩阵——因为要考虑相机的位置和朝向。

### 2. 光栅化（Rasterization）

**核心问题**：3D 三角形如何变成屏幕上的像素？

```
3D 三角形顶点 ──→ 投影到 2D ──→ 边缘函数判断 ──→ 着色 ──→ 像素
     ↓                ↓              ↓            ↓
  顶点着色器       透视除法       光栅化器      片段着色器
```

与 2D 光栅化的区别：
- 需要考虑 **透视投影**（近大远小）
- 需要 **Z-Buffer** 处理遮挡关系
- 需要 **透视校正插值**（纹理坐标不能线性插值）

### 3. 着色（Shading）

**核心问题**：像素应该是什么颜色？

| 着色模型 | 特点 | 应用 |
|---------|------|------|
| **平面着色** | 每个三角形一种颜色 | 极简风格 |
| **高洛德着色** | 顶点计算颜色，内部插值 | 早期 3D 游戏 |
| **冯氏着色** | 每个像素单独计算光照 | 现代游戏基础 |

光照模型要考虑：
- 光源类型（平行光、点光源、聚光灯）
- 材质属性（漫反射、镜面反射、环境光）
- 法线方向（顶点法线、面法线）

## How：3D 渲染管线总览

现代 GPU 的渲染管线是一个**数据流**处理过程：

```
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│  3D 模型   │───→│   顶点处理  │───→│   光栅化   │───→│  像素处理  │
│  (OBJ/FBX) │    │ (MVP 变换)  │    │(三角形填充)│    │ (着色输出) │
└────────────┘    └────────────┘    └────────────┘    └────────────┘
       │                │                │                │
       ↓                ↓                ↓                ↓
   顶点数据         屏幕空间         片段(Fragment)      最终颜色
   (x,y,z,u,v)      坐标 (x,y)       带插值属性          (RGBA)
```

### 关键组件说明

| 组件        | 功能               | 我们之前学过的              |         |
| --------- | ---------------- | -------------------- | ------- |
| **顶点着色器** | 处理每个顶点，进行 MVP 变换 | -                    |         |
| **图元装配**  | 顶点组装成三角形         | 2D 三角形的概念            |         |
| **光栅化器**  | 三角形 → 像素         | ✅ [[光栅化 - 从直线到三角形 | 边缘函数]]  |
| **片段着色器** | 计算每个像素的颜色        | 需要学习                 |         |
| **帧缓冲**   | 存储最终图像           | ✅ [[从像素开始         | 像素缓冲区]] |

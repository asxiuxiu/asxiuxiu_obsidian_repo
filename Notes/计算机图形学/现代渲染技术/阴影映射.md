# 阴影映射

## 问题 0：PBR 让立方体像金属了，但为什么它还像“飘”在空中？

你在 [[Notes/计算机图形学/现代渲染技术/PBR基础|PBR基础]] 里已经让立方体对光产生正确的漫反射和镜面反射。但如果没有阴影，观众的大脑会拒绝承认这是一个“放在地面上的物体”——它看起来就像悬浮在背景前面。

**最 naive 的方案**：在物体正下方画一个黑色椭圆当“影子”。

```glsl
// 在片段着色器里假装有个影子
if (dot(worldNormal, vec3(0,1,0)) > 0.9 && vWorldPos.y < 0.01) {
    FragColor.rgb *= 0.5; // 脚下变黑
}
```

**立刻发现的问题**：

- 影子不会随光源方向变化。太阳在左边，影子还在正下方。
- 影子不会投射到斜坡、台阶或其他物体表面。
- 多个物体重叠时，影子不会互相遮挡。
- 形状永远是椭圆，和物体轮廓无关。

**根本原因**：真实的阴影是“光被物体挡住”的结果。要正确计算遮挡，必须知道从光源看过去，场景每个点是否被其他几何体遮住。

---

## 问题 1：怎么知道一个点是否被光“看到”？

想象你站在光源的位置，朝场景拍一张照片。照片里每个像素记录的不是颜色，而是**从光源到该像素的距离**（深度）。

现在回到相机视角，看场景中的某个点 $P$：

- 把 $P$ 变换到光源的坐标系，得到它在光源视角下的深度 $d_P$。
- 把这个深度和 shadow map 里对应位置的深度 $d_{map}$ 比较。
- 如果 $d_P > d_{map}$，说明从光源看，有另一个点挡在 $P$ 前面——$P$ 在阴影里。
- 如果 $d_P \approx d_{map}$，说明 $P$ 就是光源能看到的那个点——$P$ 被照亮。

这就是 **Shadow Mapping（阴影映射）** 的核心思想：**光能看到的点是亮的，光看不到的点是暗的**。

## 问题 2：Shadow Map 的两 Pass 算法

整个算法分成两个渲染 Pass：

### Pass 1：从光源视角渲染深度图

```
渲染目标：一个深度纹理（Shadow Map）
相机位置：光源位置
投影方式：方向光用正交投影（Orthographic），点光源用透视投影（Perspective）
着色器：只需要顶点着色器把位置写到 gl_Position，片段着色器可以空着
输出：深度缓冲的值自动写入 Shadow Map
```

```glsl
// shadow_pass.vert
#version 330 core
layout (location = 0) in vec3 aPos;

uniform mat4 uLightSpaceMatrix;
uniform mat4 uModel;

void main() {
    gl_Position = uLightSpaceMatrix * uModel * vec4(aPos, 1.0);
}
```

```glsl
// shadow_pass.frag
#version 330 core
void main() {
    // OpenGL 会自动把 gl_FragCoord.z 写入深度附件
    // 我们也可以显式输出深度，但通常不需要
}
```

### Pass 2：正常渲染场景，采样 Shadow Map

```
渲染目标：默认帧缓冲（或场景颜色纹理）
相机位置：正常相机
着色器：在片段着色器里把当前像素变换到光源空间，采样 Shadow Map 做深度比较
```

```glsl
// scene_pass.frag（简化版）
#version 330 core
in vec3 vWorldPos;
in vec3 vNormal;

uniform sampler2D uShadowMap;
uniform mat4 uLightSpaceMatrix;
uniform vec3 uLightPos;
uniform vec3 uViewPos;

out vec4 FragColor;

void main() {
    // 1. 把世界空间位置变换到光源裁剪空间
    vec4 lightSpacePos = uLightSpaceMatrix * vec4(vWorldPos, 1.0);

    // 2. 做透视除法，得到 NDC [-1, 1]，再映射到 [0, 1] 的纹理坐标
    vec3 projCoords = lightSpacePos.xyz / lightSpacePos.w;
    projCoords = projCoords * 0.5 + 0.5;

    // 3. 当前像素在光源视角下的深度
    float currentDepth = projCoords.z;

    // 4. 从 shadow map 读取最近可见点的深度
    float closestDepth = texture(uShadowMap, projCoords.xy).r;

    // 5. 比较
    float shadow = currentDepth > closestDepth ? 1.0 : 0.0;

    // 6. 把阴影因子乘到光照结果上
    vec3 color = CalculateLighting(vWorldPos, vNormal, uLightPos, uViewPos);
    FragColor = vec4(color * (1.0 - shadow * 0.5), 1.0);
}
```

> 这里的光照计算可以替换为你已有的 Blinn-Phong 或 PBR。

## 问题 3：`uLightSpaceMatrix` 到底是什么？

`uLightSpaceMatrix` 把世界空间坐标变换到光源的裁剪空间，和 MVP 矩阵的作用类似：

$$
\mathbf{M}_{lightSpace} = \mathbf{P}_{light} \cdot \mathbf{V}_{light}
$$

其中：

- $\mathbf{V}_{light}$：光源的视图矩阵（LookAt）。对于方向光，光源位置可以任意（因为方向光没有位置），通常放在场景中心或相机附近，朝光照反方向看。
- $\mathbf{P}_{light}$：光源的投影矩阵。方向光用正交投影，因为它模拟太阳光，光线平行。

```cpp
// 方向光的光空间矩阵示例
vec3 lightDir = normalize(vec3(-1.0f, -2.0f, -1.0f));
vec3 lightPos = sceneCenter - lightDir * 50.0f; // 沿光照反方向退后

mat4 lightView = lookAt(lightPos, sceneCenter, vec3(0,1,0));
mat4 lightProj = ortho(-20.0f, 20.0f, -20.0f, 20.0f, 1.0f, 100.0f);
mat4 lightSpaceMatrix = lightProj * lightView;
```

> **关键决策**：正交投影的左右上下远近平面必须覆盖整个场景。如果包围盒太小，物体会被裁掉；如果太大，Shadow Map 的精度会被浪费。

## 问题 4：为什么 Shadow Map 会有「条纹痘痘」？

第一个能跑通的 Shadow Map 通常会出现这种画面：物体表面布满了规则的黑色条纹，像长了痘痘。这叫 **Shadow Acne（阴影痤疮）**。

**原因**：Shadow Map 的分辨率有限，每个 texel 对应一片表面区域。由于浮点精度问题，同一片区域在 Pass 1 里写进去的深度和 Pass 2 里算出来的深度不完全相等。原本应该“刚好等于”的表面，被误判为“比 shadow map 深一点点”，于是产生自阴影。

**最 naive 的修复**：采样时加一个固定偏移。

```glsl
float bias = 0.005;
float shadow = currentDepth - bias > closestDepth ? 1.0 : 0.0;
```

**更好的修复**：根据表面与光线夹角动态调整 bias。夹角越大（表面越倾斜），需要的 bias 越大。

```glsl
float bias = max(0.05 * (1.0 - dot(normal, lightDir)), 0.005);
float shadow = currentDepth - bias > closestDepth ? 1.0 : 0.0;
```

**诚实边界**：bias 是一剂“止痛药”，不是根治。太大的 bias 会导致另一个问题——Peter Panning。

---

## 问题 5：为什么阴影和物体“脱钩”了？

当你把 bias 调大后，可能会发现阴影不再贴着物体底部，而是向下“飘”了一截，像彼得潘的影子一样。这叫 **Peter Panning（彼得潘现象）**。

**原因**：bias 本质上是把 shadow map 里的深度往光源方向推了一点。推得太多，物体最下层的像素也会被判定为“在阴影里”，于是阴影开始从物体底部脱离。

**修复方案**：

1. **只在渲染 Shadow Map 时剔除正面（Front Face Culling）**：

```cpp
glEnable(GL_CULL_FACE);
glCullFace(GL_FRONT); // 渲染 shadow map 时只画背面
RenderShadowPass();
glCullFace(GL_BACK);  // 恢复正常
```

这样 shadow map 记录的是物体背面的深度。背面比正面离光源更远，所以正面在比较时天然有“余量”，不需要很大的 bias 就能避免 acne。

2. **不要用过大的 bias**：动态 bias 已经够用，不要为了追求“彻底没有 acne”而牺牲阴影贴合度。

## 问题 6：Shadow Map 边缘为什么锯齿严重？

即使 acne 和 Peter Panning 都解决了，把画面放大还是会看到阴影边缘像乐高积木一样一块一块的。这是因为 Shadow Map 分辨率有限：一个 texel 可能覆盖屏幕上的多个像素，所有像素都得到同一个“在阴影里/不在阴影里”的答案。

**最 naive 的修复**：把 Shadow Map 分辨率从 1024×1024 提到 4096×4096。

**立刻发现的问题**：

- 显存占用从 4MB 涨到 64MB（如果是 16-bit 深度）。
- 每帧渲染 Shadow Map 的填充率翻倍。
- 远处物体仍然锯齿，因为一张大纹理要覆盖整个视锥。

**更好的方案：PCF（Percentage-Closer Filtering）**

PCF 的核心思想是：不要只采样 shadow map 的一个点，而是采样周围一片点，分别做深度比较，最后把结果平均。

```glsl
float PCFShadow(sampler2D shadowMap, vec3 projCoords, float currentDepth, float bias) {
    vec2 texelSize = 1.0 / textureSize(shadowMap, 0);
    float shadow = 0.0;

    for (int x = -1; x <= 1; ++x) {
        for (int y = -1; y <= 1; ++y) {
            float pcfDepth = texture(shadowMap, projCoords.xy + vec2(x,y) * texelSize).r;
            shadow += currentDepth - bias > pcfDepth ? 1.0 : 0.0;
        }
    }

    return shadow / 9.0; // 3x3 采样
}
```

> **关键认知**：PCF 不是“模糊 shadow map 的深度值”。深度值不能直接平均（平均两个深度没有意义）。PCF 平均的是**深度比较的结果**（0 或 1），所以得到的是介于 0 和 1 之间的软阴影。

## 问题 7：大场景里远处的阴影糊成一片，怎么办？

一张 Shadow Map 覆盖整个场景时，存在根本矛盾：

- 相机附近需要高分辨率（看清楚脚下细节）。
- 远处需要大覆盖范围（看见地平线）。
- 同一张纹理无法同时满足两者。

**Cascade Shadow Maps（CSM，级联阴影图）** 就是解决这个矛盾的方案。

**核心思想**：把相机视锥沿深度方向切成几段（如 4 段），每一段单独渲染一张 Shadow Map。

```
Cascade 0：近处 0.1m ~ 10m，Shadow Map 1024×1024，精度最高
Cascade 1：10m ~ 40m，Shadow Map 1024×1024
Cascade 2：40m ~ 100m，Shadow Map 1024×1024
Cascade 3：100m ~ 300m，Shadow Map 1024×1024
```

渲染时，根据像素到相机的距离选择用哪一级 cascade。

```glsl
// 简化示意
int cascadeIndex = ChooseCascadeByDepth(viewSpaceDepth);
float shadow = SampleCascadeShadow(cascadeIndex, lightSpacePos[cascadeIndex], bias);
```

**CSM 的光空间矩阵**：每一级 cascade 都要算自己的 `lightSpaceMatrix`。近处 cascade 的投影矩阵包围盒小，远处 cascade 的包围盒大。

**级联分界线怎么定？**

最常用的是对数分割：

$$
c_i = near \cdot \left(\frac{far}{near}\right)^{\frac{i}{n}}
$$

其中 $n$ 是 cascade 数量。这样对近处分配更多精度。但实际中通常在对数分割和线性分割之间取一个混合值，避免最近处过于狭窄。

> 本篇只做概念引入。CSM 的完整实现涉及纹理数组、级联混合、稳定 CSM 等主题，会在后续笔记深入。

## 完整可运行示例：方向光 Shadow Map + PCF

### C++ 端：创建 Shadow Map FBO

```cpp
// flags: override -std=c++20 -Wall -O2 -lglfw -lGL -lX11 -lpthread -lXrandr -lXi -ldl
#include <glad/glad.h>
#include <GLFW/glfw3.h>
#include <glm/glm.hpp>
#include <glm/gtc/matrix_transform.hpp>
#include <glm/gtc/type_ptr.hpp>
#include <iostream>

const unsigned int SHADOW_WIDTH = 1024, SHADOW_HEIGHT = 1024;
GLuint depthMapFBO, depthMap;

void InitShadowMap() {
    glGenFramebuffers(1, &depthMapFBO);

    // 深度附件用纹理，因为 Pass 2 要采样它
    glGenTextures(1, &depthMap);
    glBindTexture(GL_TEXTURE_2D, depthMap);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_DEPTH_COMPONENT,
                 SHADOW_WIDTH, SHADOW_HEIGHT, 0,
                 GL_DEPTH_COMPONENT, GL_FLOAT, nullptr);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_NEAREST);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_NEAREST);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_BORDER);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_BORDER);
    float borderColor[] = {1.0f, 1.0f, 1.0f, 1.0f};
    glTexParameterfv(GL_TEXTURE_2D, GL_TEXTURE_BORDER_COLOR, borderColor);

    glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT,
                           GL_TEXTURE_2D, depthMap, 0);
    // 不需要颜色输出
    glDrawBuffer(GL_NONE);
    glReadBuffer(GL_NONE);

    if (glCheckFramebufferStatus(GL_FRAMEBUFFER) != GL_FRAMEBUFFER_COMPLETE)
        std::cerr << "Shadow FBO 不完整" << std::endl;

    glBindFramebuffer(GL_FRAMEBUFFER, 0);
}
```

> `GL_CLAMP_TO_BORDER` + 白色边界：当投影坐标超出 shadow map 范围时，返回深度 1.0，表示“不被遮挡”，避免远处出现奇怪阴影。

### 顶点着色器

```glsl
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec3 aNormal;
layout (location = 2) in vec2 aTexCoord;

uniform mat4 uModel;
uniform mat4 uView;
uniform mat4 uProjection;
uniform mat4 uLightSpaceMatrix;

out vec3 vWorldPos;
out vec3 vNormal;
out vec4 vLightSpacePos;

void main() {
    vWorldPos = vec3(uModel * vec4(aPos, 1.0));
    vNormal = mat3(transpose(inverse(uModel))) * aNormal;
    vLightSpacePos = uLightSpaceMatrix * vec4(vWorldPos, 1.0);
    gl_Position = uProjection * uView * vec4(vWorldPos, 1.0);
}
```

### 片段着色器（带 PCF）

```glsl
#version 330 core
in vec3 vWorldPos;
in vec3 vNormal;
in vec4 vLightSpacePos;

uniform sampler2D uShadowMap;
uniform vec3 uLightPos;
uniform vec3 uViewPos;
uniform vec3 uLightColor;
uniform vec3 uObjectColor;

out vec4 FragColor;

float ShadowCalculation(vec4 lightSpacePos, vec3 normal, vec3 lightDir) {
    vec3 projCoords = lightSpacePos.xyz / lightSpacePos.w;
    projCoords = projCoords * 0.5 + 0.5;

    // 超出 shadow map 范围 → 照亮
    if (projCoords.z > 1.0) return 0.0;

    float currentDepth = projCoords.z;
    float bias = max(0.05 * (1.0 - dot(normal, lightDir)), 0.005);

    // PCF 3x3
    vec2 texelSize = 1.0 / textureSize(uShadowMap, 0);
    float shadow = 0.0;
    for (int x = -1; x <= 1; ++x) {
        for (int y = -1; y <= 1; ++y) {
            float pcfDepth = texture(uShadowMap, projCoords.xy + vec2(x,y) * texelSize).r;
            shadow += currentDepth - bias > pcfDepth ? 1.0 : 0.0;
        }
    }
    shadow /= 9.0;

    return shadow;
}

void main() {
    vec3 normal = normalize(vNormal);
    vec3 lightDir = normalize(-vec3(-1.0, -2.0, -1.0)); // 方向光方向

    // 简化 Blinn-Phong
    float diff = max(dot(normal, lightDir), 0.0);
    vec3 ambient = 0.1 * uLightColor;
    vec3 diffuse = diff * uLightColor;

    float shadow = ShadowCalculation(vLightSpacePos, normal, lightDir);
    vec3 lighting = (ambient + (1.0 - shadow) * diffuse) * uObjectColor;

    FragColor = vec4(lighting, 1.0);
}
```

### 渲染循环骨架

```cpp
void RenderFrame() {
    // ===== Pass 1：渲染 Shadow Map =====
    glViewport(0, 0, SHADOW_WIDTH, SHADOW_HEIGHT);
    glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
    glClear(GL_DEPTH_BUFFER_BIT);

    glUseProgram(shadowShaderProgram);
    glUniformMatrix4fv(lightSpaceMatrixLoc, 1, GL_FALSE,
                       glm::value_ptr(lightSpaceMatrix));

    glEnable(GL_CULL_FACE);
    glCullFace(GL_FRONT); // 避免 Peter Panning

    for (auto& obj : sceneObjects) {
        glUniformMatrix4fv(modelLoc, 1, GL_FALSE, glm::value_ptr(obj.model));
        DrawMesh(obj.mesh);
    }

    glCullFace(GL_BACK);
    glBindFramebuffer(GL_FRAMEBUFFER, 0);

    // ===== Pass 2：正常渲染场景 =====
    glViewport(0, 0, windowWidth, windowHeight);
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

    glUseProgram(sceneShaderProgram);
    glUniformMatrix4fv(viewLoc, 1, GL_FALSE, glm::value_ptr(view));
    glUniformMatrix4fv(projectionLoc, 1, GL_FALSE, glm::value_ptr(projection));
    glUniformMatrix4fv(lightSpaceMatrixLoc2, 1, GL_FALSE,
                       glm::value_ptr(lightSpaceMatrix));

    glActiveTexture(GL_TEXTURE0);
    glBindTexture(GL_TEXTURE_2D, depthMap);
    glUniform1i(shadowMapLoc, 0);

    for (auto& obj : sceneObjects) {
        glUniformMatrix4fv(modelLoc2, 1, GL_FALSE, glm::value_ptr(obj.model));
        DrawMesh(obj.mesh);
    }
}
```

## 状态变化图：Shadow Map 在两 Pass 中的流动

```
Pass 1（Shadow Pass）
  当前渲染目标 = depthMapFBO
  当前深度附件 = depthMap（深度纹理）
  当前光空间矩阵 = lightSpaceMatrix
  剔除模式 = GL_FRONT（仅背面）
        │
        ▼
  场景几何 → 深度写入 depthMap

Pass 2（Scene Pass）
  当前渲染目标 = 默认帧缓冲
  当前纹理单元 0 = depthMap
  当前着色器 = sceneShader
        │
        ▼
  对每个像素：
    worldPos ──► lightSpaceMatrix ──► projCoords
    projCoords.xy 采样 depthMap 得 closestDepth
    projCoords.z 与 closestDepth 比较
    结果 = shadow 因子
```

> **关键状态机事实**：Shadow Map 在同一次绘制中**不能同时作为渲染目标和采样源**。必须先完成 Pass 1，解绑 FBO，再把它绑定到纹理单元。

## 与 SelfGameEngine 的关系

### Shadow Pass = RenderGraph 中的新增节点

在 [[Notes/SelfGameEngine/渲染管线与画面/RenderGraph与多Pass资源管理|RenderGraph与多Pass资源管理]] 里，一帧被组织成多个 Pass。加入阴影后，图结构变成：

```
Shadow Pass ──► Scene/Lighting Pass ──► PostProcess Pass
```

```cpp
graph.AddPass("Shadow", [&](PassBuilder& b) {
    b.WriteDepth(shadowMap);
    b.Execute([=](ICommandContext* ctx) {
        ctx->SetPipeline(shadowPipeline);
        ctx->SetCullMode(CullMode::Front); // 仅背面剔除
        for (auto& item : shadowCasters) DrawItem(ctx, item);
    });
});

graph.AddPass("Lighting", [&](PassBuilder& b) {
    b.Read(shadowMap);       // 声明读 ShadowMap
    b.WriteColor(0, sceneColor);
    b.Execute([=](ICommandContext* ctx) {
        ctx->BindTexture(4, shadowMap); // binding 4 = shadow map
        // ... 正常渲染
    });
});
```

### 与 ECS 的映射

阴影投射者（Shadow Caster）和阴影接收者（Shadow Receiver）可以用 ECS 组件标记：

```cpp
struct ShadowCaster {};    // 这个实体会被画进 Shadow Pass
struct ShadowReceiver {};  // 这个实体的材质会采样 Shadow Map
```

渲染 System 在 `Prepare` 阶段：

1. 查询所有带 `ShadowCaster` 的实体，生成 `shadow_phase.items`。
2. 查询所有带 `ShadowReceiver` 的实体，生成 `scene_phase.items`。

### 与材质系统的关系

在 [[Notes/SelfGameEngine/渲染管线与画面/材质系统架构|材质系统架构]] 中，阴影不是一个独立材质，而是**光照模型的一个输入**：

```cpp
struct StandardMaterial {
    ShaderTemplateHandle templateRef;
    TextureHandle albedoMap;
    TextureHandle normalMap;
    // ...
    bool receiveShadows = true; // 是否采样 shadow map
};
```

对应的 Shader 变体关键字：

- `RECEIVE_SHADOWS`：是否采样 shadow map
- `SHADOW_PCF`：是否启用 PCF
- `SHADOW_CSM`：是否启用级联阴影

## API 对照：我们在解决「从光源视角生成深度图并用于遮挡判断」这个问题

| 概念 | OpenGL | Vulkan | D3D12 |
|------|--------|--------|-------|
| 深度图渲染目标 | FBO + 深度纹理附件 | `VkFramebuffer` + Depth `VkImageView` | `ID3D12Resource` + DSV |
| 深度比较采样 | `sampler2D` + 手动比较 | `sampler2DShadow` + `VkCompareOp` | `SampleCmp` / `SampleCmpLevelZero` |
| 硬件 PCF | `GL_TEXTURE_COMPARE_MODE` + `sampler2DShadow` | `VkSamplerCompareCreateInfo` | `D3D12_FILTER_COMPARISON_MIN_MAG_LINEAR_MIP_POINT` |
| 多 cascade | 纹理数组 / 多个纹理 | `VkImageView` array layers | Texture2DArray + SRV |

> **个人项目推荐**：学习阶段用 OpenGL 手动 PCF（3x3 循环采样），清晰可控。引擎阶段可以切换到硬件 `sampler2DShadow` 做 2x2 PCF，再手动扩展更大核。CSM 建议用纹理数组（`GL_TEXTURE_2D_ARRAY`）管理多 cascade。

## 设计 checklist

| 检查项 | 标准 |
|--------|------|
| Shadow Map 尺寸 | 1024×1024 起步，质量敏感场景 2048×2048 或更高 |
| 光空间矩阵 | 方向光用正交投影，包围盒覆盖场景 |
| Bias | 动态 bias：`max(0.05 * (1 - NdotL), 0.005)` |
| 背面剔除 | Shadow Pass 用 `glCullFace(GL_FRONT)` |
| 越界处理 | `GL_CLAMP_TO_BORDER` + 白色边界 |
| 软阴影 | PCF 3x3 起步，需要更好效果再上更大核或 PCSS |
| 大场景 | 引入 CSM，近处高精度、远处低精度 |

## 常见陷阱

### 陷阱 1：Shadow Pass 和 Scene Pass 用同一个 Model Matrix

```cpp
// ❌ 错误：Shadow Pass 没传 model 矩阵
glUniformMatrix4fv(lightSpaceMatrixLoc, ...);
// 忘了 glUniformMatrix4fv(shadowModelLoc, ...);
```

Shadow Pass 里的 model 矩阵必须和 Scene Pass 一致，否则深度比较的是两个不同位置。

### 陷阱 2：没做透视除法

```glsl
// ❌ 错误：直接用 lightSpacePos.xyz 当纹理坐标
vec3 projCoords = vLightSpacePos.xyz;

// ✅ 正确
vec3 projCoords = vLightSpacePos.xyz / vLightSpacePos.w;
projCoords = projCoords * 0.5 + 0.5;
```

### 陷阱 3：Shadow Map 同时作为渲染目标和纹理

```cpp
// ❌ 错误：还没解绑 FBO 就绑定纹理采样
glBindFramebuffer(GL_FRAMEBUFFER, depthMapFBO);
glBindTexture(GL_TEXTURE_2D, depthMap); // 错误！
```

### 陷阱 4：没切换视口

Shadow Map 和屏幕分辨率不同，Pass 1 和 Pass 2 之间必须重新设置 `glViewport`。

### 陷阱 5：正交投影包围盒太小

如果物体超出光空间正交投影的远平面，它就不会投射阴影。调试时可以临时把 shadow map 可视化到屏幕，检查覆盖范围。

# 纹理映射与UV坐标

## 问题 0：纯色三角形太丑了，怎么加细节？

在 [[Notes/计算机图形学/软光栅化与3D数学/光栅化：从直线到三角形|光栅化：从直线到三角形]] 中，我们用纯色填充三角形。但真实世界的物体表面有丰富的细节：

- 角色的皮肤纹理、衣物布料
- 木头的年轮、石头的裂纹
- 砖墙的缝隙、金属的划痕

**最 naive 的方案**：用更多的三角形来表达这些细节。

**发现的问题**：面数会爆炸。一个角色的毛孔如果每个都用三角形表示，可能需要几亿个面。

### 改进：纹理映射

**核心思想**：用一张 2D 图片（纹理）贴在三角形表面，代替纯色填充。

```
    3D 模型表面                    2D 纹理图片
┌─────────────┐                ┌─────────────┐
│  A        B │                │  A───────B  │
│   \      /  │      UV 映射    │   │ 皮肤  │  │
│    \    /   │  ───────────→  │   │ 纹理  │  │
│     \  /    │                │   │ 图片  │  │
│      C      │                │  C─────────  │
└─────────────┘                └─────────────┘
```

## 问题 1：UV 坐标是什么？

### 核心问题：3D 三角形是任意形状，纹理图片是矩形，怎么对应？

**答案**：每个顶点存储一个 **UV 坐标** `(u, v)`，表示该顶点在纹理图片上的位置。

```
纹理空间 (2D)：
   v↑
1.0├────────┐
   │        │
   │  纹理  │
   │  图片  │
0.0└────────┘→ u
   0.0    1.0
```

- `(0, 0)`：左下角（或左上角，取决于图形 API）
- `(1, 1)`：右上角
- 超出 `[0, 1]` 时，根据寻址模式处理（重复、拉伸、镜像等）

### 三角形内部怎么确定纹理颜色？——重心坐标插值

GPU 使用 [[Notes/计算机图形学/软光栅化与3D数学/光栅化：从直线到三角形|光栅化：从直线到三角形]] 中提到的**重心坐标**进行插值：

```
三角形内任意一点 P = αA + βB + γC
对应的 UV 坐标 (u, v) = α(uA,vA) + β(uB,vB) + γ(uC,vC)
```

根据插值后的 `(u, v)` 去纹理图片中采样颜色。

> 注意：透视投影后，UV 坐标不能直接线性插值，需要做**透视校正插值**（见 [[Notes/计算机图形学/软光栅化与3D数学/3D光栅化与深度缓冲|3D光栅化与深度缓冲]]）。

## 问题 2：UV 映射是怎么产生的？

### 关键认知：UV 映射是人为设计的

你可能以为 UV 坐标是自动计算出来的。实际上：**UV 映射是美术师设计的**，不是算法自动生成的。

```
3D 建模软件中的 UV 编辑：

┌─────────────────────────────┐      ┌─────────────────────────────┐
│      3D 模型（球体）          │      │     UV 编辑器（2D 平面）      │
│                             │  →   │                             │
│         ╭─────╮             │ 展   │    ┌─────────────────┐     │
│        ╱ 北极 ↑ ╲            │ 开   │    │  ╭──╮   ╭──╮   │     │
│       │    ●    │    ─────→ │ 操   │    │  │  │   │  │   │     │
│        ╲  南极 ↓ ╱            │ 作   │    │  ╰──╯   ╰──╯   │     │
│         ╰─────╯             │      │    │  展开的 UV 岛屿   │     │
│                             │      │    └─────────────────┘     │
└─────────────────────────────┘      └─────────────────────────────┘

就像把橘子皮剥下来摊平，需要切割和展开
```

### UV 展开的基本方法

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **平面投影** | 从一个方向"拍扁"到 2D | 简单形状、平面物体 |
| **圆柱投影** | 像包罐头标签一样包裹 | 柱状物体（手臂、腿）|
| **球形投影** | 像地球仪展开成地图 | 球状物体（头部、眼球）|
| **自动展开** | 算法切割网格，最小化变形 | 复杂形状 |

### 好的 UV 布局原则

```
好的 UV 布局：              坏的 UV 布局：
┌─────────────────┐        ┌─────────────────┐
│  ┌───┐ ┌───┐   │        │  ┌───┐          │
│  │头 │ │身 │   │        │  │头 │  大量     │
│  └───┘ └───┘   │        │  └───┘  空白     │
│  ┌───┐ ┌───┐   │        │       ┌───┐     │
│  │手 │ │腿 │   │        │       │手 │     │
│  └───┘ └───┘   │        │  浪费 │   │     │
│   均匀分布      │        │  空间 └───┘     │
│   利用率高      │        │                 │
└─────────────────┘        └─────────────────┘
```

## 问题 3：纹理采样中的走样问题

### 为什么远处纹理会闪烁？

当一个三角形在屏幕上很小（远处），但纹理分辨率很高时，一个屏幕像素可能覆盖纹理上的大片区域。只采样一个点会导致颜色剧烈变化 → **闪烁**。

```
屏幕像素（大）          纹理像素（小）
┌─────────┐            ┌─┬─┬─┬─┐
│         │            ├─┼─┼─┼─┤
│    ●    │  ───────→  ├─┼─┼─┼─┤  一个屏幕像素覆盖了16个纹理像素！
│         │            ├─┼─┼─┼─┤
└─────────┘            └─┴─┴─┴─┘
```

### 解决方案：Mipmap

**核心思想**：预计算多级分辨率的纹理，根据距离选择合适级别。

```
Level 0: 512×512（原始分辨率）
Level 1: 256×256（每 2×2 像素平均成 1 个）
Level 2: 128×128
Level 3: 64×64
...
Level N: 1×1
```

**GPU 怎么选择用哪一级？**

根据屏幕像素在纹理空间中覆盖的区域大小。覆盖越大（物体越远），选择越低级别的 Mipmap。

**三线性过滤（Trilinear Filtering）**：

在两个相邻的 Mipmap 级别之间做线性插值，避免级别切换时的跳变。

# 多重纹理与材质

## 问题 0：一张纹理到底够不够用？

在 [[Notes/计算机图形学/纹理系统/纹理对象与加载|纹理对象与加载]] 里，我们让一个立方体贴上了一张木箱图片。在 [[Notes/计算机图形学/纹理系统/纹理寻址与采样|纹理寻址与采样]] 里，我们又学会了控制 UV 越界行为和采样状态。

但现实里的材质远比“一张颜色图片”复杂：

- 水泥墙：基础颜色（Diffuse/Albedo）+ 表面凹凸细节（Normal）+ 粗糙度变化（Roughness）+ 缝隙里的阴影（AO）
- 生锈金属：基础颜色 + 法线 + 金属度（Metallic）+ 粗糙度 + 自发光（Emissive）
- 角色皮肤：漫反射颜色 + 法线 + 高光强度 + 次表面散射遮罩

**这些属性不是“一个像素的一种颜色”能表达的。** 现代材质通常需要 3~7 张不同的纹理，每张纹理在光照公式里扮演不同角色。

这就引出了本篇要解决的具体问题：

1. GPU 怎么同时让 Shader 读到多张纹理？
2. 这些纹理在 Shader 里怎么组织？
3. 法线贴图为什么需要 TBN 矩阵？
4. 引擎里怎么抽象“材质”这个概念？

---

## 问题 1：最 naive 的方案——把所有信息塞进一张纹理的 RGBA 四个通道

既然一张纹理有 RGBA 四个通道，那把 Diffuse 存 R、Normal 存 G、Roughness 存 B、Metallic 存 A 不就行了？

```glsl
#version 330 core
in vec2 vTexCoord;
uniform sampler2D uPackedTexture;
out vec4 FragColor;

void main() {
    vec4 packed = texture(uPackedTexture, vTexCoord);
    float diffuse = packed.r;   // 当成灰度颜色
    float normalX = packed.g;   // 法线被压成一维？
    float roughness = packed.b;
    float metallic = packed.a;
    // ... 这怎么算光照？
}
```

**立刻发现的问题**：

- 漫反射需要 3 通道颜色，你只剩 1 个通道给它，画面变成黑白
- 法线需要 3 通道方向向量，你只剩 2 个通道，方向信息丢失
- 不同材质需要的属性不同，强行打包导致语义混乱
- 美术没法独立编辑某一张贴图

**结论**：RGBA 通道打包只在“同一类数据的多通道”场景下合理（比如把 Normal 的 xyz 存 RGB），不能用来混装不同语义的贴图。

## 问题 2：那能不能在 Shader 里写多个 `sampler2D`？

可以。GLSL 允许一个 Shader 声明多个 `sampler2D` uniform，每个 sampler 从不同的纹理采样。但这里有一个关键问题：**这些 sampler 怎么知道去读哪张纹理？**

在 [[Notes/计算机图形学/纹理系统/纹理对象与加载|纹理对象与加载]] 里我们已经知道，Shader 里的 `sampler2D` 不是直接保存像素数据，而是保存一个**纹理单元索引**。单纹理时默认用纹理单元 0，所以看起来“绑了纹理就能读到”。多纹理时，我们必须显式告诉每个 sampler 用哪个单元。

---

## 问题 3：OpenGL 的纹理单元到底是什么？

纹理单元（Texture Unit）是 GPU 内部的一组“采样工位”。OpenGL 把它们暴露为 `GL_TEXTURE0`、`GL_TEXTURE1`、`GL_TEXTURE2`……

```
纹理单元 0：可以绑定一个 GL_TEXTURE_2D（或 CUBE_MAP、3D 等目标）
纹理单元 1：可以绑定一个 GL_TEXTURE_2D
纹理单元 2：可以绑定一个 GL_TEXTURE_2D
...
```

OpenGL 规范至少保证 **16 个纹理单元**（`GL_TEXTURE0` ~ `GL_TEXTURE15`），现代桌面 GPU 通常支持 **32 个甚至更多**。你可以用 `GL_MAX_COMBINED_TEXTURE_IMAGE_UNITS` 查询具体上限。

```cpp
int maxUnits = 0;
glGetIntegerv(GL_MAX_COMBINED_TEXTURE_IMAGE_UNITS, &maxUnits);
// 现代桌面 GPU 通常返回 32 或更高
```

> 注意：这个上限是“一个 Shader 能同时使用的纹理单元总数”，不是“整个 GPU 能创建的纹理对象总数”。纹理对象可以创建很多，但同一时刻一个 Shader 里只能同时采样这么多张。

## 问题 4：多纹理绑定的精确状态变化

假设我们要同时绑定两张纹理：

- `diffuseMap` 绑定到纹理单元 0
- `normalMap` 绑定到纹理单元 1

C++ 端代码：

```cpp
// 激活纹理单元 0，把 diffuse 纹理绑上去
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, diffuseMap);

// 激活纹理单元 1，把 normal 纹理绑上去
glActiveTexture(GL_TEXTURE1);
glBindTexture(GL_TEXTURE_2D, normalMap);

// 告诉 Shader：uDiffuse 用单元 0，uNormal 用单元 1
glUseProgram(shaderProgram);
glUniform1i(glGetUniformLocation(shaderProgram, "uDiffuse"), 0);
glUniform1i(glGetUniformLocation(shaderProgram, "uNormal"), 1);
```

**状态变化图**：

```
绑定前：
  纹理单元 0：空
  纹理单元 1：空
  Shader uDiffuse：未定义
  Shader uNormal：未定义

glActiveTexture(GL_TEXTURE0) + glBindTexture(diffuseMap) 后：
  纹理单元 0 ──→ diffuseMap

glActiveTexture(GL_TEXTURE1) + glBindTexture(normalMap) 后：
  纹理单元 0 ──→ diffuseMap
  纹理单元 1 ──→ normalMap

glUniform1i(uDiffuse, 0) + glUniform1i(uNormal, 1) 后：
  Shader uDiffuse ──→ 纹理单元 0 ──→ diffuseMap
  Shader uNormal  ──→ 纹理单元 1 ──→ normalMap
```

对应的片段着色器：

```glsl
#version 330 core
in vec2 vTexCoord;

uniform sampler2D uDiffuse;
uniform sampler2D uNormal;

out vec4 FragColor;

void main() {
    vec3 diffuse = texture(uDiffuse, vTexCoord).rgb;
    vec3 normal = texture(uNormal, vTexCoord).rgb;
    // 暂时先直接输出，验证两张纹理都能读到
    FragColor = vec4(diffuse * normal, 1.0);
}
```

> **关键规则**：`glActiveTexture` 切换的是“当前操作的纹理单元”，后续 `glBindTexture` 会作用到这个单元上。很多初学者忘了切单元，结果两张纹理都被绑到了单元 0，第二张覆盖了第一张。

## 问题 5：现代 GLSL 的 `layout(binding = N)` 能省掉 `glUniform1i` 吗？

可以。OpenGL 4.2+ 支持在 Shader 里直接声明 sampler 的绑定槽位：

```glsl
#version 420 core
layout(binding = 0) uniform sampler2D uDiffuse;
layout(binding = 1) uniform sampler2D uNormal;
layout(binding = 2) uniform sampler2D uMetallicRoughness;
```

这样 C++ 端只需要绑定纹理到对应单元，不需要再 `glUniform1i`：

```cpp
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, diffuseMap);
glActiveTexture(GL_TEXTURE1);
glBindTexture(GL_TEXTURE_2D, normalMap);
glActiveTexture(GL_TEXTURE2);
glBindTexture(GL_TEXTURE_2D, metallicRoughnessMap);
```

**状态变化图**：

```
Shader 编译/链接时：
  uDiffuse            ──→ 绑定到纹理单元 0
  uNormal             ──→ 绑定到纹理单元 1
  uMetallicRoughness  ──→ 绑定到纹理单元 2

运行时绑定纹理后：
  纹理单元 0 ──→ diffuseMap            ← uDiffuse 自动读这里
  纹理单元 1 ──→ normalMap             ← uNormal 自动读这里
  纹理单元 2 ──→ metallicRoughnessMap  ← uMetallicRoughness 自动读这里
```

**学习阶段建议**：
- 如果你用 OpenGL 3.3，只能用 `glUniform1i` 手动指定
- 如果你用 OpenGL 4.2+，推荐用 `layout(binding = N)`，代码更清晰，也更容易和引擎的材质槽位抽象对齐

## 问题 6：法线贴图到底在贴什么？

现在我们能绑定多张纹理了。假设第二张纹理是法线贴图（Normal Map），它看起来是奇怪的蓝紫色：

```glsl
vec3 normalFromMap = texture(uNormal, vTexCoord).rgb;
// normalFromMap 的每个分量在 [0,1] 范围
```

这张图里的 RGB 不是颜色，而是**法线向量**。但法线向量的每个分量范围是 `[-1, 1]`，而纹理像素只能存 `[0, 1]`，所以需要映射：

```
纹理像素值 0.0  → 法线分量 -1.0
纹理像素值 0.5  → 法线分量  0.0
纹理像素值 1.0  → 法线分量  1.0
```

转换公式：

```glsl
vec3 normal = normalFromMap * 2.0 - 1.0;
```

**但这里有个更深层的问题**：这个法线是在哪个坐标系下的？

- 如果法线贴图是在**模型空间**烘焙的，那它直接是世界/模型空间的绝对方向，可以直接用于光照
- 但工业界几乎不用模型空间法线贴图，因为它不能复用、不能镜像、不能动画

**工业标准做法**：法线贴图存的是**切线空间（Tangent Space）**下的法线扰动。

---

## 问题 7：为什么法线贴图要用切线空间？

切线空间是**贴在模型表面上的一个局部坐标系**：

- **T（Tangent，切线）**：指向 UV 的 u 方向
- **B（Bitangent，副切线）**：指向 UV 的 v 方向
- **N（Normal，法线）**：垂直于表面，指向外侧

```
        N ↑
          │
          │    模型表面
          │
          └──────────→ T
            B 指向屏幕外
```

法线贴图里存的是“相对于表面法线的微小扰动”。一张 predominantly 蓝色的法线贴图意味着大部分像素接近 `(0.5, 0.5, 1.0)`，也就是切线空间里“几乎不扰动，基本沿 N 方向”。

**切线空间的好处**：

- 同一张法线贴图可以复用到不同朝向的面上（砖墙、地面、圆柱都能用同一套砖缝法线）
- 模型可以旋转、缩放、动画，法线贴图仍然有效
- 可以镜像复用 UV，模型空间法线镜像后会出错

**代价**：Shader 里必须做切线空间 → 世界空间的转换，这就是 TBN 矩阵的由来。

---

## 问题 8：TBN 矩阵怎么构造？

TBN 矩阵由 T、B、N 三个向量组成：

```
TBN = [T  B  N]
```

它是一个 3×3 旋转矩阵，作用是把**切线空间**的向量变换到**世界空间**（或模型空间，取决于你构造它时用的坐标系）。

在顶点着色器里，我们需要把 T、N 从模型空间变换到世界空间（B 可以用 `cross(N, T)` 得到）：

```glsl
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec3 aNormal;
layout (location = 2) in vec2 aTexCoord;
layout (location = 3) in vec3 aTangent;

uniform mat4 uModel;
uniform mat4 uView;
uniform mat4 uProjection;

out vec2 vTexCoord;
out vec3 vWorldPos;
out mat3 vTBN;

void main() {
    vec4 worldPos = uModel * vec4(aPos, 1.0);
    vWorldPos = worldPos.xyz;
    vTexCoord = aTexCoord;

    // 把切线和法线变换到世界空间
    // 注意：方向向量 w=0，避免平移分量
    vec3 T = normalize(mat3(uModel) * aTangent);
    vec3 N = normalize(mat3(transpose(inverse(uModel))) * aNormal);

    // 用 Gram-Schmidt 正交化，确保 T 和 N 严格垂直
    T = normalize(T - dot(T, N) * N);

    // B 由 cross(N, T) 得到
    vec3 B = cross(N, T);

    vTBN = mat3(T, B, N);

    gl_Position = uProjection * uView * worldPos;
}
```

对应的片段着色器：

```glsl
#version 330 core
in vec2 vTexCoord;
in vec3 vWorldPos;
in mat3 vTBN;

uniform sampler2D uDiffuse;
uniform sampler2D uNormal;
uniform vec3 uLightPos;
uniform vec3 uViewPos;

out vec4 FragColor;

void main() {
    // 1. 从法线贴图读取切线空间法线
    vec3 tangentNormal = texture(uNormal, vTexCoord).rgb * 2.0 - 1.0;

    // 2. 用 TBN 矩阵把切线空间法线转到世界空间
    vec3 worldNormal = normalize(vTBN * tangentNormal);

    // 3. 漫反射颜色
    vec3 diffuseColor = texture(uDiffuse, vTexCoord).rgb;

    // 4. 简单 Blinn-Phong 光照
    vec3 lightDir = normalize(uLightPos - vWorldPos);
    vec3 viewDir = normalize(uViewPos - vWorldPos);
    vec3 halfwayDir = normalize(lightDir + viewDir);

    vec3 ambient = 0.1 * diffuseColor;
    float diff = max(dot(worldNormal, lightDir), 0.0);
    float spec = pow(max(dot(worldNormal, halfwayDir), 0.0), 32.0);

    vec3 color = ambient + diff * diffuseColor + spec * vec3(1.0);
    FragColor = vec4(color, 1.0);
}
```

**为什么 TBN 矩阵长这样？**

想象切线空间里有一个向量 `(1, 0, 0)`，它应该被映射到世界空间中的 T 方向。所以 TBN 矩阵的第一列就是 T。同理：

```
TBN * (1, 0, 0) = T
TBN * (0, 1, 0) = B
TBN * (0, 0, 1) = N
```

因此 `TBN = mat3(T, B, N)`。

## 问题 9：顶点切线数据从哪来？

切线不是凭空出现的。它需要由模型导入器或 DCC 工具（Blender/Maya）计算，并随顶点数据一起上传到 GPU。

计算切线的核心思路：对三角形的边向量和 UV 差分求解线性方程组。给定三角形三个顶点 `p0, p1, p2` 和对应 UV `uv0, uv1, uv2`：

```
e1 = p1 - p0 = (u1 - u0) * T + (v1 - v0) * B
e2 = p2 - p0 = (u2 - u0) * T + (v2 - v0) * B
```

解这个方程组得到 T 和 B。一个顶点如果被多个三角形共享，通常把这些三角形的切线加权平均作为该顶点的切线。

在引擎里，切线通常和位置、法线、UV 一起作为顶点属性：

```cpp
struct Vertex {
    float pos[3];
    float normal[3];
    float tangent[3];  // ← 新增
    float uv[2];
};
```

> 注意：OpenGL 默认没有自动计算切线的功能。要么在资源导入时由工具生成，要么在引擎启动时自己算。简单项目可以用 assimp/stb 等库；自研引擎建议在资源导入管线里离线计算好。

## 问题 10：材质到底该怎么抽象？

现在我们有能力同时绑定多张纹理了。但如果每个 Shader 都手写 `glActiveTexture + glBindTexture + glUniform1i`，代码会很快爆炸。引擎需要一个**材质（Material）**抽象。

**材质的最小定义**：

```
Material = Shader + 纹理槽位绑定表 + Uniform 参数集
```

例如一个 Standard PBR 材质：

```cpp
struct Material {
    ShaderHandle shader;

    // 纹理槽位：材质属性名 → （纹理单元索引，纹理对象）
    HashMap<StringId, TextureBinding> textureSlots;

    // Uniform 参数：颜色、粗糙度缩放、金属度缩放等
    Vec4 baseColorTint;
    float normalStrength;
    float metallicScale;
    float roughnessScale;
};

struct TextureBinding {
    int unit;          // 绑定到哪个纹理单元
    GpuTexture texture;
    SamplerDesc sampler;
};
```

渲染时，材质系统的工作流程：

```
1. glUseProgram(material.shader)
2. 遍历 material.textureSlots：
     glActiveTexture(GL_TEXTURE0 + unit)
     glBindTexture(GL_TEXTURE_2D, texture.id)
     glBindSampler(unit, sampler)  // 如果有 Sampler Object
     glUniform1i(shaderLoc("slotName"), unit)  // 如果没用 layout(binding)
3. 上传 Uniform 参数：
     glUniform4f(baseColorTintLoc, ...)
     glUniform1f(normalStrengthLoc, ...)
4. 绘制
```

**为什么这样抽象？**

- 美术只关心“这个材质用了哪些贴图、哪些参数”，不关心纹理单元编号
- 同一张纹理可以被多个材质引用到不同槽位（配合 Sampler Object 实现不同采样状态）
- 切换材质时，只需要重新绑定该材质声明的纹理槽位，不用碰其他状态

## 问题 11：PBR 材质有哪些标准槽位？

工业界最常用的是 **Metallic-Roughness 工作流**（glTF 标准）。它的核心思想是：把材质拆成几张“属性图”，每张图回答一个独立问题——

- **Albedo**：物体本身是什么颜色？（不含光照）
- **Normal**：表面微观凹凸怎么影响法线？
- **Metallic**：这是金属还是非金属？
- **Roughness**：表面光滑还是粗糙？
- **Ambient Occlusion**：缝隙里是不是更暗？
- **Emissive**：哪些部分自己在发光？

这些属性分别对应不同的纹理槽位：

| 槽位 | 用途 | 颜色空间 | 典型通道 |
|------|------|----------|----------|
| Albedo / Base Color | 物体本身颜色 | sRGB | RGB，A 可选透明度 |
| Normal | 表面法线扰动 | Linear | RGB 存切线空间法线 |
| Metallic | 金属度 | Linear | 单通道 R，0=非金属，1=金属 |
| Roughness | 粗糙度 | Linear | 单通道 R，0=光滑，1=粗糙 |
| Ambient Occlusion | 环境光遮蔽 | Linear | 单通道 R |
| Emissive | 自发光颜色 | sRGB | RGB |

**槽位数量 vs 纹理单元限制**：

如果每个槽位都用一张独立纹理，一个 PBR 材质就要 6 张纹理。纹理单元有 16~32 个，单个材质看起来够，但复杂场景里 Shader 可能还要采样阴影贴图、环境贴图、LUT 等，纹理单元会变得紧张。

**工业优化：通道打包（Channel Packing）**

把多个单通道纹理合并到一张纹理的不同通道里：

```
ORM 纹理：
  R 通道 = Ambient Occlusion
  G 通道 = Roughness
  B 通道 = Metallic
  A 通道 = 可选 Height / Emission Mask
```

这样 6 个槽位可以压缩到 3~4 张纹理：Albedo、Normal、ORM、Emissive。

```glsl
vec4 orm = texture(uORM, vTexCoord);
float ao = orm.r;
float roughness = orm.g;
float metallic = orm.b;
```

> 这是**空间换带宽**的优化：一张 RGBA 纹理和一张 R 纹理的内存占用相同（都是 4 字节/像素），但 Shader 只需采样一次就能拿到 3 个参数，减少了纹理采样次数和绑定压力。

## 问题 12：纹理单元不够了怎么办？

虽然现代 GPU 有 32+ 个纹理单元，但以下场景会很快吃光：

- Deferred / Clustered 光照需要采样多张 G-Buffer 纹理
- 阴影贴图（Shadow Map）会占 1~4 个单元
- 环境光遮蔽、反射探针、LUT、Noise 贴图各占单元
- 地形混合多层材质，每层都需要 Albedo + Normal + ORM

**应对方案**（按推荐度排序）：

1. **通道打包**：把单通道数据合并到 RGBA 纹理，减少 sampler 数量
2. **纹理数组（Texture Array）**：同一类纹理（如地形多层）放进数组，用一个 sampler 采样不同索引
3. **Bindless Textures / Descriptor Indexing**：现代 API（Vulkan/D3D12）支持运行时从一个大纹理表里按索引采样，OpenGL 4.6 也有 `GL_ARB_bindless_texture`。这是工业主流方向
4. **减少同时活跃的 sampler**：通过 RenderPass 拆分，每个 Pass 只绑定它需要的纹理

**个人项目建议**：
- 学习阶段：用手动 `glActiveTexture + glBindTexture + layout(binding = N)`，完全够用到阶段九
- 引擎阶段：把“纹理槽位”抽象成 Material Asset 的配置，内部按上述策略优化
- 向现代 API 迁移时：把纹理单元 mentally map 为 DescriptorSet / Root Signature 里的纹理描述符索引

## 完整可运行示例：Diffuse + Normal 双纹理材质

```cpp
// flags: -O0 -g -lGL -lGLEW -lglfw
#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <iostream>
#include <glm/glm.hpp>
#include <glm/gtc/matrix_transform.hpp>
#include <glm/gtc/type_ptr.hpp>

#define STB_IMAGE_IMPLEMENTATION
#include <stb_image.h>

GLuint loadTexture(const char* path, bool sRGB = true) {
    stbi_set_flip_vertically_on_load(true);
    int w, h, c;
    unsigned char* pixels = stbi_load(path, &w, &h, &c, 0);
    if (!pixels) { std::cerr << "Failed to load " << path << std::endl; return 0; }

    GLuint tex;
    glGenTextures(1, &tex);
    glBindTexture(GL_TEXTURE_2D, tex);

    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_REPEAT);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR_MIPMAP_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

    glPixelStorei(GL_UNPACK_ALIGNMENT, 1);

    GLenum internal = (c == 4) ? (sRGB ? GL_SRGB8_ALPHA8 : GL_RGBA) : (sRGB ? GL_SRGB8 : GL_RGB);
    GLenum format   = (c == 4) ? GL_RGBA : GL_RGB;

    glTexImage2D(GL_TEXTURE_2D, 0, internal, w, h, 0, format, GL_UNSIGNED_BYTE, pixels);
    glGenerateMipmap(GL_TEXTURE_2D);

    stbi_image_free(pixels);
    return tex;
}

const char* vs = R"(
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec3 aNormal;
layout (location = 2) in vec2 aTexCoord;
layout (location = 3) in vec3 aTangent;

uniform mat4 uModel;
uniform mat4 uView;
uniform mat4 uProjection;

out vec2 vTexCoord;
out vec3 vWorldPos;
out mat3 vTBN;

void main() {
    vec4 worldPos = uModel * vec4(aPos, 1.0);
    vWorldPos = worldPos.xyz;
    vTexCoord = aTexCoord;

    vec3 N = normalize(mat3(transpose(inverse(uModel))) * aNormal);
    vec3 T = normalize(mat3(uModel) * aTangent);
    T = normalize(T - dot(T, N) * N);
    vec3 B = cross(N, T);

    vTBN = mat3(T, B, N);
    gl_Position = uProjection * uView * worldPos;
}
)";

const char* fs = R"(
#version 330 core
in vec2 vTexCoord;
in vec3 vWorldPos;
in mat3 vTBN;

layout(binding = 0) uniform sampler2D uDiffuse;
layout(binding = 1) uniform sampler2D uNormal;

uniform vec3 uLightPos;
uniform vec3 uViewPos;

out vec4 FragColor;

void main() {
    vec3 diffuseColor = texture(uDiffuse, vTexCoord).rgb;

    vec3 tangentNormal = texture(uNormal, vTexCoord).rgb * 2.0 - 1.0;
    vec3 worldNormal = normalize(vTBN * tangentNormal);

    vec3 lightDir = normalize(uLightPos - vWorldPos);
    vec3 viewDir = normalize(uViewPos - vWorldPos);
    vec3 halfwayDir = normalize(lightDir + viewDir);

    vec3 ambient = 0.1 * diffuseColor;
    float diff = max(dot(worldNormal, lightDir), 0.0);
    float spec = pow(max(dot(worldNormal, halfwayDir), 0.0), 32.0);

    FragColor = vec4(ambient + diff * diffuseColor + spec * vec3(0.5), 1.0);
}
)";

// 立方体顶点：位置、法线、UV、切线（为每个面单独展开，避免共享顶点法线/切线冲突）
// 这里只列出前面和顶面作为示意，完整实现需要 36 个顶点
// ...

int main() {
    // ... GLFW/GLEW 初始化、编译 Shader、创建 VAO/VBO ...

    GLuint diffuseMap = loadTexture("brickwall.jpg", true);
    GLuint normalMap  = loadTexture("brickwall_normal.jpg", false);

    // 渲染循环中
    glActiveTexture(GL_TEXTURE0);
    glBindTexture(GL_TEXTURE_2D, diffuseMap);
    glActiveTexture(GL_TEXTURE1);
    glBindTexture(GL_TEXTURE_2D, normalMap);

    // 因为用了 layout(binding = N)，不需要 glUniform1i
    glUseProgram(program);
    // ... 设置 MVP / light / view uniform ...
    glDrawArrays(GL_TRIANGLES, 0, 36);

    return 0;
}
```

## 常见问题诊断

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 第二张纹理显示不出来 | 忘了 `glActiveTexture` 切换单元 | 每次绑定前调用 `glActiveTexture` |
| 法线贴图效果错误/闪烁 | TBN 矩阵没有正交化或没有归一化 | 用 Gram-Schmidt 正交化，normalize T/N/B |
| 法线贴图让表面“整体变蓝/变紫” | 把法线贴图颜色直接当法线用了 | 先做 `* 2.0 - 1.0` 再乘以 TBN |
| 镜像 UV 处法线方向反了 | 切线 handedness 没处理 | 用 vec4 tangent（xyz + w 表示手性），`B = cross(N, T) * w` |
| 纹理单元报错/黑图 | 超过 GPU 最大纹理单元数 | 用 `glGetIntegerv` 查询上限，或用通道打包 |

## 与 SelfGameEngine 的关系

这篇笔记对应引擎 **阶段 5.3 材质系统** 的最核心抽象。

在引擎中，你不会直接写 `glActiveTexture`，而是定义一个 `MaterialAsset`：

```cpp
struct MaterialAsset {
    ShaderHandle shader;

    // 纹理槽位：由 Shader 的 layout(binding = N) 决定 unit
    TextureHandle albedoMap;
    TextureHandle normalMap;
    TextureHandle ormMap;       // Occlusion/Roughness/Metallic 打包
    TextureHandle emissiveMap;

    // 可调参数
    Vec4 baseColorTint;
    float normalStrength;
    float metallicScale;
    float roughnessScale;
    float emissiveScale;
};
```

渲染系统绘制时：

```cpp
void RenderSystem::ApplyMaterial(const MaterialAsset& material) {
    glUseProgram(material.shader.program);

    if (material.albedoMap)   BindTexture(0, material.albedoMap,   material.albedoSampler);
    if (material.normalMap)   BindTexture(1, material.normalMap,   material.normalSampler);
    if (material.ormMap)      BindTexture(2, material.ormMap,      material.ormSampler);
    if (material.emissiveMap) BindTexture(3, material.emissiveMap, material.emissiveSampler);

    glUniform4f(baseColorTintLoc, material.baseColorTint...);
    glUniform1f(normalStrengthLoc, material.normalStrength);
    // ...
}
```

你现在写的：
- `glActiveTexture + glBindTexture` → 未来 `RenderDevice::BindTexture(slot, texture)`
- `layout(binding = N)` → 未来 Shader 和 Material Asset 之间的**槽位契约**
- `Material = Shader + Texture Slots + Uniform Params` → 引擎材质系统的最小可运行原型

## API 对照：我们在解决“多纹理绑定”这个问题

| 概念 | OpenGL | Vulkan | D3D12 |
|------|--------|--------|-------|
| 纹理单元/描述符槽位 | `GL_TEXTURE0 + N` / `layout(binding = N)` | `DescriptorSet` 中的 binding | `Root Signature` 中的 descriptor table slot |
| 绑定纹理 | `glActiveTexture + glBindTexture` | `vkCmdBindDescriptorSets` | `SetGraphicsRootDescriptorTable` |
| 采样器状态 | `Sampler Object` / `glTexParameter` | `VkSampler` + `VkDescriptorSet` | `D3D12_SAMPLER_DESC` + Root Signature |
| 绕过单元限制 | `GL_ARB_bindless_texture` | `Descriptor Indexing` | `Bindless Resources` |

**个人项目推荐**：
- 学习阶段：OpenGL 手动绑定 + `layout(binding = N)`，清晰可控
- 引擎阶段：把纹理槽位抽象成 Material Asset 的字段，渲染系统自动绑定
- 现代 API 迁移：把“纹理单元编号”对应到 DescriptorSet 的 binding index

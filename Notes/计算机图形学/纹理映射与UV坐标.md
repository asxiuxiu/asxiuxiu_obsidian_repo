# 纹理映射与 UV 坐标

> [!info] 前置知识
> 本文详细解释 3D 模型如何贴上 2D 纹理，包括 UV 坐标的原理、生成方法和实际工作流程。
> 
> 快速回顾：[[3D图形学概览|3D 图形学概览]] 中对纹理的基本介绍。

---

## Why：为什么要学习纹理映射？

在 [[光栅化 - 从直线到三角形|2D 光栅化]] 中，我们用纯色填充三角形。但真实世界的物体表面有丰富的细节：

- 角色的皮肤纹理、衣物布料
- 木头的年轮、石头的裂纹
- 砖墙的缝隙、金属的划痕

如果试图用更多的三角形来表达这些细节，面数会爆炸。**纹理映射**让我们用一张 2D 图片来给 3D 模型"化妆"，既节省几何数据，又能呈现丰富的表面细节。

---

## What：纹理映射的核心概念

### 外观数据：让三角形有颜色、有质感

如果只有几何数据，模型就是一堆白色的线框。我们需要**外观数据**来让它看起来像真实物体。

| 概念                | 通俗解释                | 作用                      |
| ----------------- | ------------------- | ----------------------- |
| **纹理 (Texture)**  | 一张贴在模型表面的 2D 图片     | 给模型添加细节，比如皮肤、木纹、砖墙      |
| **贴图 (Map)**      | 纹理的另一种说法，或特指某种用途的纹理 | 颜色贴图、法线贴图、粗糙度贴图等        |
| **材质 (Material)** | 一组"表面属性"的参数包        | 定义颜色、反光程度、透明度、金属感等      |
| **纹理坐标 (UV)**     | 2D 图片上的坐标 `(u, v)`  | 告诉计算机"三角形的这个角对应图片的哪个像素" |

```
        3D 模型表面                    2D 纹理图片
    ┌─────────────┐                ┌─────────────┐
    │  A        B │                │  A───────B  │
    │   \      /  │      UV 映射    │   │ 皮肤  │  │
    │    \    /   │  ───────────→  │   │ 纹理  │  │
    │     \  /    │                │   │ 图片  │  │
    │      C      │                │  C─────────  │
    └─────────────┘                └─────────────┘
```

### UV 映射的工作原理

你可能会有疑问：**3D 三角形是任意形状，纹理图片是矩形，怎么对应？**

这就是 UV 坐标的精妙之处：

```
3D 空间中的三角形              纹理空间 (2D) 中的对应区域
       y↑                           v↑
        |   C                      1.0┃      C'(u₂,v₂)
        |  /╲                          |     /╲
        | /  ╲                         |    /  ╲
        |/    ╲                        |   /    ╲
       A━━━━━━B                     0.0━━A'━━━━B'━━━━→ u
                                      0.0         1.0
                                      
    每个顶点存储 (x,y,z) + (u,v)      三个顶点对应纹理上的三个点
    形成一个 3D → 2D 的映射           构成一个三角形区域
```

**三角形内部如何确定颜色？插值！**

GPU 使用[[Notes/数学基础/线性组合与张成空间与基|重心坐标]]进行插值：
- 三角形内任意一点 $P = \alpha A + \beta B + \gamma C$
- 对应的 UV 坐标 $(u,v) = \alpha(u_A,v_A) + \beta(u_B,v_B) + \gamma(u_C,v_C)$
- 根据插值后的 $(u,v)$ 去纹理图片中采样颜色

> [!note] 一句话理解
> UV 坐标像是给 3D 顶点打"邮戳"，告诉它应该显示纹理的哪个位置。三角形内部通过**重心坐标插值**自动过渡，所以任意形状的 3D 三角形都能贴上矩形纹理。

**UV 坐标范围**：通常 $u, v \in [0, 1]$，表示纹理的相对位置
- $(0, 0)$：左下角（或左上角，取决于图形 API）
- $(1, 1)$：右上角
- 超出 $[0,1]$ 时，根据寻址模式处理（重复平铺、边缘拉伸等）

## How：UV 映射是如何生成的？

### UV 映射不是自动计算的

你可能还会问：**我怎么知道顶点 A 应该对应 UV (0.1, 0.2)，而不是 (0.5, 0.8)？**

答案是：**UV 映射是人为设计的**，不是自动计算出来的。

```
3D 建模软件中的 UV 编辑过程：

┌─────────────────────────────┐      ┌─────────────────────────────┐
│      3D 模型（球体）          │      │     UV 编辑器（2D 平面）      │
│                             │      │                             │
│         ╭─────╮             │  →   │    ┌─────────────────┐     │
│        ╱ 北极 ↑ ╲            │ 展   │    │  ╭──╮   ╭──╮   │     │
│       │    ●    │    ─────→ │ 开   │    │  │  │   │  │   │     │
│        ╲  南极 ↓ ╱            │ 操   │    │  ╰──╯   ╰──╯   │     │
│         ╰─────╯             │ 作   │    │  展开的 UV 岛屿   │     │
│                             │      │    └─────────────────┘     │
└─────────────────────────────┘      └─────────────────────────────┘

就像把橘子皮剥下来摊平，需要切割和展开
```

### UV 展开的基本方法

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **平面投影** | 从一个方向"拍扁"到 2D | 简单形状、平面物体 |
| **圆柱投影** | 像包罐头标签一样包裹 | 柱状物体（手臂、腿）|
| **球形投影** | 像地球仪展开成地图 | 球状物体（头部、眼球）|
| **自动展开** | 算法切割网格，最小化变形 | 复杂形状 |

### 实际工作流程（以 Blender/Maya 为例）

1. **自动展开**：软件根据几何形状自动生成初始 UV
2. **手动调整**：美术师在 UV 编辑器中拖动顶点，调整布局
3. **优化排布**：把各个"UV 岛屿"排列在 $[0,1]$ 范围内，最大化利用纹理空间
4. **绘制纹理**：在 Photoshop/Substance 中根据 UV 布局绘制贴图

```
好的 UV 布局 vs 坏的 UV 布局：

┌─────────────────┐          ┌─────────────────┐
│  ┌───┐ ┌───┐   │          │  ┌───┐          │
│  │头 │ │身 │   │          │  │头 │  大量     │
│  └───┘ └───┘   │          │  └───┘  空白     │
│  ┌───┐ ┌───┐   │          │       ┌───┐     │
│  │手 │ │腿 │   │          │       │手 │     │
│  └───┘ └───┘   │          │  浪费 │   │     │
│   均匀分布      │          │  空间 └───┘     │
│   利用率高      │          │                 │
└─────────────────┘          └─────────────────┘
```

> [!tip] 关键理解
> - **顶点位置 (x,y,z)**：描述 3D 形状 → 由建模师雕刻/调整
> - **纹理坐标 (u,v)**：描述如何贴图 → 也由建模师设计/调整
> 
> 两者都是模型数据的一部分，存储在同一个模型文件中（如 `.obj`、`.fbx`）

## 从建模到贴图的完整流程

了解了原理，那实际工作中是怎么做的呢？

```
完整的 3D 美术工作流程：

步骤 1: 建模（ sculpting / modeling ）
┌─────────────────────────────────────────┐
│  基础形状（立方体/球体）                  │
│        ↓                                │
│  雕刻/拉伸/切割 → 高模（高面数）          │
│        ↓                                │
│  拓扑重绘 → 低模（游戏用，面数优化）       │
└─────────────────────────────────────────┘
                    ↓
步骤 2: UV 展开（ UV Unwrapping ）
┌─────────────────────────────────────────┐
│  标记缝合边（Seam）← 告诉软件从哪里"切开" │
│        ↓                                │
│  自动展开 → 得到初始 UV 布局              │
│        ↓                                │
│  手动调整：拖动、旋转、缩放 UV 岛屿         │
│        ↓                                │
│  打包：排列在 [0,1] 范围内，最大化利用率    │
└─────────────────────────────────────────┘
                    ↓
步骤 3: 纹理绘制（ Texturing ）
┌─────────────────────────────────────────┐
│  导出 UV 线框图（Wireframe）              │
│        ↓                                │
│  在 Photoshop / Substance 中绘制纹理      │
│  或：直接在 3D 模型上绘制（3D Coat等）      │
│        ↓                                │
│  生成：颜色贴图、法线贴图、粗糙度贴图等      │
└─────────────────────────────────────────┘
                    ↓
            导出到游戏引擎（Unity/Unreal）
```

### 如何验证贴得对不对？

建模师有几种方法来检查 UV 是否正确：

```
方法 1: UV 可视化检查
┌─────────────────────────────┐
│   在 3D 视图中显示 UV 网格    │
│                             │
│    ╭────╮                   │
│   ╱▓▓▓▓╲   ← 棋盘格图案      │
│   ▓▓▓▓▓▓▓     均匀 = UV 好   │
│   ╲▓▓▓▓╱      扭曲 = UV 有问题│
│    ╰────╯                   │
└─────────────────────────────┘

方法 2: 棋盘格纹理测试
- 给模型贴上一张棋盘格图案
- 如果格子大小均匀 → UV 展开得好
- 如果有拉伸/压缩 → 需要调整 UV

方法 3: 着色器可视化
- 把 UV 坐标直接输出为颜色（R=u, G=v）
- 肉眼可见 UV 分布是否均匀
```

## 深入：UV 缝合边（Seam）

### 什么是缝合边？

展开 3D 模型就像剥橘子皮，需要选择从哪里"切开"：

```
3D 模型上的缝合边              展开后的 UV 岛屿
┌─────────────────┐           ┌─────────────────┐
│      ●━━●       │           │  ┌───┐ ┌───┐   │
│     /│  │\      │  切开     │  │前 │ │后 │   │
│    ●─●  ●─●     │  ─────→  │  └───┘ └───┘   │
│    │      │     │           │  ┌───┐ ┌───┐   │
│    ●━━━━━━●     │           │  │左 │ │右 │   │
│      立方体      │           │  └───┘ └───┘   │
│  红色 = 缝合边    │           │  四个面展开    │
└─────────────────┘           └─────────────────┘

缝合边选得不好 → 展开后有严重拉伸
缝合边选得好 → 展开后形状规整、利用率高
```

### 缝合边的选择原则

**缝合边不是随意画的**，有经验的美术师会遵循以下原则：

| 原则 | 说明 | 示例 |
|------|------|------|
| **藏在隐蔽处** | 缝合边会出现在纹理的"切口"，尽量放在看不见的地方 | 角色放在头后、腋下、裤缝 |
| **顺着自然边界** | 沿模型的结构线切开 | 衣服的边缘、装甲的接缝 |
| **减少扭曲** | 优先切开能让展开更平整的区域 | 圆柱侧面切一条纵向边 |
| **控制岛屿数量** | 太多碎片不好管理，太少会有拉伸 | 人体通常 3~6 个 UV 岛屿 |

### 缝合边 vs "绕一圈"

你提到的"从一个点出发绕一圈"，可能是指**圆柱投影**的情况：

```
柱状物体（如手臂）的 UV 展开：

      3D 手臂                  UV 展开后
    ┌───────┐                ┌─────────────┐
   ╱   ↑   ╲               ┌─┤  缝合边     │
  │  切开线  │   ─────→     │ │   （切开）   │
  │    │    │               │ │            │
   ╲   ↓   ╱                │ │   展开成    │
    └───────┘               │ │   矩形      │
   圆柱投影需要在            │ │            │
   侧面标一条                └─┤            │
   纵向缝合边                 └─────────────┘
   
   这条缝合边 "绕" 了一圈，把圆柱切成一个平面
```

但这只是圆柱投影的特殊情况。一般复杂的模型（如角色），缝合边是**多条独立的边**，不是必须绕圈。

## 真实案例：游戏角色的 UV 展开

一个游戏角色通常需要 1~4 张 2048×2048 的纹理：

| 部位 | UV 策略 | 说明 |
|------|---------|------|
| **头部** | 单独的 UV 岛屿 | 面部需要最高分辨率 |
| **躯干** | 一个大的 UV 区域 | 前胸后背连贯处理 |
| **四肢** | 手臂、腿分别展开 | 像展开袖子、裤腿 |
| **手/脚** | 可能单独处理 | 细节丰富，需要更多空间 |

**UV 展开的核心目标**：
- 重要部位（脸、手）获得更多纹理空间
- 不太重要的部位（背部、腿部内侧）可以少一些
- 缝合边尽量藏在隐蔽处

# 变换续——视图与投影

## 问题 0：模型摆好了，怎么"拍照"？

在 [[Notes/计算机图形学/软光栅化与3D数学/3D变换与齐次坐标|3D变换与齐次坐标]] 中，我们用模型矩阵把物体从局部坐标摆到了世界坐标。但 3D 场景里有无数物体，屏幕只有那么大。

**核心问题**：怎么把 3D 世界"拍"成 2D 照片？

需要两个变换：
1. **视图变换（View）**：把相机架到标准位置
2. **投影变换（Projection）**：把 3D 场景"拍扁"成 2D

---

## 问题 1：视图变换——怎么把相机移回原点？

### 核心观察

如果相机和物体的**相对位置**不变，渲染结果就不变。

**策略**：把相机移动到标准位置，其他物体跟着一起动。

**标准相机配置**：
- 位置：原点 `(0, 0, 0)`
- 朝向：`-Z` 方向（看向屏幕里面）
- 上方：`+Y` 方向

### 用三个参数定义相机

| 参数 | 符号 | 含义 |
|-----|------|------|
| 位置 | `eye` | 相机在哪 |
| 朝向 | `target` | 看向哪里 |
| 上方 | `up` | 相机的上方向 |

### 视图矩阵推导

**步骤 1：平移相机到原点**

```
T_view = Translate(-eye.x, -eye.y, -eye.z)
```

**步骤 2：旋转相机到标准朝向**

需要把相机的坐标轴 `(right, up, -forward)` 旋转到世界坐标轴 `(X, Y, Z)`。

先计算相机的三个轴：

```cpp
Vec3 forward = Normalize(target - eye);    // 相机看向的方向
Vec3 right   = Normalize(Cross(forward, up)); // 右方向
Vec3 newUp   = Cross(right, forward);       // 重新计算正交的上方向
```

视图矩阵：

```cpp
Mat4 LookAt(Vec3 eye, Vec3 target, Vec3 up) {
    Vec3 f = Normalize(target - eye);
    Vec3 r = Normalize(Cross(f, up));
    Vec3 u = Cross(r, f);
    
    Mat4 view = {
        { r.x,  r.y,  r.z, -Dot(r, eye) },
        { u.x,  u.y,  u.z, -Dot(u, eye) },
        { -f.x, -f.y, -f.z, Dot(f, eye) },
        { 0,    0,    0,    1           }
    };
    return view;
}
```

> 记忆要点：先平移再旋转，矩阵乘法从右到左。`view = R × T`。

## 问题 2：投影——怎么把 3D"拍扁"成 2D？

### 两种投影方式

| 类型 | 特点 | 应用 |
|-----|------|------|
| **正交投影** | 平行线保持平行，无近大远小 | 工程制图、2D 游戏、策略游戏小地图 |
| **透视投影** | 近大远小，符合人眼视觉 | 3D 游戏、写实渲染 |

### 正交投影

本质：**先平移中心到原点，再缩放到 `[-1, 1]` 的标准范围**。

```
M_ortho = S_ortho × T_ortho

其中：
T_ortho = Translate(-(r+l)/2, -(t+b)/2, -(n+f)/2)
S_ortho = Scale(2/(r-l), 2/(t-b), 2/(n-f))
```

> 注意：`n > f`（相机看向 `-Z`，近平面离相机更近，`z` 值更大）

### 透视投影（核心难点）

透视投影的目标：把**视锥体**（金字塔形状）映射到**标准立方体** `[-1, 1]³`。

#### 核心思想：两步走

1. **挤压**：把视锥体"挤压"成一个长方体
2. **正交投影**：把长方体映射到标准立方体（这个我们已经会了）

#### 为什么"挤压"能用矩阵表示？

直接做需要除以 `z`：

```
x' = (n/z) × x
y' = (n/z) × y
```

这是**非线性**操作，普通矩阵无法实现。

**齐次坐标的巧妙**：把 `z` 存入 `w` 分量，等矩阵运算结束后再做**透视除法**（除以 `w`）。

```
4D点 (x, y, z, 1)
    │ 乘以 4×4 矩阵（纯线性运算）
    ▼
4D点 (nx, ny, Az+B, z)  ← z 被存入了 w 分量
    │ 透视除法（除以 w=z）
    ▼
3D点 (nx/z, ny/z, (Az+B)/z)
```

#### 挤压矩阵

利用两个约束条件求解第三行：
- 近平面 `z=n` 不变 → `An + B = n²`
- 远平面中心 `z=f` 不变 → `Af + B = f²`

解得：`A = n + f`，`B = -nf`

```
| n   0   0    0 |
| 0   n   0    0 |
| 0   0  n+f  -nf |
| 0   0   1    0 |
```

#### 完整透视投影矩阵

```
M_persp = M_ortho × M_persp→ortho
```

用 FOV 和 Aspect 定义视锥体：

```cpp
Mat4 Perspective(float fovY, float aspect, float near, float far) {
    float top = near * tan(fovY * PI / 360.0f);
    float right = top * aspect;
    
    Mat4 persp = {
        { near/right, 0,          0,                      0 },
        { 0,          near/top,   0,                      0 },
        { 0,          0,          (near+far)/(near-far),  2*near*far/(near-far) },
        { 0,          0,         -1,                      0 }
    };
    return persp;
}
```

## 问题 3：MVP 变换的完整链路

```
模型空间 ──[M]──→ 世界空间 ──[V]──→ 相机空间 ──[P]──→ 裁剪空间 ──[透视除法]──→ NDC ──[视口]──→ 屏幕空间
 (局部)            (全局)            (视角)            [-w,w]³            [-1,1]³            [0,width]
                                                                                         [0,height]
```

```cpp
// 完整变换
Mat4 MVP = P * V * M;  // 从右往左：先 Model，再 View，再 Projection
Vec4 clip = MVP * Vec4(x, y, z, 1);
Vec3 ndc = Vec3(clip.x, clip.y, clip.z) / clip.w;  // 透视除法
Vec2 screen = Vec2(
    (ndc.x + 1) * 0.5 * width,
    (ndc.y + 1) * 0.5 * height
);
```

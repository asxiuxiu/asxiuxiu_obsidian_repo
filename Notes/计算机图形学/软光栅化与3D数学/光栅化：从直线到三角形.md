# 光栅化——从直线到三角形

## 问题 0：手动设置每个像素太痛苦了

在 [[Notes/计算机图形学/像素与渲染管线全貌/像素、帧缓冲与渲染管线全貌|像素、帧缓冲与渲染管线全貌]] 里，你写了一个双重循环，手动设置每个像素的颜色来生成渐变图。

但如果我想画一条线呢？最 naive 的做法：

```cpp
// 画一条从 (10, 10) 到 (200, 150) 的线
// 需要遍历所有像素，判断每个像素是否在线段上...
for (int y = 0; y < height; y++) {
    for (int x = 0; x < width; x++) {
        if (pixelOnLine(x, y, 10, 10, 200, 150)) {
            image[y * width + x] = {255, 255, 255};
        }
    }
}
```

**问题很明显**：
1. 判断"像素是否在线段上"的代价很高（浮点距离计算）
2. 遍历全屏像素画一条线，99% 的判断都是浪费的
3. 线稍微移动一下，所有计算全部重来

**我们需要什么**：给定线段的两个端点，**自动算出它经过的所有像素**。

这就是**光栅化（Rasterization）**——把连续的几何描述离散化为像素的过程。

## 问题 1：怎么高效地画一条直线？

### 最 naive 的方案：基于直线方程采样

数学上，直线方程是 `y = kx + b`。我们可以让 x 从 x1 走到 x2，计算对应的 y：

```cpp
void drawLineNaive(Image& img, int x1, int y1, int x2, int y2, Color c) {
    float k = (float)(y2 - y1) / (x2 - x1);
    float b = y1 - k * x1;
    for (int x = x1; x <= x2; x++) {
        int y = (int)(k * x + b);
        img.setPixel(x, y, c);
    }
}
```

**立刻发现的问题**：
1. **浮点运算慢**：每个像素都要做乘法和加法
2. **精度累积误差**：浮点数迭代会产生偏差，长直线的末端可能偏离
3. **不适用于陡线**：当 `|k| > 1` 时，y 方向跳跃，线会断开

### 改进：Bresenham 直线算法（只用整数加法）

1962 年，Jack Bresenham 提出一个天才方案：**用决策参数代替浮点运算**。

核心观察：对于平缓直线（0 < 斜率 < 1），每次 x 增加 1，y 要么不变，要么增加 1。选哪个？看哪个离真实直线更近。

```cpp
void drawLineBresenham(Image& img, int x1, int y1, int x2, int y2, Color c) {
    int dx = x2 - x1;
    int dy = y2 - y1;
    int y = y1;
    int e = 0;  // 决策参数（误差积累）
    
    for (int x = x1; x <= x2; x++) {
        img.setPixel(x, y, c);
        e += 2 * dy;        // 累积误差
        if (e >= dx) {      // 偏离超过半个像素
            y++;            // 向上走一步
            e -= 2 * dx;    // 重置误差
        }
    }
}
```

**关键优化**：两边同乘 `2*dx`，消除了所有浮点运算。现在每个像素只需要**整数加法和比较**。

> 这个算法在 GPU 中仍然有用——虽然现代 GPU 的直线光栅化由硬件完成，但 Bresenham 的"增量计算"思想在边缘函数中得到了延续。

## 问题 2：怎么填充一个实心三角形？

画线可以画三角形的边框，但 3D 渲染需要的是**实心填充**的三角形表面。

### 最 naive 的方案：全屏遍历+内外判断

```cpp
for (int y = 0; y < height; y++) {
    for (int x = 0; x < width; x++) {
        if (pointInTriangle(x + 0.5f, y + 0.5f, tri)) {
            image[y * width + x] = color;
        }
    }
}
```

**立刻发现的问题**：遍历全屏效率太低。一个只占画面 10% 的三角形，90% 的判断都是浪费的。

### 改进 1：包围盒优化

只遍历三角形周围的矩形区域：

```cpp
int minX = floor(min(A.x, B.x, C.x));
int maxX = ceil(max(A.x, B.x, C.x));
int minY = floor(min(A.y, B.y, C.y));
int maxY = ceil(max(A.y, B.y, C.y));

for (int y = minY; y <= maxY; y++) {
    for (int x = minX; x <= maxX; x++) {
        if (pointInTriangle(x + 0.5f, y + 0.5f, tri)) {
            image[y * width + x] = color;
        }
    }
}
```

**效果**：对于只占画面 10% 的三角形，只需判断约 10% 的像素。

### 改进 2：叉乘法判断点是否在三角形内

怎么判断点 `P` 是否在三角形 `ABC` 内部？

**几何直觉**：如果 `P` 在三角形内部，那么它一定在三条边的"同一侧"。用叉乘判断：

```cpp
float cross(const Vec2& a, const Vec2& b) {
    return a.x * b.y - a.y * b.x;  // 2D 叉乘 = 有向面积 × 2
}

bool pointInTriangle(const Vec2& p, const Triangle& tri) {
    // 三条边
    Vec2 e1 = tri.p2 - tri.p1;
    Vec2 e2 = tri.p3 - tri.p2;
    Vec2 e3 = tri.p1 - tri.p3;
    // 从顶点指向 P 的向量
    Vec2 v1 = p - tri.p1;
    Vec2 v2 = p - tri.p2;
    Vec2 v3 = p - tri.p3;
    // 三次叉乘
    float c1 = cross(e1, v1);
    float c2 = cross(e2, v2);
    float c3 = cross(e3, v3);
    // 同号则在内部
    bool hasNeg = (c1 < 0) || (c2 < 0) || (c3 < 0);
    bool hasPos = (c1 > 0) || (c2 > 0) || (c3 > 0);
    return !(hasNeg && hasPos);
}
```

## 问题 3：现代 GPU 用什么方法光栅化？

叉乘法很直观，但有个问题：**只做了一件事**——判断点在不在三角形内。

现代 GPU 需要的不只是"在内/在外"，还需要知道**在内部的哪里**，才能做颜色插值、纹理采样、深度计算。

### 边缘函数法（Edge Function）

边缘函数是现代 GPU 光栅化器的核心。它的优势是：**一次计算，同时得到内外测试结果和插值权重**。

对于三角形的一条边（从 `P1` 到 `P2`），边缘函数定义为：

```
E(x, y) = (x - x1)(y2 - y1) - (y - y1)(x2 - x1)
        = cross(P - P1, P2 - P1)
```

**几何意义**：`E(x,y)` 的正负号表示点 `(x,y)` 在边的哪一侧。

### 关键优势：增量计算

从一个像素到相邻像素，边缘函数值只需要加一个常数：

```cpp
E(x+1, y) = E(x, y) + (y2 - y1)  // 沿 X 方向：加 Δy
E(x, y+1) = E(x, y) - (x2 - x1)  // 沿 Y 方向：减 Δx
```

| 计算方式 | 每次像素运算 | 100×100 三角形 |
|----------|-------------|---------------|
| 完整叉乘 | 2次减法 + 2次乘法 + 1次减法 = **5次运算** | 50,000 次 |
| 增量更新 | **1次加法** | 10,000 次 |

### 边缘函数 → 重心坐标

边缘函数最神奇的地方：**结果可以直接推导出重心坐标**。

对三角形的三个顶点 `P1, P2, P3`，定义三个边缘函数 `e0, e1, e2`。它们正好对应重心坐标的分子：

```cpp
float area2 = e0 + e1 + e2;  // 总面积（2倍有向面积）
float alpha = e1 / area2;     // 对应顶点 P1 的权重
float beta  = e2 / area2;     // 对应顶点 P2 的权重
float gamma = e0 / area2;     // 对应顶点 P3 的权重
```

**重心坐标的意义**：三角形内任意一点的颜色，可以用三个顶点的颜色加权平均：

```cpp
Color finalColor = alpha * colorA + beta * colorB + gamma * colorC;
```

> 这意味着 GPU 光栅化器在判断"像素是否在三角形内"的同时，就已经算出了**插值权重**。颜色插值、纹理坐标插值、深度插值，全部基于这三个权重。

## 完整光栅化代码

```cpp
void rasterizeTriangle(Image& img, Vertex v[3]) {
    // 1. 计算总面积
    float area2 = cross2D(v[1].pos - v[0].pos, v[2].pos - v[0].pos);
    if (fabs(area2) < 1e-6) return;
    
    // 2. 计算包围盒
    int minX = std::max(0, (int)floor(min({v[0].pos.x, v[1].pos.x, v[2].pos.x})));
    int maxX = std::min(img.width-1, (int)ceil(max({v[0].pos.x, v[1].pos.x, v[2].pos.x})));
    int minY = std::max(0, (int)floor(min({v[0].pos.y, v[1].pos.y, v[2].pos.y})));
    int maxY = std::min(img.height-1, (int)ceil(max({v[0].pos.y, v[1].pos.y, v[2].pos.y})));
    
    // 3. 预计算增量步长
    float edge01StepX = v[0].pos.y - v[1].pos.y;
    float edge12StepX = v[1].pos.y - v[2].pos.y;
    float edge20StepX = v[2].pos.y - v[0].pos.y;
    
    // 4. 光栅化循环
    for (int y = minY; y <= maxY; y++) {
        Vec2 p = {minX + 0.5f, y + 0.5f};
        float e01 = cross2D(v[1].pos - v[0].pos, p - v[0].pos);
        float e12 = cross2D(v[2].pos - v[1].pos, p - v[1].pos);
        float e20 = cross2D(v[0].pos - v[2].pos, p - v[2].pos);
        
        for (int x = minX; x <= maxX; x++) {
            // 用 area2 统一符号判断
            bool inside = (e01 * area2 >= 0) && (e12 * area2 >= 0) && (e20 * area2 >= 0);
            
            if (inside) {
                float alpha = e12 / area2;
                float beta  = e20 / area2;
                float gamma = e01 / area2;
                
                // 颜色插值
                Color c = {
                    (uint8_t)(alpha * v[0].color.r + beta * v[1].color.r + gamma * v[2].color.r),
                    (uint8_t)(alpha * v[0].color.g + beta * v[1].color.g + gamma * v[2].color.g),
                    (uint8_t)(alpha * v[0].color.b + beta * v[1].color.b + gamma * v[2].color.b),
                    255
                };
                img.setPixel(x, y, c);
            }
            
            e01 += edge01StepX;
            e12 += edge12StepX;
            e20 += edge20StepX;
        }
    }
}
```

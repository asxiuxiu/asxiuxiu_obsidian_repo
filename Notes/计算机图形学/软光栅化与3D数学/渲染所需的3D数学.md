# 线性代数速查

## 问题 0：为什么图形学需要线性代数？

想象你要写一个 3D 游戏。你需要做这些事：
- 角色从 A 点走到 B 点（**平移**）
- 角色转身面向敌人（**旋转**）
- 角色蹲下变小（**缩放**）
- 计算光线和表面的夹角（**点乘**）
- 判断角色面向的左右方向（**叉乘**）

**最 naive 的方案**：每个操作写一组独立的函数。

```cpp
// 平移：直接加坐标
void translate(float& x, float& y, float& z, float dx, float dy, float dz) {
    x += dx; y += dy; z += dz;
}

// 旋转：用三角函数硬算
void rotateX(float& y, float& z, float angle) {
    float newY = y * cos(angle) - z * sin(angle);
    float newZ = y * sin(angle) + z * cos(angle);
    y = newY; z = newZ;
}
```

**立刻发现的问题**：
1. 每个操作都是独立的函数，组合起来代码爆炸
2. 先旋转再平移 ≠ 先平移再旋转，顺序管理混乱
3. 无法统一处理"先缩放、再旋转、再平移"这种复合变换

**线性代数的价值**：用**矩阵乘法**统一表达所有几何变换，用**向量运算**统一表达方向、距离、角度。

## 问题 1：向量是什么？为什么用向量？

### 最 naive 的理解：向量就是一个有方向的箭头

在图形学中，向量有两个角色：
- **位置向量**：从原点指向某一点（如角色的位置）
- **方向向量**：只有方向，没有固定起点（如法线、速度）

```cpp
struct Vec3 {
    float x, y, z;
};
```

### 为什么需要向量运算？

| 运算 | 解决什么问题 | 图形学场景 |
|------|-------------|-----------|
| **加法** | 两个方向叠加 | 速度 + 加速度 = 新速度 |
| **减法** | 从一个点指向另一个点 | 角色位置 - 敌人位置 = 方向向量 |
| **数乘** | 改变长度，不改变方向 | 速度 × 2 = 加速 |
| **点乘** | 计算夹角、判断前后 | 光线和法线的夹角 → 光照强度 |
| **叉乘** | 求垂直方向、判断左右 | 两个边向量叉乘 → 法线向量 |

### 点乘（Dot Product）

```cpp
float Dot(const Vec3& a, const Vec3& b) {
    return a.x * b.x + a.y * b.y + a.z * b.z;
}
```

**几何意义**：`a · b = |a| × |b| × cos(θ)`

```
Dot(a, b) > 0   →  夹角 < 90°（方向大致相同）
Dot(a, b) = 0   →  夹角 = 90°（垂直）
Dot(a, b) < 0   →  夹角 > 90°（方向相反）
```

**图形学应用**：
- 判断一个点是否在相机前方：用视线方向点乘该点相对相机的方向
- 计算漫反射光照强度：`L · N`（光线方向和法线的点乘）

### 叉乘（Cross Product）

```cpp
Vec3 Cross(const Vec3& a, const Vec3& b) {
    return {
        a.y * b.z - a.z * b.y,
        a.z * b.x - a.x * b.z,
        a.x * b.y - a.y * b.x
    };
}
```

**几何意义**：结果向量垂直于 `a` 和 `b` 构成的平面，长度等于平行四边形的面积。

```
    b
    ↑
    │   ↗ cross(a, b) 垂直向外
    │  /
    │ /
    └──────→ a
```

**图形学应用**：
- 已知三角形的两条边，叉乘得到法线
- 判断点是否在三角形内（2D 叉乘的符号）

## 问题 2：为什么需要矩阵？

### 单个变换的表示

| 变换 | 对向量的操作 | 矩阵形式 |
|------|-------------|---------|
| **缩放** | `(x*sx, y*sy, z*sz)` | 对角矩阵 |
| **旋转** | 三角函数组合 | 正交矩阵 |
| **平移** | `(x+tx, y+ty, z+tz)` | **不能用 3×3 矩阵！** |

**关键问题**：平移是**加法**，不是乘法。3×3 矩阵只能表示线性变换（旋转/缩放/切变），无法表示平移。

### 解决方案：齐次坐标 + 4×4 矩阵

给 3D 点增加一个维度 `w`：
- **点**：`(x, y, z, 1)` — 受平移影响
- **向量**：`(x, y, z, 0)` — 不受平移影响（向量只表示方向）

现在平移可以写成 4×4 矩阵：

```
| 1  0  0  tx |   | x |   | x + tx |
| 0  1  0  ty | × | y | = | y + ty |
| 0  0  1  tz |   | z |   | z + tz |
| 0  0  0  1  |   | 1 |   |   1    |
```

**核心价值**：所有变换（旋转、缩放、平移）都统一为**矩阵乘法**，可以任意组合。

```
复合变换 = T(平移) × R(旋转) × S(缩放)
         （从右往左读：先缩放，再旋转，再平移）
```

## 问题 3：矩阵乘法为什么从右往左读？

假设要对向量 `v` 先缩放 `S`，再旋转 `R`，再平移 `T`：

```
v' = T × (R × (S × v))
   = (T × R × S) × v        （矩阵乘法结合律）
```

**从右往左读**：`S` 最先作用，然后 `R`，然后 `T`。

**一个常见的坑**：
```
先旋转再平移 ≠ 先平移再旋转
```
- 先旋转再平移：物体先转身，再整体移动（方向变了，但平移方向固定）
- 先平移再旋转：物体先移开原点，再绕原点旋转（相当于圆周运动）

## 核心公式速查表

### 向量运算

```cpp
struct Vec3 {
    float x, y, z;
    
    Vec3 operator+(const Vec3& o) const { return {x+o.x, y+o.y, z+o.z}; }
    Vec3 operator-(const Vec3& o) const { return {x-o.x, y-o.y, z-o.z}; }
    Vec3 operator*(float s)       const { return {x*s, y*s, z*s}; }
    
    float Dot(const Vec3& o) const { return x*o.x + y*o.y + z*o.z; }
    Vec3 Cross(const Vec3& o) const {
        return {y*o.z - z*o.y, z*o.x - x*o.z, x*o.y - y*o.x};
    }
    float Length() const { return std::sqrt(Dot(*this)); }
    Vec3 Normalized() const {
        float len = Length();
        return len > 0 ? (*this) * (1.0f/len) : Vec3{0,0,0};
    }
};
```

### 基础变换矩阵（4×4）

| 变换 | 矩阵 |
|------|------|
| **平移** `(tx, ty, tz)` | 单位矩阵 + 最后一列 `(tx, ty, tz, 1)` |
| **缩放** `(sx, sy, sz)` | 对角线 `(sx, sy, sz, 1)` |
| **绕 Z 旋转 θ** | `c=cosθ, s=sinθ`：第0,1行/列填入旋转子矩阵 |
| **绕 X 旋转 θ** | 同上，操作 YZ 平面 |
| **绕 Y 旋转 θ** | 同上，操作 XZ 平面（注意 `sin` 符号相反） |

# 高斯模糊

## 问题 0：直接做 N×N 卷积为什么太贵？

在 [[Notes/计算机图形学/帧缓冲与后处理/后处理管线|后处理管线]] 里，我们已经知道后处理就是对全屏纹理做图像处理。模糊（Blur）是最基础的后处理效果之一。

最直观的模糊做法是：对每个像素，读取周围 N×N 区域内的所有像素，加权平均。这就是 **卷积（Convolution）**。

假设用一个 9×9 的卷积核：

```glsl
vec3 blur = vec3(0.0);
for (int x = -4; x <= 4; x++) {
    for (int y = -4; y <= 4; y++) {
        blur += texture(uInput, uv + vec2(x, y) * texelSize).rgb;
    }
}
blur /= 81.0;  // 9×9 = 81 次采样
```

**每像素 81 次纹理采样**。如果是 1920×1080 的分辨率，一帧就要采样 1.68 亿次。更不用说工业级 Bloom 常常需要 15×15 甚至更大的核。

这就引出了核心问题：能不能用少得多的采样次数得到同样的模糊效果？

---

## 问题 1：高斯模糊和简单平均模糊有什么区别？

简单平均（Box Blur）是给卷积窗口内每个像素相同的权重：

```
Box 3×3 核：
┌─────┬─────┬─────┐
│ 1/9 │ 1/9 │ 1/9 │
├─────┼─────┼─────┤
│ 1/9 │ 1/9 │ 1/9 │
├─────┼─────┼─────┤
│ 1/9 │ 1/9 │ 1/9 │
└─────┴─────┴─────┘
```

**高斯模糊**则按照 **高斯分布（正态分布）** 给像素赋权：离中心越近的像素权重越大，离中心越远的权重越小。

```
Gaussian 3×3 核（近似）：
┌───────┬───────┬───────┐
│ 1/16  │ 2/16  │ 1/16  │
├───────┼───────┼───────┤
│ 2/16  │ 4/16  │ 2/16  │
├───────┼───────┼───────┤
│ 1/16  │ 2/16  │ 1/16  │
└───────┴───────┴───────┘
```

高斯模糊的好处：

- **更自然**：模拟真实镜头失焦、大气散射、视网膜模糊
- **无方块感**：Box Blur 会产生明显的“方块”边缘
- **数学性质好**：可分离、可叠加、无振铃

> 高斯核的权重来自二维高斯函数：
> ```
> G(x, y) = (1 / 2πσ²) * e^(-(x² + y²) / 2σ²)
> ```
> 其中 σ（sigma）控制模糊半径：σ 越大，核越宽，模糊越强。

---

## 问题 2：为什么高斯核可以拆成两个一维核？

这是高斯模糊最关键的数学性质：**可分离性（Separability）**。

二维高斯函数可以拆成两个一维高斯函数的乘积：

```
G(x, y) = G(x) * G(y)

其中：
G(x) = (1 / √(2π)σ) * e^(-x² / 2σ²)
G(y) = (1 / √(2π)σ) * e^(-y² / 2σ²)
```

这意味着：

> **对图像先做水平方向的一维高斯模糊，再做垂直方向的一维高斯模糊，结果等价于直接做一次二维高斯模糊。**

**计算量对比**：

| 方法 | 9×9 核 | 15×15 核 |
|------|--------|---------|
| 2D 卷积 | 81 次采样/像素 | 225 次采样/像素 |
| 可分离 1D×2 | 9 + 9 = 18 次 | 15 + 15 = 30 次 |
| 加速比 | **4.5×** | **7.5×** |

核越大，可分离的优势越明显。

## 问题 3：双 Pass 高斯模糊具体怎么实现？

我们需要：

1. 一个水平的 1D 高斯模糊 Shader
2. 一个垂直的 1D 高斯模糊 Shader
3. 两个 FBO 做乒乓缓冲

```
Scene Color
    │
    ▼
[Horizontal Blur Pass]  →  Ping FBO
    │
    ▼
[Vertical Blur Pass]    →  Pong FBO（或屏幕）
```

**水平模糊片段着色器**：

```glsl
#version 330 core
in vec2 vUV;
uniform sampler2D uInputTexture;
uniform vec2 uTexelSize;     // 1.0 / textureSize
uniform float uWeights[5];   // 预计算的高斯权重
uniform int uRadius;         // 核半径
out vec4 FragColor;

void main() {
    vec3 color = texture(uInputTexture, vUV).rgb * uWeights[0];

    for (int i = 1; i <= uRadius; i++) {
        vec2 offset = vec2(float(i) * uTexelSize.x, 0.0);
        color += texture(uInputTexture, vUV + offset).rgb * uWeights[i];
        color += texture(uInputTexture, vUV - offset).rgb * uWeights[i];
    }

    FragColor = vec4(color, 1.0);
}
```

**垂直模糊片段着色器**：只需要把 `offset` 的 x 和 y 互换：

```glsl
vec2 offset = vec2(0.0, float(i) * uTexelSize.y);
```

> 技巧：可以只写一个 Shader，通过 uniform `uDirection` 控制是水平还是垂直。

## 问题 4：高斯权重怎么计算？

一维高斯权重公式：

```
G(i) = (1 / √(2π)σ) * e^(-i² / 2σ²)
```

代码生成（C++）：

```cpp
std::vector<float> GenerateGaussianWeights(int radius, float sigma) {
    std::vector<float> weights(radius + 1);
    float sum = 0.0f;

    for (int i = 0; i <= radius; i++) {
        float x = float(i);
        weights[i] = std::exp(-(x * x) / (2.0f * sigma * sigma));
        sum += (i == 0) ? weights[i] : 2.0f * weights[i];
    }

    // 归一化，使所有权重的和为 1
    for (float& w : weights) {
        w /= sum;
    }

    return weights;
}
```

**参数选择**：

- `radius` 越大，模糊范围越大
- `sigma` 通常取 `radius / 3.0` 左右，这样核边缘的权重已经很小，截断不会引入明显误差
- 一个经验规则：`sigma = radius / 3.0`，核大小 `= 2 * radius + 1`

## 问题 5：还能不能更快？——线性采样技巧

双 Pass 已经把 81 次采样降到 18 次。GPU 的纹理采样器支持 **双线性过滤（Bilinear Filtering）**，可以进一步把 18 次降到约 10 次。

**原理**：

普通采样时，我们在像素中心采样，每次只拿到一个像素值：

```
采样位置：   ●     ●     ●     ●     ●
像素：     [0]   [1]   [2]   [3]   [4]
```

如果我们在两个像素之间采样，GPU 会自动用双线性插值返回加权平均：

```
采样位置：     ×         ×         ×
覆盖像素：   [0]─[1]   [2]─[3]   [4]
            权重按距离分配
```

这样一次采样就能拿到两个像素的加权结果，相当于“一次纹理读取换两次点采样”。

**线性采样的权重和偏移计算**：

假设我们要对两个相邻像素 `a` 和 `b` 加权，原始权重分别是 `w_a` 和 `w_b`。我们可以：

1. 计算新的合并权重：`w_ab = w_a + w_b`
2. 计算采样位置：`offset = (w_b / w_ab) * pixelDistance`

这样一次双线性采样得到的结果就是 `(w_a * a + w_b * b) / (w_a + w_b)`，再乘以 `w_ab` 就是正确的加权和。

**效果**：对于 9-tap 的高斯核，线性采样可以把采样次数从 9 降到 5，双 Pass 后从 18 降到 10。

## 问题 6：为什么模糊常常配合降采样？

如果模糊半径很大（比如 50 像素），即使做了可分离优化，每帧还是要采样很多次。更常见的工业做法是：

```
全分辨率 Scene Color
    │
    ▼ 降采样到 1/2 或 1/4 分辨率
Blur Input (低分辨率)
    │
    ▼ 高斯模糊（采样次数不变，但像素总数少 4~16 倍）
Blurred Low-Res
    │
    ▼ 上采样回全分辨率
Final Blur
```

**为什么可以降采样？**

因为模糊本身就会丢失高频细节，低分辨率处理后再上采样，人眼几乎看不出差别。

**性能收益**：降到 1/4 分辨率，像素数减少 16 倍，即使 Shader 复杂度不变，总计算量也减少 16 倍。

> 这是 Bloom、SSAO、软阴影等效果的标准做法。

## 完整可运行示例：双 Pass 高斯模糊

```cpp
// flags: -O0 -g -lGL -lGLEW -lglfw
#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <vector>
#include <cmath>
#include <iostream>

// 假设已有 FBO / PingPongRT 封装，见《后处理管线》

// 高斯权重生成
std::vector<float> GenerateGaussianWeights(int radius, float sigma) {
    std::vector<float> weights(radius + 1);
    float sum = 0.0f;
    for (int i = 0; i <= radius; i++) {
        float x = float(i);
        weights[i] = std::exp(-(x * x) / (2.0f * sigma * sigma));
        sum += (i == 0) ? weights[i] : 2.0f * weights[i];
    }
    for (float& w : weights) w /= sum;
    return weights;
}

// 可分离高斯模糊 Shader（direction 控制水平/垂直）
const char* gaussianFS = R"(
#version 330 core
in vec2 vUV;
uniform sampler2D uInputTexture;
uniform vec2 uTexelSize;
uniform vec2 uDirection;     // (1,0) 水平 或 (0,1) 垂直
uniform float uWeights[9];   // 最大半径 8
uniform int uRadius;
out vec4 FragColor;

void main() {
    vec3 color = texture(uInputTexture, vUV).rgb * uWeights[0];

    for (int i = 1; i < uRadius; i++) {
        vec2 offset = uDirection * float(i) * uTexelSize;
        color += texture(uInputTexture, vUV + offset).rgb * uWeights[i];
        color += texture(uInputTexture, vUV - offset).rgb * uWeights[i];
    }

    FragColor = vec4(color, 1.0);
}
)";

void ApplyGaussianBlur(GLuint sceneTexture, PingPongRT& pp,
                       GLuint blurProgram, int width, int height,
                       int radius, float sigma)
{
    auto weights = GenerateGaussianWeights(radius, sigma);

    // Pass 1：水平模糊
    glBindFramebuffer(GL_FRAMEBUFFER, pp.WriteFBO());
    glViewport(0, 0, width, height);
    glClear(GL_COLOR_BUFFER_BIT);

    glUseProgram(blurProgram);
    glActiveTexture(GL_TEXTURE0);
    glBindTexture(GL_TEXTURE_2D, sceneTexture);
    glUniform1i(glGetUniformLocation(blurProgram, "uInputTexture"), 0);
    glUniform2f(glGetUniformLocation(blurProgram, "uTexelSize"),
                1.0f / width, 1.0f / height);
    glUniform2f(glGetUniformLocation(blurProgram, "uDirection"), 1.0f, 0.0f);
    glUniform1i(glGetUniformLocation(blurProgram, "uRadius"), radius);
    glUniform1fv(glGetUniformLocation(blurProgram, "uWeights"),
                 weights.size(), weights.data());

    DrawFullscreenQuad();
    pp.Swap();

    // Pass 2：垂直模糊
    glBindFramebuffer(GL_FRAMEBUFFER, pp.WriteFBO());
    glViewport(0, 0, width, height);
    glClear(GL_COLOR_BUFFER_BIT);

    glActiveTexture(GL_TEXTURE0);
    glBindTexture(GL_TEXTURE_2D, pp.ReadTexture());
    glUniform2f(glGetUniformLocation(blurProgram, "uDirection"), 0.0f, 1.0f);

    DrawFullscreenQuad();
    pp.Swap();

    // 最终结果在 pp.ReadTexture()
}
```

## 性能对比总结

| 实现方式 | 9×9 核采样次数 | 15×15 核采样次数 | 备注 |
|----------|---------------|-----------------|------|
| 2D 卷积 | 81 | 225 | 直观但太慢 |
| 可分离双 Pass | 18 | 30 | 推荐的学习阶段实现 |
| + 线性采样 | ~10 | ~16 | 进一步提速 |
| + 降采样到 1/4 | 等效更低 | 等效更低 | 工业级大半径模糊标配 |

## 常见问题诊断

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 模糊后画面变暗 | 权重没归一化 | 确保权重和为 1 |
| 模糊有条纹/方向性 | 水平和垂直 Pass 没有严格分离 | 检查 uDirection 和偏移方向 |
| 边缘出现黑边 | FBO 颜色附件用了 REPEAT | 改为 `GL_CLAMP_TO_EDGE` |
| 模糊不均匀 | sigma 和 radius 不匹配 | sigma ≈ radius / 3 |
| 大半径模糊仍然卡 | 没降采样 | 先在低分辨率做模糊，再上采样 |
| 多次模糊后画面过曝/变糊 | 乒乓缓冲读写冲突 | 确保每次 Pass 读写的是不同纹理 |

## 与 SelfGameEngine 的关系

这篇笔记对应引擎 **阶段 5.19 后处理效果算法** 中 Bloom、SSAO、软阴影等效果的基础模块。

在引擎中，高斯模糊会封装成一个独立的后处理效果：

```cpp
struct GaussianBlurEffect {
    ShaderHandle shader;
    int radius = 5;
    float sigma = 2.0f;
    bool useDownsample = true;
    float downsampleScale = 0.5f;
};

void GaussianBlurEffect::Render(
    RenderDevice& device,
    TextureHandle input,
    RenderTarget& output)
{
    // 1. 可选：降采样
    // 2. 水平 Pass
    // 3. 垂直 Pass
    // 4. 可选：上采样
}
```

你现在写的：
- `GenerateGaussianWeights` → 引擎后处理效果的参数预处理
- 可分离双 Pass → 引擎 Bloom 的下采样链/上采样链核心
- 乒乓缓冲 → 引擎 `PostProcessContext`
- 降采样优化 → 引擎 Bloom/HBAO/软阴影的性能基础

理解了高斯模糊，你就理解了为什么 Bloom 能用“多级降采样 + 模糊 + 上采样”做出大范围光晕，而不是用巨大的卷积核硬算。

## API 对照：我们在解决“高效模糊”这个问题

| 概念 | OpenGL | Vulkan | D3D12 |
|------|--------|--------|-------|
| 可分离卷积 | 两个 Fullscreen Pass | 两个 Subpass / RenderPass | 两个 Draw 事件 |
| 线性采样优化 | `GL_LINEAR` 纹理过滤 | `VkSampler` bilinear | Sampler 状态 bilinear |
| 降采样 | 渲染到更小 RenderTarget | 更小 `VkFramebuffer` | 更小 RTV |
| 现代进一步优化 | Compute Shader | `vkCmdDispatch` + UAV | `Dispatch` + UAV |

**个人项目推荐**：
- 学习阶段：可分离双 Pass +  uniforms 传权重
- 引擎阶段：封装 `GaussianBlurEffect`，支持半径/sigma/降采样配置
- 现代 API 迁移：Compute Shader 直接写 UAV，避免全屏几何开销

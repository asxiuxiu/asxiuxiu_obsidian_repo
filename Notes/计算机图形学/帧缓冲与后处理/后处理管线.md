# 后处理管线

## 问题 0：一个 Pass 只能输出一个画面，但复杂效果需要多步

在 [[Notes/计算机图形学/帧缓冲与后处理/离屏渲染与FBO|离屏渲染与FBO]] 里，我们已经能把场景先画到 FBO 的颜色纹理上，再画一个全屏四边形把它显示到屏幕上。这相当于“一个后处理 Pass”。

但很多效果没法一步完成：

- **高斯模糊**：直接做 N×N 卷积太贵，通常拆成“水平模糊 + 垂直模糊”两个 Pass
- **泛光 Bloom**：先要提取高光，再降采样、模糊、升采样、混合，至少 5~10 个 Pass
- **运动模糊**：需要速度缓冲 + 多个方向采样
- **色调映射 + FXAA +  vignette**：常常希望分阶段叠加

**核心矛盾**：每个 Pass 只能输出到一个目标，但很多效果需要“上一步的输出作为下一步的输入”。

这就引出了本篇的问题：怎么组织多个后处理 Pass？

---

## 问题 1：最 naive 的方案——把所有效果塞进一个巨型 Shader

既然每个 Pass 只能输出一个结果，那把灰度、模糊、泛光、色调映射全写进一个片段着色器不就行了？

```glsl
#version 330 core
in vec2 vUV;
uniform sampler2D uSceneTexture;
out vec4 FragColor;

void main() {
    vec3 color = texture(uSceneTexture, vUV).rgb;

    // 1. 灰度
    float gray = dot(color, vec3(0.299, 0.587, 0.114));
    color = vec3(gray);

    // 2. 简单模糊（N×N 采样）
    vec3 blur = vec3(0.0);
    for (int x = -4; x <= 4; x++)
        for (int y = -4; y <= 4; y++)
            blur += texture(uSceneTexture, vUV + vec2(x,y) * texelSize).rgb;
    color = blur / 81.0;

    // 3. 色调映射
    color = color / (color + vec3(1.0));

    // 4. FXAA ...

    FragColor = vec4(color, 1.0);
}
```

**立刻发现的问题**：

- **组合爆炸**：N 个效果，每个开/关，就需要 2^N 个 Shader 变体
- **难以调试**：效果全耦合，出了问题很难定位是哪一步
- **性能浪费**：即使只开一个效果，也要编译执行所有代码
- **无法复用中间结果**：Bloom 需要多次降采样，单 Pass 里只能采样原始纹理

**结论**：后处理必须以独立的、可组合的小 Shader 形式存在，通过多 Pass 串联。

---

## 问题 2：后处理 Pass 的本质是什么？

每个后处理 Pass 都只做一件简单的事：

```
输入：一张纹理（上一步的结果）
处理：一个全屏片段着色器，对每个像素做某种图像操作
输出：一张新的纹理（通过 FBO）
```

**关键：后处理不是“画 3D 模型”，而是“对整个画面做图像处理”。**

因为没有 3D 几何，只需要一个覆盖整个屏幕的四边形（或三角形），让片段着色器对每一个像素执行一次。这个四边形叫 **Full-Screen Quad（全屏四边形）**，这个 Pass 叫 **Fullscreen Pass**。

---

## 问题 3：全屏四边形怎么画？

最简单的全屏四边形顶点：

```cpp
float quadVertices[] = {
    // 位置 (NDC)   // UV
    -1.0f, -1.0f,   0.0f, 0.0f,
     1.0f, -1.0f,   1.0f, 0.0f,
     1.0f,  1.0f,   1.0f, 1.0f,
    -1.0f,  1.0f,   0.0f, 1.0f
};
unsigned int quadIndices[] = { 0, 1, 2, 0, 2, 3 };
```

顶点着色器把位置和 UV 直接传过去：

```glsl
#version 330 core
layout (location = 0) in vec2 aPos;
layout (location = 1) in vec2 aUV;
out vec2 vUV;

void main() {
    gl_Position = vec4(aPos, 0.0, 1.0);
    vUV = aUV;
}
```

片段着色器只对 UV 采样：

```glsl
#version 330 core
in vec2 vUV;
uniform sampler2D uInputTexture;
out vec4 FragColor;

void main() {
    FragColor = texture(uInputTexture, vUV);
}
```

> **关于全屏三角形**：一些引擎用一个大三角形而不是四边形（节省 3 个顶点调用，避免四边形对角线处的 2×2 quad 重复计算）。学习阶段用四边形最直观，理解了原理后迁移到三角形只需改顶点数据。

## 问题 4：两个 Pass 怎么串联？

假设我们要做“灰度 → 反色”两个效果：

```
场景纹理 → [灰度 Pass] → 中间纹理 A → [反色 Pass] → 屏幕
```

C++ 代码：

```cpp
// Pass 1：灰度 Pass，输出到 FBO A
glBindFramebuffer(GL_FRAMEBUFFER, fboA);
glViewport(0, 0, w, h);
glClear(GL_COLOR_BUFFER_BIT);

glUseProgram(grayscaleShader);
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, sceneTexture);  // 输入：场景
DrawFullscreenQuad();

// Pass 2：反色 Pass，输出到默认帧缓冲（屏幕）
glBindFramebuffer(GL_FRAMEBUFFER, 0);
glViewport(0, 0, windowW, windowH);
glClear(GL_COLOR_BUFFER_BIT);

glUseProgram(invertShader);
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, fboAColorTexture);  // 输入：Pass 1 的输出
DrawFullscreenQuad();
```

**状态变化图**：

```
Pass 1:
  绑定目标 = FBO A
  输入纹理 = sceneTexture
  Shader   = grayscaleShader
  输出     = fboAColorTexture

Pass 2:
  绑定目标 = 默认帧缓冲
  输入纹理 = fboAColorTexture
  Shader   = invertShader
  输出     = 屏幕
```

## 问题 5：如果需要更多 Pass，纹理不够用怎么办？

假设有 4 个后处理效果：灰度 → 模糊 → 边缘检测 → 色调映射。

如果每个 Pass 都创建一个新的 FBO 和纹理，4 个 Pass 就要 4 套 FBO，显存和管理都麻烦。

**更聪明的做法：乒乓缓冲（Ping-Pong Buffering）**。

只需要两个 FBO（Ping 和 Pong），每个 Pass 交替读写：

```
Pass 1：sceneTexture → [灰度]    → Ping
Pass 2：Ping         → [模糊]    → Pong
Pass 3：Pong         → [边缘检测] → Ping
Pass 4：Ping         → [色调映射] → Pong（或屏幕）
```

**为什么两个就够了？**

因为每个 Pass 只需要读“上一步的输出”和写“当前步的输出”。写完后，旧的输入就没用了，可以变成下一步的输出目标。

```cpp
struct PingPongRT {
    GLuint fbo[2];
    GLuint colorTexture[2];
    int readIndex = 0;

    void Swap() { readIndex = 1 - readIndex; }
    GLuint ReadTexture() const { return colorTexture[readIndex]; }
    GLuint WriteFBO() const { return fbo[1 - readIndex]; }
};

// 使用
PingPongRT pp;
pp.Init(width, height);

for (size_t i = 0; i < effects.size(); i++) {
    bool isLast = (i == effects.size() - 1);

    if (isLast) {
        glBindFramebuffer(GL_FRAMEBUFFER, 0);  // 最后一步画到屏幕
        glViewport(0, 0, windowW, windowH);
    } else {
        glBindFramebuffer(GL_FRAMEBUFFER, pp.WriteFBO());
        glViewport(0, 0, width, height);
    }

    glUseProgram(effects[i].shader);
    glActiveTexture(GL_TEXTURE0);
    glBindTexture(GL_TEXTURE_2D, pp.ReadTexture());
    glUniform1i(glGetUniformLocation(effects[i].shader, "uInputTexture"), 0);

    DrawFullscreenQuad();

    if (!isLast) pp.Swap();
}
```

> **关键规则**：不能同时把一张纹理既当作 FBO 的颜色附件（写目标），又当作当前 Shader 的输入纹理（读目标）。乒乓缓冲通过错开读写来避免这个冲突。

## 问题 6：后处理栈怎么抽象？

工业引擎会把后处理组织成一个 **PostProcess Stack（后处理栈）**：

```cpp
struct PostProcessEffect {
    ShaderHandle shader;
    // 效果参数，如模糊半径、泛光阈值等
};

struct PostProcessStack {
    std::vector<PostProcessEffect> effects;
    PingPongRT pingPong;

    void Render(const RenderTarget& sceneColorRT, int windowW, int windowH) {
        // 第一个效果的输入是场景颜色
        GLuint currentInput = sceneColorRT.colorTexture;

        for (size_t i = 0; i < effects.size(); i++) {
            bool isLast = (i == effects.size() - 1);

            // 绑定输出目标
            if (isLast) {
                glBindFramebuffer(GL_FRAMEBUFFER, 0);
                glViewport(0, 0, windowW, windowH);
            } else {
                glBindFramebuffer(GL_FRAMEBUFFER, pingPong.WriteFBO());
                glViewport(0, 0, pingPong.width, pingPong.height);
            }

            // 绑定输入纹理并绘制
            glUseProgram(effects[i].shader);
            glActiveTexture(GL_TEXTURE0);
            glBindTexture(GL_TEXTURE_2D, currentInput);
            DrawFullscreenQuad();

            // 更新下一步的输入
            if (!isLast) {
                currentInput = pingPong.ReadTexture();
                pingPong.Swap();
            }
        }
    }
};
```

**这个抽象的好处**：

- 新增效果只需要往栈里加一个 `PostProcessEffect`
- 效果的开关、顺序、参数都可以运行时配置
- 乒乓缓冲自动管理中间纹理，上层无感知

## 问题 7：后处理 Pass 需要深度测试吗？

通常不需要。

后处理是对“已经渲染好的 2D 画面”做图像处理，不存在 3D 几何遮挡关系。因此：

```cpp
// 后处理 Pass 通常关闭深度测试
glDisable(GL_DEPTH_TEST);
glDepthMask(GL_FALSE);

// 如果不需要模板，也关掉
glDisable(GL_STENCIL_TEST);
```

**为什么？** 开启深度测试不仅没有意义，还可能因为深度缓冲残留导致全屏四边形被意外剔除。

---

## 问题 8：混合模式要注意什么？

后处理 Pass 通常是“全屏四边形覆盖整个画面”，如果不关闭混合，后处理结果会和默认帧缓冲里残留的内容混合，产生奇怪的画面：

```cpp
// 如果之前开启了 Alpha 混合，后处理前要关闭或设为 Opaque
glDisable(GL_BLEND);
```

如果某个后处理效果确实需要混合（比如把 Bloom 叠加回 Scene Color），则显式设置：

```cpp
glEnable(GL_BLEND);
glBlendFunc(GL_ONE, GL_ONE);  // 加法混合
```

> **原则**：后处理栈的每一个 Pass 默认都是 Opaque 覆盖，不要继承上一步的混合状态。

## 完整可运行示例：灰度 + 反色后处理链

```cpp
// flags: -O0 -g -lGL -lGLEW -lglfw
#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <vector>
#include <iostream>

struct FBO {
    GLuint fbo = 0, colorTex = 0, depthRBO = 0;
    int w = 0, h = 0;

    bool Init(int width, int height) {
        w = width; h = height;
        glGenFramebuffers(1, &fbo);
        glBindFramebuffer(GL_FRAMEBUFFER, fbo);

        glGenTextures(1, &colorTex);
        glBindTexture(GL_TEXTURE_2D, colorTex);
        glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, w, h, 0,
                     GL_RGBA, GL_UNSIGNED_BYTE, nullptr);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
        glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                               GL_TEXTURE_2D, colorTex, 0);

        glGenRenderbuffers(1, &depthRBO);
        glBindRenderbuffer(GL_RENDERBUFFER, depthRBO);
        glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8, w, h);
        glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_STENCIL_ATTACHMENT,
                                  GL_RENDERBUFFER, depthRBO);

        if (glCheckFramebufferStatus(GL_FRAMEBUFFER) != GL_FRAMEBUFFER_COMPLETE)
            return false;
        glBindFramebuffer(GL_FRAMEBUFFER, 0);
        return true;
    }

    void Bind() const {
        glBindFramebuffer(GL_FRAMEBUFFER, fbo);
        glViewport(0, 0, w, h);
    }

    void Destroy() {
        if (colorTex) glDeleteTextures(1, &colorTex);
        if (depthRBO) glDeleteRenderbuffers(1, &depthRBO);
        if (fbo) glDeleteFramebuffers(1, &fbo);
    }
};

struct PingPongRT {
    FBO rt[2];
    int readIdx = 0;

    bool Init(int w, int h) {
        return rt[0].Init(w, h) && rt[1].Init(w, h);
    }

    GLuint ReadTexture() const { return rt[readIdx].colorTex; }
    GLuint WriteFBO() const { return rt[1 - readIdx].fbo; }

    void Swap() { readIdx = 1 - readIdx; }

    void Destroy() {
        rt[0].Destroy();
        rt[1].Destroy();
    }
};

// 全屏四边形
float quadVertices[] = {
    -1, -1,  0, 0,
     1, -1,  1, 0,
     1,  1,  1, 1,
    -1,  1,  0, 1
};
unsigned int quadIndices[] = { 0, 1, 2, 0, 2, 3 };

const char* vs = R"(
#version 330 core
layout (location = 0) in vec2 aPos;
layout (location = 1) in vec2 aUV;
out vec2 vUV;
void main() {
    gl_Position = vec4(aPos, 0.0, 1.0);
    vUV = aUV;
}
)";

const char* grayscaleFS = R"(
#version 330 core
in vec2 vUV;
uniform sampler2D uInputTexture;
out vec4 FragColor;
void main() {
    vec3 c = texture(uInputTexture, vUV).rgb;
    float g = dot(c, vec3(0.299, 0.587, 0.114));
    FragColor = vec4(g, g, g, 1.0);
}
)";

const char* invertFS = R"(
#version 330 core
in vec2 vUV;
uniform sampler2D uInputTexture;
out vec4 FragColor;
void main() {
    FragColor = vec4(1.0 - texture(uInputTexture, vUV).rgb, 1.0);
}
)";

// 简化：假设 CompileShader 和 CreateProgram 已实现
GLuint CreateProgram(const char* vs, const char* fs);

int main() {
    // ... 初始化 GLFW、GLEW、窗口 ...

    // 1. 创建场景 FBO 和乒乓 FBO
    FBO sceneFBO;
    sceneFBO.Init(800, 600);

    PingPongRT pp;
    pp.Init(800, 600);

    GLuint grayscaleProgram = CreateProgram(vs, grayscaleFS);
    GLuint invertProgram    = CreateProgram(vs, invertFS);

    std::vector<GLuint> effectShaders = { grayscaleProgram, invertProgram };

    while (!glfwWindowShouldClose(window)) {
        // ===== Pass 0：渲染场景到 sceneFBO =====
        sceneFBO.Bind();
        glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
        DrawScene();  // 你的 3D 场景

        // ===== Pass 1~N：后处理栈 =====
        glDisable(GL_DEPTH_TEST);
        glDisable(GL_BLEND);

        GLuint currentInput = sceneFBO.colorTex;

        for (size_t i = 0; i < effectShaders.size(); i++) {
            bool isLast = (i == effectShaders.size() - 1);

            if (isLast) {
                glBindFramebuffer(GL_FRAMEBUFFER, 0);
                glViewport(0, 0, 800, 600);
            } else {
                glBindFramebuffer(GL_FRAMEBUFFER, pp.WriteFBO());
                glViewport(0, 0, 800, 600);
            }

            glClear(GL_COLOR_BUFFER_BIT);
            glUseProgram(effectShaders[i]);
            glActiveTexture(GL_TEXTURE0);
            glBindTexture(GL_TEXTURE_2D, currentInput);
            glUniform1i(glGetUniformLocation(effectShaders[i], "uInputTexture"), 0);

            DrawFullscreenQuad();

            if (!isLast) {
                currentInput = pp.ReadTexture();
                pp.Swap();
            }
        }

        glfwSwapBuffers(window);
        glfwPollEvents();
    }

    sceneFBO.Destroy();
    pp.Destroy();
    return 0;
}
```

## 常见问题诊断

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 后处理没有效果 | 输入纹理没绑定或 uniform 没设置 | 检查 `glBindTexture` 和 `glUniform1i` |
| 画面全黑 | 同时把某纹理作为输入和 FBO 输出 | 使用乒乓缓冲，读写分离 |
| 画面撕裂/闪烁 | FBO 没正确解绑或乒乓交换逻辑错误 | 每帧结束时确认 read/write 索引状态 |
| 后处理结果有深度残留 | 后处理 Pass 没关深度测试 | `glDisable(GL_DEPTH_TEST)` |
| 混合异常 | 继承了上一步的混合状态 | 后处理前 `glDisable(GL_BLEND)`，需要时再开 |
| 纹理边缘有黑边 | 采样模式不对 | FBO 颜色附件用 `GL_CLAMP_TO_EDGE` |
| 移动端性能差 | 频繁切换 FBO / 乒乓次数太多 | 减少 Pass 数，合并效果，降低中间纹理分辨率 |

## 与 SelfGameEngine 的关系

这篇笔记直接对应引擎 **阶段 5.18 后处理栈架构** 的最小实现。

在引擎中，你会把“后处理效果”抽象成可配置的资源：

```cpp
struct PostProcessEffectAsset {
    ShaderHandle shader;
    // 参数... 如 threshold、intensity、radius
};

struct PostProcessStackAsset {
    std::vector<PostProcessEffectAsset> effects;
};
```

渲染器每帧执行：

```cpp
void RenderSystem::RenderPostProcess(
    const RenderTarget& sceneColor,
    const PostProcessStackAsset& stack)
{
    // 1. 初始化/复用乒乓 RenderTarget
    // 2. 遍历 stack.effects，依次执行 Fullscreen Pass
    // 3. 最后一个 Pass 输出到默认帧缓冲
}
```

你现在写的：
- `PingPongRT` → 引擎 `PostProcessContext` 或 `PingPongRenderTarget`
- `std::vector<GLuint> effectShaders` → 引擎 `PostProcessStack`
- `DrawFullscreenQuad` → 引擎 `FullscreenPassRenderer`
- “每个 Pass 一个 Shader，输入输出都是纹理” → 引擎后处理栈的设计核心

## API 对照：我们在解决“多 Pass 后处理链”这个问题

| 概念 | OpenGL | Vulkan | D3D12 |
|------|--------|--------|-------|
| 乒乓缓冲 | 两个 FBO 交替绑定 | 两个 `VkFramebuffer` 交替作为 render target | 两个 `ID3D12Resource` 交替作为 RTV |
| 全屏 Pass | 四边形 / 三角形 | 四边形 / 三角形 | 四边形 / 三角形 |
| 切换输出目标 | `glBindFramebuffer` | `vkCmdBeginRenderPass` | `OMSetRenderTargets` |
| 输入纹理绑定 | `glActiveTexture + glBindTexture` | `vkCmdBindDescriptorSets` | `SetGraphicsRootDescriptorTable` |
| 现代优化 | — | Compute Shader 直接写 UAV | Compute Shader 直接写 UAV |

**个人项目推荐**：
- 学习阶段：OpenGL 双 FBO 乒乓 + 全屏四边形
- 引擎阶段：`PostProcessStack` 抽象，支持效果注册、动态开关、参数混合
- 现代 API 迁移：后期可用 Compute Shader 替代全屏四边形，进一步减少几何开销

# 反走样与深度缓存

> [!info] 前置知识
> 请先阅读 [[计算机图形学/5.  光栅化Rasterization(Triangles)|光栅化基础]] 了解采样和走样现象。

---

## Why：两个核心问题

光栅化过程中需要解决两个关键问题：

1. **反走样 (Antialiasing)** — 消除采样产生的锯齿
2. **可见性 (Visibility)** — 正确处理物体间的遮挡关系

---

## Part 1: 反走样深入

### 1.1 采样瑕疵的类型

#### 空间维度

| 瑕疵               | 描述                 |
| ---------------- | ------------------ |
| **锯齿 (Jaggies)** | 边缘的阶梯状效果           |
| **摩尔纹 (Moiré)**  | 高频模式与采样频率相近产生的干涉纹样 |

#### 时间维度

| 瑕疵       | 描述              |
| -------- | --------------- |
| **车轮效应** | 快速旋转的轮子看起来在反向转动 |

> **核心原理**：采样瑕疵的根本原因是 **信号变化太快（高频），但采样速度太慢**。
> 
> *"Signals are changing too fast (high frequency), but sampled too slowly."*

![[Assets/graphics/01-sampling-artifacts.png]]

### 1.2 采样前滤波 (Pre-Filtering)

**正确顺序**：
```
原始信号 → [滤波/模糊] → 低通信号 → [采样] → 无走样结果
```

**错误顺序**：
```
原始信号 → [采样] → 走样信号 → [滤波] → 模糊的结果（仍有走样）
```

> 先采样后模糊无法消除已经产生的混叠！

![[Assets/graphics/02-blur-pre-filter-1.png]]
![[Assets/graphics/03-blur-pre-filter-2.png]]

### 1.3 从频域理解走样

#### 频率与周期

- **频率 (f)**：表示信号变化的快慢
- **周期 (T)**：信号重复一次的长度

![[Assets/graphics/04-frequency-period.png]]

#### 傅里叶变换

**核心思想**：任何信号都可以分解为不同频率的正弦/余弦波的叠加。

**傅里叶变换**：时域信号 ↔ 频域表示

![[Assets/graphics/07-fourier-transform.png]]

#### 走样的本质

**关键发现**：用相同的采样频率去采样两个**频率截然不同**的函数，结果可能**完全相同**。

- **原因**：高频信号采样不足，采样点错误地表现为低频信号
- **结果**：无法根据采样结果区分原始信号的频率
- **定义**：这种现象称为**走样 (Aliasing)**

![[Assets/graphics/08-aliasing-diagram.png]]

### 1.4 滤波的本质

**滤波** = 去除特定频率的内容

**卷积定理**：
- 时域卷积 = 频域乘积
- 时域乘积 = 频域卷积

> **Convolution in spatial domain = Multiplication in frequency domain**

![[Assets/graphics/15-convolution-theorem.png]]

#### 滤波类型

| 类型       | 作用     | 效果     |
| -------- | ------ | ------ |
| **高通滤波** | 保留高频   | 提取边缘   |
| **低通滤波** | 保留低频   | 模糊平滑   |
| **带通滤波** | 保留中间频段 | 特定特征提取 |

![[Assets/graphics/11-high-pass-filter.png]]
*高通滤波：提取边缘*

![[Assets/graphics/12-low-pass-filter.png]]
*低通滤波：模糊平滑*

### 1.5 从频域理解采样

**采样在频域中的含义**：
- 采样 = 重复原始信号的频谱

**走样的频域解释**：
- 走样 = 重复的频谱之间出现**重叠**

![[Assets/graphics/18-sampling-frequency-domain.png]]
*采样导致频谱重复*

![[Assets/graphics/19-aliasing-overlap.png]]
*走样 = 频谱重叠*

**奈奎斯特采样定理**：
- 为了避免频谱重叠，采样频率必须至少是信号最高频率的 **2 倍**
- 不满足就会发生混叠（走样）

### 1.6 反走样方法总结

| 方法 | 原理 | 特点 |
|-----|------|------|
| **增加分辨率** | 提高采样率 | 硬件限制，成本高 |
| **模糊预处理** | 先低通滤波再采样 | 主流方案 |
| **MSAA** | 子像素采样计算覆盖率 | 质量与性能平衡 |
| **FXAA** | 图像后处理识别边缘 | 极快，可能模糊 |
| **TAA** | 时间维度复用采样 | 高效，需处理运动 |

![[Assets/graphics/20-antialiasing-effect.png]]
*反走样效果对比*

### 1.7 超分辨率 (DLSS)

**目标**：将低分辨率图像（如 512×512）放大到高分辨率（如 1024×1024），同时避免锯齿。

**DLSS (Deep Learning Super Sampling)**：
- 利用深度学习重建高频细节
- 性能优于传统插值方法
- NVIDIA 显卡硬件加速

## Part 2: 遮挡与可见性

### 2.1 问题定义

当多个三角形叠加时，如何正确渲染可见的部分？

**解决方案**：深度缓存 (Z-Buffering)

### 2.2 画家算法 (Painter's Algorithm)

**基本思路**：
- 先绘制远处（深度大）的三角形
- 再绘制近处（深度小）的三角形

**问题**：
- 三角形深度交错时无法排序
- 需要 O(n log n) 的排序开销

![[Assets/graphics/24-painter-algorithm-issue.png]]

### 2.3 深度缓存 (Z-Buffer)

**核心思想**：
- 每个像素记录当前**最近的深度值**
- 绘制新像素时，比较深度，只保留更近的

**需要两个缓冲**：
- **帧缓冲 (Frame Buffer)**：存储颜色
- **深度缓冲 (Depth Buffer / Z-Buffer)**：存储深度值

![[Assets/graphics/25-z-value-definition.png]]

#### Z-Buffer 算法

```cpp
// 初始化
for (each pixel (x, y))
    zBuffer[x][y] = +∞;  // 最大深度

// 光栅化过程中
for (each Triangle T)
    for (each sample (x, y, z) in T)
        if (z < zBuffer[x][y]) {      // 更近？
            frameBuffer[x][y] = rgb;  // 更新颜色
            zBuffer[x][y] = z;        // 更新深度
        }
        // 否则被遮挡，不做任何事
```

![[Assets/graphics/26-z-buffer-diagram.png]]

#### Z-Buffer 的优点

| 特性 | 说明 |
|-----|------|
| **时间复杂度** | O(n)，n 为三角形数量（不需要排序） |
| **空间复杂度** | O(w × h)，w、h 为图像尺寸 |
| **顺序无关性** | 不管绘制顺序如何，结果相同 |

> Z-Buffer 是图形学最重要的算法之一，被所有 GPU 硬件实现。

## How：实践要点

### C++ 伪代码

```cpp
class Rasterizer {
    std::vector<Color> frameBuffer;
    std::vector<float> zBuffer;
    int width, height;
    
public:
    Rasterizer(int w, int h) : width(w), height(h) {
        frameBuffer.resize(w * h);
        zBuffer.resize(w * h, FLT_MAX);  // 初始最大深度
    }
    
    void clear() {
        std::fill(frameBuffer.begin(), frameBuffer.end(), Color::Black);
        std::fill(zBuffer.begin(), zBuffer.end(), FLT_MAX);
    }
    
    void drawTriangle(const Triangle& tri) {
        // 计算包围盒
        auto [xmin, xmax, ymin, ymax] = computeBoundingBox(tri);
        
        for (int y = ymin; y <= ymax; ++y) {
            for (int x = xmin; x <= xmax; ++x) {
                // 在像素中心采样
                float px = x + 0.5f;
                float py = y + 0.5f;
                
                // 计算重心坐标
                auto [alpha, beta, gamma] = barycentric(tri, px, py);
                
                // 判断是否在三角形内
                if (alpha < 0 || beta < 0 || gamma < 0) continue;
                
                // 插值深度值
                float z = alpha * tri.v0.z + beta * tri.v1.z + gamma * tri.v2.z;
                
                // Z-Buffer 测试
                int idx = y * width + x;
                if (z < zBuffer[idx]) {
                    // 更新深度和颜色
                    zBuffer[idx] = z;
                    frameBuffer[idx] = interpolateColor(tri, alpha, beta, gamma);
                }
            }
        }
    }
};
```

### 关键注意事项

| 问题 | 解决方案 |
|-----|---------|
| **Z-Fighting** | 两个表面深度相近导致的闪烁，避免将物体放得太近 |
| **精度问题** | 远平面的深度精度较低，注意设置合理的 near/far 比例 |
| **透明物体** | Z-Buffer 无法正确处理透明混合，需要特殊处理（从后往前绘制） |

## 总结

| 技术 | 解决的问题 | 核心原理 |
|-----|-----------|---------|
| **反走样** | 采样产生的锯齿和混叠 | 采样前去除高频信息（模糊） |
| **MSAA** | 精确计算覆盖程度 | 增加子像素采样点 |
| **FXAA** | 快速抗锯齿 | 图像后期处理，识别并平滑边缘 |
| **TAA** | 高效抗锯齿 | 时间维度复用上一帧采样结果 |
| **DLSS** | 超分辨率重建 | 深度学习智能重建细节 |
| **Z-Buffer** | 遮挡和可见性判断 | 维护最小深度值，顺序无关 |

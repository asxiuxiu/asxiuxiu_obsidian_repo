## 📐 坐标系约定

**本文所有讨论基于左手系（Left-Handed Coordinate System）**，与 DirectX、Unity、Unreal 等主流引擎一致。

| 约定项 | 左手系定义 | 与右手系的区别 |
|--------|-----------|---------------|
| **Z轴方向** | 指向屏幕**内**（远离观察者） | 右手系 Z 轴指向屏幕外 |
| **正旋转方向** | **顺时针** | 右手系为逆时针 |
| **正面顶点顺序** | **顺时针**排列为正面 | 右手系为逆时针 |

**口诀**：左手中指**戳进屏幕**。

> [!tip] 叉积判断
> 叉积公式 `cross(A, B) = A.x * B.y - A.y * B.x` 与坐标系**无关**。
> - `cross > 0`：按右手定则，B 在 A 的**左侧**
> - `cross < 0`：按右手定则，B 在 A 的**右侧**
>
> 在左手系中判断点是否在三角形内：若顶点按**顺时针**排列，则三个边缘函数值应**同号（全 ≤ 0）**。

## Part 1：直线光栅化

### Why：为什么要改进画法？

| 画法 | 代码量 | 移动线段 | 适用场景 |
|------|--------|----------|----------|
| 逐像素设置 | 数百行 | 重写所有像素 | 基本不会用 |
| **定义端点自动连线** | 几行 | 只改 2 个坐标 | 所有线条绘制 |

核心思路：**用几何描述代替逐像素操作。**

这种"根据两个端点自动填充中间像素"的技术，就是 **直线光栅化（Line Rasterization）**。

### What：直线光栅化要做什么？

```
输入：线段的 2 个端点 (x1,y1), (x2,y2)
         ↓
步骤 1：计算线段经过的所有像素
         ↓
步骤 2：把这些像素涂成指定颜色
         ↓
输出：一条画好的线段
```

#### 直观思路：基于方程的采样

数学上，直线方程是 `y = kx + b`。我们可以这样画：

```cpp
void drawLineNaive(Image& img, int x1, int y1, int x2, int y2, Color c) {
    float k = (float)(y2 - y1) / (x2 - x1);  // 斜率
    float b = y1 - k * x1;                    // 截距
    
    for (int x = x1; x <= x2; x++) {
        int y = (int)(k * x + b);   // 浮点运算 + 类型转换
        img.setPixel(x, y, c);
    }
}
```

**问题很明显**：
1. **浮点运算慢**：每个像素都要做乘法和加法
2. **精度累积误差**：浮点数存储和迭代会产生偏差
3. **不适用于陡线**：当 `|k| > 1` 时，y 方向跳跃，会断开

### How：Bresenham 直线算法

1962 年，IBM 的 Jack Bresenham 提出了一个天才方案：

> **只用整数加法，就能决定下一个像素该点哪里。**

#### 核心思想：增量计算

假设我们在画一条**平缓的直线**（`0 < 斜率 < 1`），每次 x 增加 1，y 要么不变，要么增加 1：

```
当前像素在 (x, y)
下一个像素只能是：
    候选 A: (x+1, y)     ← 保持 y
    候选 B: (x+1, y+1)   ← y 增加 1

选哪个？看谁离真实直线更近！
```

用 **决策参数 `e`** 记录"当前 y 与理想直线的垂直偏差"：

```
初始化：e = 0

每一步：
    e = e + 斜率   // 累积误差
    
    if (e >= 0.5) {
        y = y + 1;   // 偏离太多，向上走一步
        e = e - 1;   // 重置误差基准
    }
    
    画点 (x, y)
```

#### 优化：消除浮点

上面的 `0.5` 和斜率 `dy/dx` 都是浮点数。两边同乘 `2*dx`：

```
令 e' = e * 2*dx  （变成整数！）

判断条件 e >= 0.5  变成  e' >= dx
更新 e = e + dy/dx 变成 e' = e' + 2*dy
重置 e = e - 1    变成 e' = e' - 2*dx
```

**现在所有运算都是整数了！**

#### 代码实现

```cpp
// Bresenham 直线算法（适用于 0 < 斜率 < 1 的情况）
void drawLineBresenham(Image& img, int x1, int y1, int x2, int y2, Color c) {
    int dx = x2 - x1;
    int dy = y2 - y1;
    
    int y = y1;
    int e = 0;  // 决策参数（误差积累）
    
    for (int x = x1; x <= x2; x++) {
        img.setPixel(x, y, c);
        
        e += 2 * dy;        // 累积误差
        if (e >= dx) {      // 偏离超过半个像素
            y++;            // 向上走一步
            e -= 2 * dx;    // 重置误差
        }
    }
}
```

#### 扩展到所有象限

完整实现需要处理 8 种情况，核心技巧：

```cpp
void drawLine(Image& img, int x1, int y1, int x2, int y2, Color c) {
    int dx = abs(x2 - x1);
    int dy = abs(y2 - y1);
    
    bool steep = dy > dx;  // 判断是"陡"还是"平"
    
    if (steep) {
        // 陡线：交换 x 和 y，变成平缓情况处理
        swap(x1, y1);
        swap(x2, y2);
        swap(dx, dy);
    }
    
    if (x1 > x2) {
        // 确保从左到右画
        swap(x1, x2);
        swap(y1, y2);
    }
    
    int yStep = (y2 > y1) ? 1 : -1;
    
    int y = y1;
    int e = 0;
    
    for (int x = x1; x <= x2; x++) {
        if (steep) {
            img.setPixel(y, x, c);  // 注意坐标交换回来
        } else {
            img.setPixel(x, y, c);
        }
        
        e += 2 * dy;
        if (e >= dx) {
            y += yStep;
            e -= 2 * dx;
        }
    }
}
```

> [!note] 常见问题
> - **为什么浮点运算有精度问题？** `float` 只有 7 位有效数字，迭代累积误差会越来越明显，长直线的末端可能偏离 1-2 个像素。
> - **为什么陡线会断开？** 基于 x 采样的方法假设"x 每走 1 步，y 变化不超过 1 个像素"，对陡线不成立。解决方案是陡线沿 y 方向采样。

## Part 2：三角形光栅化

### Why：从线框到填充

直线算法可以画出三角形的**边框**：

```cpp
void drawTriangleWireframe(Image& img, int x1, int y1, int x2, int y2, int x3, int y3, Color c) {
    drawLine(img, x1, y1, x2, y2, c);
    drawLine(img, x2, y2, x3, y3, c);
    drawLine(img, x3, y3, x1, y1, c);
}
```

但 3D 渲染需要的是**实心填充**的三角形表面。怎么判断一个像素在三角形**内部**？

### What：三角形光栅化要做什么？

```
输入：三角形的 3 个顶点 (x1,y1), (x2,y2), (x3,y3)
         ↓
步骤 1：找出三角形覆盖的所有像素
         ↓
步骤 2：把这些像素涂成指定颜色
         ↓
输出：一张画好三角形的图片
```

#### 关键问题：如何判断一个像素在三角形内？

假设三角形的三个顶点是 `P1`, `P2`, `P3`，要判断点 `O` 是否在三角形内：

**判断方法（左手系，顶点顺时针排列）**：
- 从 P1 走到 P2，O 在右边 → OK
- 从 P2 走到 P3，O 在右边 → OK
- 从 P3 走到 P1，O 在右边 → OK
- 三个条件都满足，O 就在三角形内部

**数学工具**：用**叉乘**判断点在边的哪一侧。

### How 1：基础方法 —— 叉乘法

这是最直观的方法：**判断点相对于三角形三条边的位置关系**。

#### 步骤 1：定义基础数据结构

```cpp
struct Vec2 {
    float x, y;
    Vec2(float x = 0, float y = 0) : x(x), y(y) {}
    Vec2 operator-(const Vec2& o) const { return Vec2(x - o.x, y - o.y); }
};

struct Triangle {
    Vec2 p1, p2, p3;
    Triangle(const Vec2& a, const Vec2& b, const Vec2& c) : p1(a), p2(b), p3(c) {}
};
```

#### 步骤 2：叉乘 —— 判断方向的工具

```cpp
// 2D 叉乘：a × b = ax * by - ay * bx
// 结果 > 0：按右手定则，b 在 a 的左侧
// 结果 < 0：按右手定则，b 在 a 的右侧
// 结果 = 0：b 与 a 共线
float cross(const Vec2& a, const Vec2& b) {
    return a.x * b.y - a.y * b.x;
}
```

#### 步骤 3：判断点是否在三角形内

```cpp
bool pointInTriangle(const Vec2& p, const Triangle& tri) {
    // 三角形的三条边
    Vec2 e1 = tri.p2 - tri.p1;  // 边 P1→P2
    Vec2 e2 = tri.p3 - tri.p2;  // 边 P2→P3
    Vec2 e3 = tri.p1 - tri.p3;  // 边 P3→P1
    
    // 从顶点指向待判断点的向量
    Vec2 v1 = p - tri.p1;
    Vec2 v2 = p - tri.p2;
    Vec2 v3 = p - tri.p3;
    
    // 三次叉乘
    float c1 = cross(e1, v1);
    float c2 = cross(e2, v2);
    float c3 = cross(e3, v3);
    
    // 符号相同则在内部（全正或全负）
    // 左手系 + 顺时针顶点：应全 ≤ 0
    bool hasNeg = (c1 < 0) || (c2 < 0) || (c3 < 0);
    bool hasPos = (c1 > 0) || (c2 > 0) || (c3 > 0);
    
    return !(hasNeg && hasPos);
}
```

#### 步骤 4：光栅化主函数 + 包围盒优化

```cpp
void rasterizeTriangle(Image& image, const Triangle& tri, const Color& color) {
    // 计算包围盒，只遍历这个区域内的像素
    int minX = std::max(0, (int)std::min({tri.p1.x, tri.p2.x, tri.p3.x}));
    int maxX = std::min(image.getWidth() - 1, (int)std::max({tri.p1.x, tri.p2.x, tri.p3.x}) + 1);
    int minY = std::max(0, (int)std::min({tri.p1.y, tri.p2.y, tri.p3.y}));
    int maxY = std::min(image.getHeight() - 1, (int)std::max({tri.p1.y, tri.p2.y, tri.p3.y}) + 1);
    
    for (int y = minY; y <= maxY; y++) {
        for (int x = minX; x <= maxX; x++) {
            // 取像素中心点坐标
            Vec2 pixelPos(x + 0.5f, y + 0.5f);
            
            if (pointInTriangle(pixelPos, tri)) {
                image.setPixel(x, y, color);
            }
        }
    }
}
```

> [!tip] 包围盒优化效果
> 对于只占画面 10% 的三角形，优化后只需判断约 10% 的像素，速度提升近 10 倍。

### How 2：现代 GPU 方法 —— 边缘函数法

叉乘法很直观，但有个问题：**只做了一件事**——判断点在不在三角形内。

现代 GPU 需要的不只是"在内/在外"，还需要知道**在内部的哪里**，才能做**颜色插值**、**纹理采样**、**深度计算**等。

#### 边缘函数（Edge Function）

现代 GPU 使用 **边缘函数** 来进行光栅化。它的核心优势是：

> **一次计算，同时得到：① 内外测试结果 ② 插值所需的权重**

对于三角形的一条边（从 `P1` 到 `P2`），边缘函数定义为：

```
E(x, y) = (x - x1)(y2 - y1) - (y - y1)(x2 - x1)
        = cross(P - P1, P2 - P1)
```

**几何意义**：
- `E(x,y) > 0`：按右手定则，点 `(x,y)` 在边的**左侧**
- `E(x,y) < 0`：按右手定则，点 `(x,y)` 在边的**右侧**
- `E(x,y) = 0`：点在边上

在左手系中，若三角形顶点按顺时针排列，则内部点应满足 `E ≤ 0`。

> [!note] 边缘函数 vs 叉乘法
> 两者用的数学工具**完全一样**——都是二维叉乘。
> 区别只在于：**怎么用**和**算完之后干什么**。

#### 关键优势：增量计算

边缘函数最重要的优化是**增量计算**。

从一个像素到相邻像素，边缘函数值只需要加/减一个常数：

```cpp
e0(x+1, y) = e0(x, y) + (y2 - y1)  // 沿 X 方向，加 Δy
e0(x, y+1) = e0(x, y) - (x2 - x1)  // 沿 Y 方向，减 Δx
```

**为什么这是巨大优化？**

| 计算方式 | 每次像素运算 | 100×100 三角形需要 |
|----------|-------------|-------------------|
| **完整叉乘** | 2 次减法 + 2 次乘法 + 1 次减法 = **5 次运算** | 50,000 次运算 |
| **增量更新** | **1 次加法** | 10,000 次运算 |

#### 边缘函数 → 重心坐标

边缘函数最神奇的地方：**结果可以直接推导出重心坐标**。

对三角形的三个顶点 `P1, P2, P3`，定义三个边缘函数：

```
e0(x,y) = cross(P2 - P1, P - P1)  // 边 P1→P2
e1(x,y) = cross(P3 - P2, P - P2)  // 边 P2→P3
e2(x,y) = cross(P1 - P3, P - P3)  // 边 P3→P1
```

**三个边缘函数值 `e0, e1, e2` 正好对应重心坐标的分子！**

```cpp
// 总面积（2倍有向面积）
float area2 = e0 + e1 + e2;

// 重心坐标 = 各子三角形面积 / 总面积
float alpha = e1 / area2;  // 对应顶点 P1
float beta  = e2 / area2;  // 对应顶点 P2
float gamma = e0 / area2;  // 对应顶点 P3
```

> [!tip] 为什么是这样？
> 二维叉乘的几何意义是**平行四边形的有向面积**。
> 边缘函数 `e0 = cross(P - P1, P2 - P1)` 实际上计算的是**三角形 P-P1-P2 面积的 2 倍**。
> 而重心坐标的定义就是**面积比**：`α = area(PBC) / area(ABC)`

#### ⚠️ 符号统一：最容易出错的细节

边缘函数法的正确性**完全依赖于符号的统一性**。

```cpp
// ❌ 错误：写死判断条件
if (edge01 <= 0 && edge12 <= 0 && edge20 <= 0)  // 只对CW有效

// ✅ 正确：与 area2 保持符号一致
if (edge01 * area2 >= 0 && edge12 * area2 >= 0 && edge20 * area2 >= 0)
```

**原理**：
- 对于内部点，三个边缘函数值应该与总面积同号
- CW 排列：`area2 < 0`，内部点三个 `edge < 0`
- CCW 排列：`area2 > 0`，内部点三个 `edge > 0`
- 乘积 `edge * area2 > 0` 在两种情况下都成立

#### 完整代码实现

```cpp
void rasterizeTriangle(Image& img, Vertex v[3]) {
    // 1. 计算总面积（2倍有向面积）
    float area2 = cross2D(v[1].pos - v[0].pos, v[2].pos - v[0].pos);
    if (fabs(area2) < 1e-6) return;  // 退化三角形
    
    // 2. 计算包围盒
    int minX = std::max(0, (int)std::floor(std::min({v[0].pos.x, v[1].pos.x, v[2].pos.x})));
    int maxX = std::min(img.getWidth()-1, (int)std::ceil(std::max({v[0].pos.x, v[1].pos.x, v[2].pos.x})));
    int minY = std::max(0, (int)std::floor(std::min({v[0].pos.y, v[1].pos.y, v[2].pos.y})));
    int maxY = std::min(img.getHeight()-1, (int)std::ceil(std::max({v[0].pos.y, v[1].pos.y, v[2].pos.y})));
    
    // 3. 预计算增量步长
    float edge01StepX = v[0].pos.y - v[1].pos.y;  // -Δy
    float edge12StepX = v[1].pos.y - v[2].pos.y;
    float edge20StepX = v[2].pos.y - v[0].pos.y;
    
    // 4. 光栅化循环
    for (int y = minY; y <= maxY; y++) {
        // 计算行首边缘函数初始值
        Vec2 p = {minX + 0.5f, y + 0.5f};
        float e01 = cross2D(v[1].pos - v[0].pos, p - v[0].pos);
        float e12 = cross2D(v[2].pos - v[1].pos, p - v[1].pos);
        float e20 = cross2D(v[0].pos - v[2].pos, p - v[2].pos);
        
        for (int x = minX; x <= maxX; x++) {
            // 关键：用 area2 统一符号判断
            bool inside = (e01 * area2 >= 0) && (e12 * area2 >= 0) && (e20 * area2 >= 0);
            
            if (inside) {
                // 重心坐标（edge 和 area2 同号，自动得到正值）
                float alpha = e12 / area2;
                float beta  = e20 / area2;
                float gamma = e01 / area2;
                
                // 颜色插值
                Color finalColor = {
                    (uint8_t)(alpha * v[0].color.r + beta * v[1].color.r + gamma * v[2].color.r),
                    (uint8_t)(alpha * v[0].color.g + beta * v[1].color.g + gamma * v[2].color.g),
                    (uint8_t)(alpha * v[0].color.b + beta * v[1].color.b + gamma * v[2].color.b),
                    255
                };
                
                // Z-Buffer测试
                float z = alpha * v[0].z + beta * v[1].z + gamma * v[2].z;
                if (z < img.getZ(x, y)) {
                    img.setZ(x, y, z);
                    img.setPixel(x, y, finalColor);
                }
            }
            
            // 沿X方向增量更新（只用加法！）
            e01 += edge01StepX;
            e12 += edge12StepX;
            e20 += edge20StepX;
        }
    }
}
```

#### 方法对比

| 特性         | 叉乘法              | 边缘函数法                 |
| ---------- | ---------------- | --------------------- |
| **核心数学**   | 叉乘               | 叉乘（形式相同）              |
| **内外测试**   | ✅ 能做             | ✅ 能做                  |
| **颜色插值**   | ❌ 需要**额外计算**重心坐标 | ✅ 边缘函数值**直接就是**重心坐标分子 |
| **计算效率**   | 每个像素算3次叉乘        | 每行初始算1次，之后纯加法增量更新     |
| **GPU友好度** | 低（无法并行优化）        | 高（SIMD友好，可剔除整块像素）     |

## 总结

### 坐标系约定回顾

| 项目 | 左手系约定 |
|------|-----------|
| Z轴方向 | 指向屏幕内 |
| 正旋转 | 顺时针 |
| 叉积 > 0 | 按右手定则，点在边的左侧 |
| 正面 | 顺时针排列的顶点 |
| 内外测试 | 三个边缘函数值同号（顺时针时全 ≤ 0）|

### 知识的递进

| 阶段 | 方法 | 核心理解 | 解决的问题 |
|------|------|---------|-----------|
| **基础** | 叉乘法 | 用叉乘判断点在哪一侧 | 点在三角形内/外？ |
| **进阶** | 边缘函数 | 叉乘的增量计算优化 | 如何高效判断大量像素？ |
| **应用** | 重心坐标 | 面积比 = 插值权重 | 内部点的属性如何计算？ |

### 关键洞察

1. **叉乘法 ≠ 被淘汰**：它是最直观、最容易理解的方法。边缘函数本质上也是叉乘，只是用法更巧妙。

2. **边缘函数 ≠ 取代重心坐标**：边缘函数用于**光栅化**（判断哪些像素在内），重心坐标用于**插值**（计算内部点的属性）。两者是**互补关系**：边缘函数的结果可以直接算出重心坐标。

3. **现代 GPU 的实际流程**：
   ```
   光栅化器(边缘函数) → 得到 e0,e1,e2 → 判断在内 + 计算重心坐标 → 插值器 → 输出颜色
   ```

# 具身智能转型构建手册

> 本手册是一份**可执行的学习与实战指南**。每一步都必须是**可验证、可演示、可写入简历**的。我们不追求先读完所有理论再动手，而是**每完成一个阶段，GitHub 上就要多出一个可运行的项目**。
>
> **核心设计理念**：以**游戏引擎开发者的仿真能力**为杠杆，走「RL + 物理仿真 + Sim2Real」的工业级路线。拒绝从 CV/NLP 绕远路，拒绝「先学完所有数学再动手」。
>
> **转型哲学**：仿真优先，真机验证。先在 MuJoCo/Isaac 里训出走路的策略，再考虑买机器人。

## 阶段 1：AI 基础

**里程碑**：掌握深度学习的核心机制，能用 PyTorch 独立实现和训练神经网络。不是成为算法研究员，而是能读懂 RL 论文里的网络结构、能 Debug 训练代码、能调参。

> **为什么不是从 RL 直接开始？** 因为 PPO/SAC 的代码里全是 Value Network、Policy Network、Entropy Loss——如果你不懂 Backprop 和 Optimizer，看 RL 代码会像看天书。

**验收标准（可验证）**：
- [ ] 手写一个 MLP，在 MNIST 上达到 95%+ 准确率

- [ ] 能在 GPU 上训练，理解 `to(device)`、`cuda()`、`mixed precision` 的基本用法
- [ ] 能用 TensorBoard 可视化 loss 曲线和权重分布

### 1.1 PyTorch 核心机制

- [ ] [[PyTorch核心机制]] — Tensor 运算与自动微分、计算图与 `backward()`、`nn.Module` 生命周期、`DataLoader` 与采样、设备管理（CPU/GPU/MPS）、混合精度训练基础

> **下一步**：理解深度学习的基本组件。

## 阶段 2：强化学习核心

**里程碑**：理解 RL 的 MDP 范式，掌握 PPO 和 SAC 的核心机制，能用 Stable-Baselines3 训出稳定的连续控制策略。能在 TensorBoard 上分析奖励曲线、价值函数、策略熵。

> **为什么是 PPO/SAC？** PPO 是离散/连续控制通吃的工业标准（OpenAI、NVIDIA、宇树都在用），SAC 是 Off-Policy 连续控制的最优样本效率选择。掌握这两个，覆盖 90% 的机器人 RL 场景。

**验收标准（可验证）**：
- [ ] 能用 Gymnasium 的 `Pendulum-v1` 训出一个稳定倒立摆的策略
- [ ] 能用 Stable-Baselines3 的 PPO 在 `Walker2d-v4` 上训出持续行走的策略，平均奖励 > 2500
- [ ] 能解释 PPO 的 Clipped Objective 为什么能防止策略崩溃
- [ ] 能解释 Actor-Critic 架构中「策略网络」和「价值网络」各自的作用
- [ ] 能独立调整超参数（learning rate、gamma、gae_lambda、entropy coef）并观察效果
- [ ] 能用视频记录训练后的策略表现，保存为 `.mp4`

### 2.1 马尔可夫决策过程与 RL 范式

- [ ] [[马尔可夫决策过程]] — 状态/动作/奖励/策略/价值函数、贝尔曼方程、探索与利用、On-Policy vs Off-Policy、Model-Free vs Model-Based

> **下一步**：理解 Value-Based 方法的基础。

### 2.2 Value-Based 方法

- [ ] [[ValueBased方法]] — Q-Learning、DQN（经验回放 + 目标网络）、Double DQN、Dueling DQN

> **临时性声明**：DQN 主要用于离散动作空间（游戏），机器人控制是连续动作空间，所以 DQN 系列在这里是「建立直觉」而非「直接应用」。真正用于机器人的是策略梯度方法。
> **下一步**：进入连续控制的核心——策略梯度。

### 2.3 Policy Gradient 与 Actor-Critic

- [ ] [[PolicyGradient与ActorCritic]] — REINFORCE、Baseline 减方差、Actor-Critic 架构、GAE（广义优势估计）、连续动作空间的高斯策略

> **下一步**：掌握工业级标准算法 PPO。

### 2.4 PPO：工业级标准

- [ ] [[PPO算法深度解析]] — Clipped Surrogate Objective、KL 散度约束、多 epoch 小批量更新、Critic 损失与 Entropy 奖励、超参数敏感性与调参指南

> **核心增量**：PPO 是当前机器人 RL 的默认选择。必须能独立实现一个简化版 PPO，或至少逐行读懂 Stable-Baselines3 的 PPO 源码。
> **下一步**：了解样本效率更高的 Off-Policy 方法。

### 2.5 SAC：样本效率之选

- [ ] [[SAC算法深度解析]] — 最大熵框架、温度参数自动调节、双 Q 网络、重参数化技巧、Off-Policy 优势与稳定性

> **下一步**：在标准环境中跑通训练。

### 2.6 经典连续控制环境实践

- [ ] [[连续控制环境实践]] — Gymnasium 机器人环境族（Pendulum/CartPole/Hopper/HalfCheetah/Walker2d/Humanoid）、环境包装（Observation/Action/Reward Wrapper）、训练流程标准化、结果评估与视频生成

> **本阶段增量**：你能独立训出一个让机器人在仿真里走路/跑步的策略。这是具身智能的「Hello World」。
> **下一步**：标准环境只是热身，真正的挑战在自定义仿真环境。

## 阶段 3：机器人仿真平台

**里程碑**：掌握 MuJoCo 和 Isaac Gym/Sim，能从零搭建自定义机器人仿真环境，设计 Reward Function，实现并行训练。这是游戏引擎开发者建立壁垒的核心阶段。

> **为什么这是你的主场？** MuJoCo 的 XML 建模 ≈ 游戏引擎的 Prefab/Scene 文件；Isaac Sim ≈ headless 游戏引擎（PhysX + RTX）。你写自定义环境的速度会比纯算法背景的人快 5 倍。

**验收标准（可验证）**：
- [ ] 能在 MuJoCo 里加载一个 URDF/MJCF 模型，理解 `body`/`joint`/`actuator`/`sensor` 的语义

- [ ] 能设计一套 Reward Function，并解释每个奖励项的物理意义
- [ ] 能在 Isaac Gym 里并行运行 1024+ 个环境，GPU 加速训练
- [ ] 能修改环境参数（摩擦、质量、重力）做 Domain Randomization 初探
- [ ] 能用 `mujoco.mjv_camera` 录制训练过程的第三人称视角视频

## 阶段 4：机器人学与控制

**里程碑**：掌握机器人运动学和动力学，能理解 WBC 和 MPC 的优化问题形式，能阅读机器人控制领域的论文和代码。

> **为什么需要补这个？** 纯 RL 训出来的策略是「黑盒」，工业界需要的是「RL + 模型驱动控制」的混合方案。WBC/MPC 提供物理约束保证，RL 提供适应性。

**验收标准（可验证）**：
- [ ] 能推导出一个 2 连杆机械臂的正运动学（DH 参数法或 PoE 法）
- [ ] 能用数值优化（如 Jacobian 转置/伪逆）解逆运动学
- [ ] 能理解浮动基（Floating Base）机器人的动力学方程形式
- [ ] 能解释 WBC 的 QP 优化问题：目标函数 + 等式/不等式约束
- [ ] 能解释 MPC 的「预测未来 N 步、滚动优化」的核心思想
- [ ] 能用 Python 实现一个简单的双足站立平衡控制器（PD 或 LQR）

### 4.1 机器人运动学

- [ ] [[机器人运动学]] — 刚体位姿表示（旋转矩阵/四元数/欧拉角）、DH 参数法、指数积（PoE）法、正运动学推导、逆运动学解析解与数值解、Jacobian 与奇异点

> **游戏引擎映射**：游戏里的 Two-Bone IK、Look-At IK、Foot Locking = 机器人的逆运动学。
> **下一步**：理解力与运动的关系。

### 4.2 刚体动力学与浮动基

- [ ] [[刚体动力学与浮动基]] — 欧拉-拉格朗日方程、浮动基坐标系、接触约束与互补条件（Contact Wrench）、质心动力学（Centroidal Dynamics）、线性化模型（LIPM）

> **下一步**：学习工业级控制器。

### 4.3 全身控制（WBC）

- [ ] [[全身控制WBC]] — 任务层级（优先级/零空间投影）、QP 优化形式（目标函数 + 约束）、接触力约束、自碰撞约束、与 RL Policy 的混合架构（RL 输出高层目标 → WBC 执行）

> **核心增量**：WBC 是 Boston Dynamics、MIT Cheetah、ANYmal 的核心控制技术。RL 训策略 + WBC 做底层执行是当前工业界的主流架构。
> **下一步**：理解预测控制。

### 4.4 模型预测控制（MPC）

- [ ] [[模型预测控制MPC]] — 滚动时域优化、离散化与 QP 求解、MPC 与 WBC 的分层关系（MPC 规划质心轨迹 → WBC 分配关节力矩）、非线性 MPC 简介

> **下一步**：了解接触动力学的特殊处理。

### 4.5 接触动力学与步态规划

- [ ] [[接触动力学与步态规划]] — 接触检测与碰撞响应、摩擦锥（Friction Cone）、步态周期（Gait Cycle）、Raibert 启发式、基于优化的落足点规划

> **本阶段增量**：你能理解「机器人为什么能站稳不倒」的数学原理，能看懂控制领域的论文。
> **下一步**：把仿真里训好的策略搬到真实世界。

## 阶段 5：Sim2Real 与部署

**里程碑**：掌握从仿真到真实的迁移技术，能用 ROS2 与机器人通信，能把 PyTorch 模型部署到边缘设备。完成一次「仿真训练 → 模型导出 → 真实/高保真仿真验证」的完整闭环。

> **为什么这是高壁垒？** 90% 的 RL 工程师卡在「训出来跑得很漂亮，一上真机就摔跤」。Sim2Real 是工业界最值钱的技能之一。

**验收标准（可验证）**：
- [ ] 能用 Domain Randomization 让策略对质量/摩擦/推力的变化鲁棒
- [ ] 能把 PyTorch 模型导出为 ONNX，并用 ONNX Runtime 做推理
- [ ] 能用 ROS2 发布一个 Topic，让仿真机器人订阅并执行动作
- [ ] 能解释 Sim2Real Gap 的 5 个主要来源（物理参数差异、观测噪声、执行器延迟、接触模型差异、未建模动力学）
- [ ] 完成一次端到端的 Sim2Real 迁移实验（即使只是在更高保真的仿真里验证）

### 5.1 Sim2Real  Gap 分析

- [ ] [[Sim2RealGap分析]] — 物理参数差异、观测噪声与延迟、执行器动力学、接触模型不匹配、未建模自由度、系统辨识基础

> **下一步**：学习弥合 Gap 的第一道防线。

### 5.2 域随机化（Domain Randomization）

- [ ] [[域随机化]] — 参数空间随机化（质量/摩擦/质心）、观测随机化（噪声/ dropout）、动力学随机化、随机化的边界设计（过随机化导致策略保守）

> **下一步**：学习弥合 Gap 的第二道防线。

### 5.3 系统辨识与模型适配

- [ ] [[系统辨识与模型适配]] — 参数辨识（最小二乘/最大似然）、残差学习（Residual Learning）、Sim-to-Sim 验证流程

> **下一步**：学习部署基础设施。

### 5.4 ROS2 基础与机器人通信

- [ ] [[ROS2基础与机器人通信]] — Node/Topic/Service/Action、QoS 策略、`rclcpp`/`rclpy` 基础、与 Isaac Sim 的 ROS2 桥接、与真实机器人的接口（宇树 SDK）

> **核心增量**：ROS2 是机器人领域的「操作系统标准」。不懂 ROS2 就无法与真实硬件对话。
> **下一步**：模型部署优化。

### 5.5 模型导出与边缘部署

- [ ] [[模型导出与边缘部署]] — PyTorch → ONNX → TensorRT/ONNX Runtime、量化（INT8/FP16）、推理延迟优化、C++ 推理部署（libtorch/onnxruntime）、Jetson/树莓派部署基础

> **游戏引擎映射**：这和你在游戏引擎里做 Shader 变体、纹理压缩、Draw Call 优化的目标一样——在有限算力下保证实时性。
> **下一步**：Teacher-Student 蒸馏等高级迁移技术。

### 5.6 Teacher-Student 与感官策略

- [ ] [[TeacherStudent与感官策略]] — 全知 Teacher Policy（Proprioception + Ground Truth）、感官 Student Policy（仅关节 + IMU）、模仿学习（Behavior Cloning）、DAgger 算法简介

> **本阶段增量**：你能完成「仿真训练 → 策略优化 → 部署验证」的完整工业流程。
> **下一步**：选一个方向深耕。

## 阶段 6：专项深入

**里程碑**：在四足/人形/机械臂/仿真平台开发中选择一个主攻方向，达到「能独立复现该方向顶会论文、能训出工业级策略、能讲清楚设计决策」的水平。

> **关键决策**：四个方向互有交集，但必须选一个作为「突破口」。你的游戏引擎背景在「仿真平台开发」和「人形/四足控制」上最有优势。

**验收标准（可验证）**：
- [ ] 在所选方向上复现至少 1 篇近 3 年的顶会论文（RSS/CoRL/ICRA/ICLR）
- [ ] 在 GitHub 上有一个完整的项目（代码 + 训练脚本 + 结果视频 + README）
- [ ] 能写出该方向的技术博客，解释核心算法和自己的调参记录
- [ ] 能回答该方向的面试级问题（如「PPO 在人形上的 Reward 怎么设计」「WBC 和 RL 怎么分层」）

### 6.1 方向 A：人形机器人全身控制

- [ ] [[人形机器人全身控制]] — 人形动力学特殊性（高自由度/浮动基/欠驱动）、Zero Moment Point（ZMP）、捕获点（Capture Point）、人形 RL 的 Observation/Action 空间设计、宇树 H1/Figure 的公开技术路线

> **工业现状**：Figure AI、1X、宇树、傅利叶、智元、银河通用都在做人形。RL + WBC 是主流。
> **推荐项目**：复现 Humanoid-Gym，训一个宇树 H1 的行走/站立策略。

### 6.2 方向 B：四足机器人 Locomotion

- [ ] [[四足机器人Locomotion]] — 足式机器人步态（Trot/Pace/Bound/Gallop）、Raibert 控制器、MIT Cheetah 控制架构、ANYmal RL 训练流程、落足点规划（ foothold planning ）

> **工业现状**：宇树 Go2、小米 CyberDog、波士顿 Dynamics Spot。四足是 RL 控制最成熟的场景。
> **推荐项目**：复现 Legged Gym 的 ANYmal 训练，做 Domain Randomization + Sim2Real 迁移。

### 6.3 方向 C：机械臂操作（VLA）

- [ ] [[机械臂操作与VLA]] — 操作空间控制（Operational Space Control）、抓取姿态检测（Grasp Pose Detection）、模仿学习（Diffusion Policy / ACT）、Vision-Language-Action（VLA）模型、Sim2Real 在操作任务中的特殊挑战

> **工业现状**：Physical Intelligence（π0）、智元、银河通用。VLA 是大模型+机器人的交叉热点。
> **注意**：这个方向需要补 CV 和多模态基础，偏离你的核心优势（控制+仿真）。建议作为第二方向，不作为首选突破口。

### 6.4 方向 D：仿真平台开发

- [ ] [[仿真平台开发]] — 可微物理仿真（Brax/MuJoCo MJX/DiffTaichi）、GPU 并行架构、传感器模型、场景生成（ProcGen）、基准测试环境设计（Benchmark）、与 Isaac Sim/SAPIEN 的架构对照

> **核心优势**：这就是「用游戏引擎技术做机器人仿真」。NVIDIA、Google DeepMind、伯克利都在招人做这件事。你的 C++、物理引擎、GPU 优化经验直接对口。
> **推荐项目**：基于 MuJoCo 或 Brax，搭建一个自定义的机器人学习基准环境，开源到 GitHub。
> **本阶段增量**：你在所选方向上达到「能独立工作」的水平。
> **下一步**：把学习成果转化为可展示的作品集。

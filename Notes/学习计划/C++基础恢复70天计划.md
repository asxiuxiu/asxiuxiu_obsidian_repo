# C++ 基础恢复 70 天计划

> 目标：从「写对一个类」开始，逐步爬升到「手写引擎模块」。每天 30-40 分钟，不赶进度，每一步都踩在上一周的肩膀上。

> [!info] 如何使用 AI 辅助框架
> 本计划配套了一套 C++ 刻意练习 Skill 框架，在 Kimi CLI 中随时通过自然语言调用：
> - **卡住时**：说"给点提示" → 三级渐进提示（方向→线索→填空），不会直接给答案
> - **写完后**：说"帮我看看" → 四维度代码评审，严格对齐当天目标不超纲
> - **结束时**：说"今天练完了" → 自动校验知识→记录能力→调整后续计划
> - **其他**："考考我" / "我的水平怎么样" / "调整计划"
> 详细用法见 `workspace/cpp-recovery/README.md`

> [!tip] 单独打开工作区时的进度追踪
> 如果你单独用 VS Code 打开 `workspace/cpp-recovery/` 写代码，进度勾选请在工作区内的 `PROGRESS.md` 中完成。该文件与这份笔记内容同步，单独打开工作区时无需切回 vault。
> 这份笔记保留完整的题目描述、示例代码和知识讲解；`PROGRESS.md` 只保留进度表格，方便每天快速勾选。

## 10 周渐进方案总览

| 阶段 | 周数 | 主题 | 目标 |
|------|------|------|------|

## 第一阶段：C++ 语法热身（第 1-2 周）

### 第 1 周：类与对象的生命周期

目标：先不碰复杂数据结构，把「写类」的肌肉记忆找回来。

| 天数    | 题目                                     | 难度  | 核心考察点                  | 状态  |
| ----- | -------------------------------------- | --- | ---------------------- | --- |
| Day 1 | 写一个 `String` 类（只含 `char*` 和 `size_t`）  | ⭐   | 构造函数、析构函数、深拷贝          | [ ] |
| Day 2 | 给昨天的 `String` 加上拷贝构造和拷贝赋值              | ⭐   | 拷贝语义、自赋值检查             | [ ] |
| Day 3 | 给 `String` 加上移动构造和移动赋值                 | ⭐⭐  | 移动语义、右值引用、`noexcept`   | [ ] |
| Day 4 | 写一个 `Array<T, N>` 固定大小数组模板             | ⭐   | 模板基础、`operator[]`、边界检查 | [ ] |
| Day 5 | 写一个 `UniquePtr<T>`（简化版，不支持数组）          | ⭐⭐  | RAII、模板、删除器            | [ ] |
| Day 6 | 写一个 `FileGuard` RAII 类（封装 `FILE*`）     | ⭐   | RAII 思想、异常安全           | [ ] |
| Day 7 | **复盘日**：不看旧代码，重写 Day 3 的 `String`（完整版） | —   | 固化肌肉记忆                 | [ ] |

> [!note] Day 1 提示
> ```cpp
> class String {
>     char* data_;
>     size_t len_;
> public:
>     String(const char* str);  // 构造函数
>     ~String();                // 析构函数
>     // 今天先不写拷贝/移动，只写构造和析构
> };
> ```
> 自检：能解释为什么析构函数要 `delete[]`，能画出内存布局图。

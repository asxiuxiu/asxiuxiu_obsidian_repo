# C++ 基础恢复 70 天计划（笔记先行版）

> 目标：从「读透一篇笔记」开始，再到「手写一个类」，逐步爬升到「手写引擎模块」。每天 30-40 分钟，**笔记先行，练习跟进**。

> [!info] 核心原则：先读后写
> 本计划已根据 `Notes/C++编程/` 下的实际笔记重新编排。每一天的任务都标注了**前置笔记**——这些笔记必须先读完，再打开编辑器写代码。如果某天标注的笔记尚未产出（⬜ 或「待产出」），你需要先完成笔记学习（自行查阅资料或请求 AI 协助产出），再进行代码练习。
>
> 刻意练习的有效性建立在「有概念输入，才有代码输出」之上。不再摸黑写代码。

> [!info] 如何使用 AI 辅助框架
> 本计划配套了一套 C++ 刻意练习 Skill 框架，在 Kimi CLI 中随时通过自然语言调用：
> - **卡住时**：说"给点提示" → 三级渐进提示（方向→线索→填空），不会直接给答案
> - **写完后**：说"帮我看看" → 四维度代码评审，严格对齐当天目标不超纲
> - **结束时**：说"今天练完了" → 自动校验知识→记录能力→调整后续计划
> - **其他**："考考我" / "我的水平怎么样" / "调整计划"
> 详细用法见 `workspace/cpp-recovery/README.md`

> [!tip] 单独打开工作区时的进度追踪
> 如果你单独用 VS Code 打开 `workspace/cpp-recovery/` 写代码，进度勾选请在工作区内的 `PROGRESS.md` 中完成。该文件与这份笔记内容同步，单独打开工作区时无需切回 vault。

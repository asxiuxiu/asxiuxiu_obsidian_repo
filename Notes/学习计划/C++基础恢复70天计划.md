# C++ 基础恢复练习计划（笔记先行版）

> 目标：以 `Notes/C++编程/索引.md` 的 17 个阶段为骨架，**读完一个阶段、写一组练习**，逐步从「能写一个安全类」爬升到「能手写引擎模块」。不再限定 70 天，节奏由你控制。

> [!info] 核心原则：先读后写
> 本计划完全依据 `Notes/C++编程/` 下的笔记索引编排。每个练习都标注了**前置笔记**——必须先读完这些笔记，再打开编辑器写代码。如果某篇笔记尚未产出（⬜），你需要先完成笔记学习（自行查阅资料或请求 AI 协助产出），再进行代码练习。
>
> 刻意练习的有效性建立在「有概念输入，才有代码输出」之上。不再摸黑写代码。

> [!info] 如何使用 AI 辅助框架
> 本计划配套了一套 C++ 刻意练习 Skill 框架，在 Kimi CLI 中随时通过自然语言调用：
> - **卡住时**：说"给点提示" → 三级渐进提示（方向→线索→填空），不会直接给答案
> - **写完后**：说"帮我看看" → 四维度代码评审，严格对齐当前阶段目标不超纲
> - **结束时**：说"今天练完了" → 自动校验知识→记录能力→调整后续计划
> - **其他**："考考我" / "我的水平怎么样" / "调整计划"
> 详细用法见 `workspace/cpp-recovery/README.md`

> [!tip] 单独打开工作区时的进度追踪
> 如果你单独用 VS Code 打开 `workspace/cpp-recovery/` 写代码，进度勾选请在工作区内的 `PROGRESS.md` 中完成。该文件与这份笔记内容同步，单独打开工作区时无需切回 vault。
> 权威完整计划见 `Notes/学习计划/C++基础恢复70天计划.md`；`PROGRESS.md` 只保留进度表格，方便每天快速勾选。

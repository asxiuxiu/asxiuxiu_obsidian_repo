# 迭代器分类与算法复杂度保证

> [!info] 一句话概括
> C++ 迭代器按能力分为输入、前向、双向、随机访问、连续五类；算法根据迭代器类别通过 **tag dispatch** 选择实现，从而同时保证正确性和复杂度承诺。

---

## 问题 0：为什么 `std::sort` 不能用于 `std::list`？

想象一下，你写了一个任务列表，想用 `std::sort` 直接排序：

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <list>

int main() {
    std::list<int> tasks = {3, 1, 4, 1, 5};

    // std::sort(tasks.begin(), tasks.end()); // ❌ 编译失败！
    tasks.sort(); // list 自己的成员函数

    for (int x : tasks) std::cout << x << " ";
    std::cout << "\n";
    return 0;
}
```

`std::sort` 需要随机访问迭代器，因为它要频繁用 `it + n`、`it[j]` 这类操作来分区、交换元素。而 `std::list` 只提供双向迭代器，无法 `O(1)` 跳到任意位置，所以标准库的 `std::sort` 拒绝编译。

---

## 问题 1：迭代器到底分哪几类？

C++ 标准把迭代器按能力从弱到强分成五类：

| 类别 | 核心能力 | 典型容器 |
|------|---------|---------|
| **Input** | 只读、单向遍历、只能走一遍 | `std::istream_iterator` |
| **Forward** | 可读可写、单向遍历、可多遍遍历 | `std::forward_list`、`std::unordered_*` |
| **Bidirectional** | 可加 `++` 和 `--` | `std::list`、`std::set`、`std::map` |
| **Random Access** | 可加 `+ n`、可 `[]`、可比较大小 | `std::vector`、`std::deque`、原始指针 |
| **Contiguous**（C++17） | 元素在内存中连续存放 | `std::vector`、原始指针、`std::array` |

> [!abstract]
> **迭代器分类（Iterator Category）**不是类型的「标签装饰」，而是对算法能力的承诺。一个算法接受某种迭代器，就意味着它只使用该类迭代器允许的操作；使用更强的迭代器只是让这个算法跑得更快或实现更简单。

---

## 问题 2：迭代器能力是「递增」的意味着什么？

每一类迭代器都保留前一类的能力，并增加新操作：

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <iterator>
#include <list>
#include <vector>
#include <forward_list>
#include <type_traits>

template<typename It>
void show_category() {
    using Cat = typename std::iterator_traits<It>::iterator_category;
    if constexpr (std::is_same_v<Cat, std::input_iterator_tag>) {
        std::cout << "input iterator\n";
    } else if constexpr (std::is_same_v<Cat, std::forward_iterator_tag>) {
        std::cout << "forward iterator\n";
    } else if constexpr (std::is_same_v<Cat, std::bidirectional_iterator_tag>) {
        std::cout << "bidirectional iterator\n";
    } else if constexpr (std::is_same_v<Cat, std::random_access_iterator_tag>) {
        std::cout << "random access iterator\n";
    } else if constexpr (std::is_same_v<Cat, std::contiguous_iterator_tag>) {
        std::cout << "contiguous iterator\n";
    }
}

int main() {
    std::vector<int> v;
    std::list<int> l;
    std::forward_list<int> fl;

    show_category<decltype(fl.begin())>();
    show_category<decltype(l.begin())>();
    show_category<decltype(v.begin())>();
    show_category<int*>(); // 原始指针是 contiguous iterator
    return 0;
}
```

> [!tip]
> C++20 起，很多容器迭代器升级成了 `std::contiguous_iterator_tag`，这让标准库和编译器能进一步做向量化优化。原始指针天然就是 contiguous iterator。

## 问题 3：算法怎么用 tag dispatch 选择实现？

`std::advance` 和 `std::distance` 是 tag dispatch 的经典例子。对于随机访问迭代器，它们可以直接用 `+ n` 或 `it2 - it1`，复杂度是 `O(1)`；对于输入/前向/双向迭代器，只能一步步走，复杂度是 `O(n)`。

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <iterator>
#include <list>
#include <vector>

namespace my {

template<typename It>
void advance_impl(It& it, int n, std::random_access_iterator_tag) {
    it += n; // O(1)
}

template<typename It>
void advance_impl(It& it, int n, std::input_iterator_tag) {
    while (n-- > 0) ++it; // O(n)
}

template<typename It>
void advance_impl(It& it, int n, std::bidirectional_iterator_tag) {
    if (n >= 0) {
        while (n-- > 0) ++it;
    } else {
        while (n++ < 0) --it;
    }
}

template<typename It>
void advance(It& it, int n) {
    using Cat = typename std::iterator_traits<It>::iterator_category;
    advance_impl(it, n, Cat{});
}

} // namespace my

int main() {
    std::vector<int> v = {0, 10, 20, 30, 40, 50};
    auto vit = v.begin();
    my::advance(vit, 3); // O(1)
    std::cout << "vector it -> " << *vit << "\n";

    std::list<int> l = {0, 10, 20, 30, 40, 50};
    auto lit = l.begin();
    my::advance(lit, 3); // O(n)
    std::cout << "list it -> " << *lit << "\n";

    // 对于双向迭代器，也支持负向移动
    my::advance(lit, -2);
    std::cout << "list it back -> " << *lit << "\n";
    return 0;
}
```

> [!abstract]
> **Tag dispatch** 就是在编译期根据迭代器类别分发到不同的实现函数。因为迭代器类别是编译期已知的类型标签，这种分发完全零开销。

## 问题 4：`std::sort` 的复杂度保证是怎么来的？

`std::sort` 通常实现为 **Introsort**：先用快速排序，当递归深度超过阈值时回退到堆排序，从而保证最坏情况 `O(n log n)`，同时保持快速排序的平均常数优势。

这些操作都依赖随机访问迭代器：

- 选 pivot 后要跳到中间、末尾元素；
- partition 过程中要交换任意两个位置的元素；
- 堆排序需要 `i`、`2i+1`、`2i+2` 这样的索引访问。

```cpp
// flags: -std=c++20 -Wall -O2
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {9, 3, 7, 1, 5, 8, 2, 6, 4};
    std::sort(v.begin(), v.end()); // O(n log n)

    for (int x : v) std::cout << x << " ";
    std::cout << "\n";
    return 0;
}
```

> [!warning]
> `std::list` 提供了自己的 `sort` 成员函数，它使用归并排序，复杂度也是 `O(n log n)`，但实现方式与 `std::sort` 完全不同——这再次说明「算法根据迭代器能力选择实现」。

## 问题 5：引擎里为什么还要手写迭代器，或者干脆不用迭代器？

标准库迭代器抽象强大，但引擎中经常遇到两类问题：

1. **SoA/AOSOA 布局**：一个「元素」的字段分散在多个数组里，传统的 `T& operator*()` 无法直接返回一个连续对象。
2. **索引压缩**：ECS 中常用 `SparseSet` 存储活跃实体 ID，遍历的是 ID 数组而不是实体对象本身。

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <vector>

// 一个简单的 IndexIterator：遍历索引数组，不直接暴露对象
class IndexIterator {
public:
    explicit IndexIterator(const int* p) : ptr_(p) {}

    int operator*() const { return *ptr_; }
    IndexIterator& operator++() { ++ptr_; return *this; }
    bool operator!=(const IndexIterator& other) const { return ptr_ != other.ptr_; }

    // 连续迭代器还可以提供 + n
    IndexIterator operator+(int n) const { return IndexIterator(ptr_ + n); }

private:
    const int* ptr_;
};

int main() {
    std::vector<int> active_entities = {3, 7, 12, 15, 21};

    for (auto it = IndexIterator(active_entities.data());
         it != IndexIterator(active_entities.data() + active_entities.size());
         ++it) {
        std::cout << "processing entity " << *it << "\n";
    }
    return 0;
}
```

> [!note]
> 手写迭代器时，要根据它实际支持的操作选择合适的 `iterator_category`。如果明明只支持 `++` 却声称是 random access，算法会调用 `+= n` 等操作导致编译错误或未定义行为。

---

## 总结

- C++ 迭代器分为输入、前向、双向、随机访问、连续五类，能力递增。
- 算法通过 **tag dispatch** 根据迭代器类别选择实现，保证复杂度承诺。
- `std::sort` 需要随机访问迭代器，因此不能用于 `std::list`；`std::advance`/`std::distance` 对随机访问迭代器是 `O(1)`，对其他迭代器是 `O(n)`。
- 引擎中手写迭代器时，要诚实声明其类别；SoA、ECS 索引遍历等场景下，往往直接使用指针或索引数组代替完整迭代器抽象。

---

## 引擎映射

| 引擎 | 应用场景 | 具体体现 |
|------|---------|---------|
| **SelfGameEngine** | 自定义 `Array` 迭代器 | `Array<T>::iterator` 通常是连续/随机访问迭代器，支持 `+=`、`-`、`[]`，能被 `std::sort` 直接使用 |
| **UE** | `TArray` 迭代器 | `TArray<T>::RangedForIteratorType` 提供随机访问能力，`TMap`、`TSet` 提供双向或前向迭代器 |
| **ECS/稀疏集** | 实体索引遍历 | `SparseSet` 遍历活跃实体 ID 数组，使用原始指针或轻量 IndexIterator，避免构造完整组件对象 |

> [!note] 关键取舍
> 标准库迭代器抽象方便算法复用，但在 SoA/ECS 等布局下，「一个对象的所有字段连续存放」这一假设不成立。引擎中常常在底层用指针/索引直接遍历，在上层用更专门的查询接口（如 `Query<Transform, Velocity>`）隐藏迭代器细节。

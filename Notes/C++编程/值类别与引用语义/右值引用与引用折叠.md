# 右值引用与引用折叠

> [!info] 一句话概括
> **右值引用（`T&&`）**可以绑定到临时对象；在模板推导中，它配合**引用折叠**规则能同时接受左值和右值，这就是**万能引用（universal reference）**，也是 `std::forward` 实现完美转发的基础。

---

## 问题 0：右值引用和左值引用到底差在哪？

左值引用 `T&` 只能绑定到有名对象，右值引用 `T&&` 可以绑定到临时对象。最直观的区别体现在函数重载上：

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <string>

void foo(const std::string& s) {
    std::cout << "lvalue overload: " << s << "\n";
}

void foo(std::string&& s) {
    std::cout << "rvalue overload: " << s << "\n";
}

int main() {
    std::string a = "hello";
    foo(a);              // 左值，走 const& 版本
    foo(std::string{"world"}); // 右值，走 && 版本
    foo("literal");      // 临时 string，走 && 版本
    return 0;
}
```

> [!abstract]
> **右值引用**是 C++11 引入的新引用类型，语法上比左值引用多一个 `&`。它的核心语义是「绑定到即将被销毁的临时对象」，因此常用于实现移动语义：既然临时对象马上就要消失，不如把它的资源直接「偷」过来。

---

## 问题 1：引用折叠是怎么一回事？

C++ 语法上不允许「引用的引用」，比如你不能写 `int&& &`。但在模板实例化过程中，编译器内部确实会产生类似 `T& &`、`T&& &` 这样的组合，这时就要用**引用折叠规则**来决定最终类型。

四条规则如下：

```
T&  &   → T&
T&  &&  → T&
T&& &   → T&
T&& &&  → T&&
```

只要有一个左值引用参与，结果就折叠成左值引用；只有全部是右值引用时，结果才是右值引用。

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <type_traits>

template<typename T> using LRef = T&;
template<typename T> using RRef = T&&;

int main() {
    static_assert(std::is_same_v<LRef<int>&,  int&>);
    static_assert(std::is_same_v<LRef<int>&&, int&>);
    static_assert(std::is_same_v<RRef<int>&,  int&>);
    static_assert(std::is_same_v<RRef<int>&&, int&&>);

    std::cout << "reference collapsing ok\n";
    return 0;
}
```

这些组合不会直接出现在你的源代码里，但通过 `using` 别名、模板参数推导、`decltype` 等机制，编译器内部会频繁遇到它们。引用折叠规则保证了最终类型总是合法的单一引用。

## 问题 2：为什么 `T&&` 在模板中能成为万能引用？

当函数模板写成下面这样时，参数 `T&&` 就是一个**万能引用**：

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <type_traits>

template<typename T>
void bar(T&& x) {
    if constexpr (std::is_lvalue_reference_v<T>) {
        std::cout << "called with lvalue\n";
    } else {
        std::cout << "called with rvalue\n";
    }
}

int main() {
    int i = 0;
    bar(i);           // T 推导为 int&，T&& 折叠为 int&，接受左值
    bar(42);          // T 推导为 int，T&& 为 int&&，接受右值
    bar(std::move(i)); // T 推导为 int&？不，std::move(i) 是 xvalue，T 推导为 int，T&& 为 int&&
    return 0;
}
```

> [!tip]
> 调用 `bar(i)` 时，因为实参 `i` 是左值，模板参数 `T` 被推导为 `int&`。于是 `T&&` 变成 `int& &&`，根据引用折叠规则，最终是 `int&`。这就是为什么同一个 `T&&` 既能接受左值，又能接受右值。

## 问题 3：什么样的 `T&&` 才是万能引用？

并不是所有 `T&&` 都是万能引用。万能引用必须同时满足两个条件：

1. 它出现在**模板参数推导**的语境中；
2. 参数类型**恰好**是 `T&&`，没有任何 `const`、`volatile` 等修饰。

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <type_traits>

template<typename T>
void universal(T&& x) {
    std::cout << "universal\n";
}

template<typename T>
void not_universal(const T&& x) { // const 修饰，不是万能引用
    std::cout << "const rvalue ref\n";
}

void concrete(int&& x) { // 不是模板，只是普通右值引用
    std::cout << "concrete rvalue ref\n";
}

int main() {
    int i = 0;
    universal(i); // OK，万能引用接受左值

    // not_universal(i); // ❌ const T&& 不能绑定到非 const 左值
    not_universal(42); // OK

    // concrete(i); // ❌ 普通右值引用不能绑定左值
    concrete(42); // OK
    return 0;
}
```

> [!warning]
> `auto&&` 在变量声明中也是万能引用，遵循同样的引用折叠规则。比如 `auto&& r = i;` 中 `r` 是左值引用；`auto&& r = 42;` 中 `r` 是右值引用。

## 问题 4：`std::move` 和 `std::forward` 为什么离不开右值引用？

`std::move` 本身并不移动数据，它只是把参数**无条件地转换成右值引用**，告诉编译器「这个对象可以被偷资源」。

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <string>
#include <utility>

void consume(std::string&& s) {
    std::cout << "consumed: " << s << "\n";
}

int main() {
    std::string a = "hello";
    // consume(a); // ❌ a 是左值，不能直接绑定到 &&
    consume(std::move(a)); // OK，move 把 a 转成右值引用
    return 0;
}
```

`std::forward` 则更精细：它只在模板参数 `T` 被推导为右值时才转回右值，否则保留左值。这正是万能引用 + 引用折叠的产物。

```cpp
// flags: -std=c++20 -Wall -O2
#include <iostream>
#include <string>
#include <utility>

struct Widget {
    Widget(const std::string& s) { std::cout << "copy ctor\n"; }
    Widget(std::string&& s)      { std::cout << "move ctor\n"; }
};

template<typename T>
Widget make_widget(T&& arg) {
    return Widget(std::forward<T>(arg));
}

int main() {
    std::string s = "data";
    auto w1 = make_widget(s);           // arg 是左值，forward 后仍是左值 → 拷贝
    auto w2 = make_widget(std::string{"tmp"}); // arg 是右值，forward 后转回右值 → 移动
    return 0;
}
```

> [!abstract]
> `std::move` 是「无条件转右值」，适合明确要转移所有权的场景；`std::forward` 是「有条件转右值」，适合把万能引用参数原封不动地传递给下一层函数，这就是**完美转发**。

## 总结

- 右值引用 `T&&` 用于绑定临时对象，是移动语义的基础。
- 引用折叠四条规则保证「引用的引用」在模板内部能坍缩成合法类型：有左值引用参与就折叠成左值引用。
- 模板中的 `T&&`（或 `auto&&`）是**万能引用**，能根据实参的值类别推导出左值引用或右值引用。
- 万能引用必须满足「模板推导」且「无 cv 修饰」两个条件；`const T&&` 和普通函数中的 `T&&` 都不是万能引用。
- `std::move` 无条件转右值，`std::forward<T>` 根据 `T` 的推导结果决定是否转右值，二者共同支撑完美转发。

---

## 引擎映射

| 引擎 | 应用场景 | 具体体现 |
|------|---------|---------|
| **SelfGameEngine** | 泛型容器原地构造 | `Array::emplace_back(Args&&... args)` 使用万能引用 + `std::forward` 把参数直接转发给元素构造函数 |
| **SelfGameEngine** | 工厂函数 | `MakeUnique<T>(Args&&...)` 通过完美转发避免不必要的拷贝，尤其是构造重型资源句柄时 |
| **UE** | `TVariant`、`TUniquePtr` 的转发构造 | UE 的模板容器和智能指针大量使用 `Forward<Args...>` 实现与标准库等效的完美转发 |

> [!note] 关键取舍
> 右值引用让 C++ 能在不引入垃圾回收的前提下实现高效的资源转移。但万能引用容易与重载产生意想不到的交互——比如为万能引用函数同时提供重载时，左值调用可能匹配到出乎意料的版本。引擎中设计泛型接口时，通常用 SFINAE 或 Concepts 对万能引用参数做约束，避免过度贪婪的匹配。

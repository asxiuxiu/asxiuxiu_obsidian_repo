# 自增自减的前置与后置

> [!info] 一句话概括
> 前置 `++i` 返回自增后的对象本身（引用），后置 `i++` 返回自增前的旧值（拷贝）；对于自定义类型，后置版本通常多一次拷贝构造，循环里优先用前置。

---

## 问题 0：内置类型的 `++i` 和 `i++` 都能让变量加 1，我为什么要注意顺序？

对于 `int` 这种内置类型，下面的代码在结果上几乎没区别：

```cpp
// flags: -O0 -g
#include <iostream>

int main() {
    int a = 0, b = 0;
    int x = ++a;  // x = 1, a = 1
    int y = b++;  // y = 0, b = 1
    std::cout << "x=" << x << ", y=" << y << "\n";
    return 0;
}
```

区别在于**返回值**：

- `++a` 先把 `a` 加到 1，然后返回 `a` 本身（一个左值，可以继续被修改）。
- `b++` 先返回 `b` 原来的值 0（一个右值），然后再把 `b` 加到 1。

对于 `int`，现代编译器会把 `i++` 优化得和 `++i` 一样快。但一旦换成**自定义类型**（比如迭代器、计数器），这个区别就变得非常明显，因为后置版本必须创建一个旧值的副本。

---

## 问题 1：自定义类型的前置和后置自增，签名到底怎么写？

C++ 用参数列表来区分前置和后置：

- 前置：`T& operator++()` —— 没有参数，返回当前对象的引用。
- 后置：`T operator++(int)` —— 带一个**无用的 `int` 哑参数**，返回自增前的旧值（按值返回）。

```cpp
// flags: -O0 -g
#include <iostream>

struct Counter {
    int value = 0;

    // 前置：先自增，再返回自身引用
    Counter& operator++() {
        ++value;
        return *this;
    }

    // 后置：先保存旧值，再自增，最后返回旧值
    Counter operator++(int) {
        Counter old = *this;  // 拷贝构造
        ++value;
        return old;           // 按值返回，再次拷贝
    }
};

int main() {
    Counter c;
    ++c;
    std::cout << "after prefix: " << c.value << "\n";  // 1

    Counter d = c++;
    std::cout << "d.value=" << d.value << ", c.value=" << c.value << "\n";  // 1, 2
    return 0;
}
```

那个 `int` 参数没有名字，只是为了让编译器能分辨"这是后置版本"。调用 `c++` 时，编译器自动传一个 `0` 进去；你不需要也不能手动传参。

## 问题 2：后置版本的代码总是和前置重复，能不能只写一遍？

可以，而且这是推荐做法：**后置版本通过调用前置版本来实现**。这样逻辑只维护一份，避免两者不一致。

```cpp
// flags: -O0 -g
#include <iostream>

struct Counter {
    int value = 0;

    Counter& operator++() {
        ++value;
        return *this;
    }

    Counter operator++(int) {
        Counter old = *this;
        ++(*this);  // 调用前置版本
        return old;
    }
};

int main() {
    Counter c;
    c++;
    std::cout << c.value << "\n";  // 1
    return 0;
}
```

对于更复杂的类型（比如容器迭代器），后置版本除了调用 `++(*this)`，还要在开头保存一个旧迭代器的副本。这个副本的构造/析构就是后置比前置慢的根源。

另外，为了尽量贴近内置类型的行为，后置版本通常返回**常量值**：

```cpp
// flags: -O0 -g
#include <iostream>

struct Counter {
    int value = 0;

    Counter& operator++() {
        ++value;
        return *this;
    }

    const Counter operator++(int) {
        Counter old = *this;
        ++(*this);
        return old;
    }
};

int main() {
    Counter c;
    c++;
    std::cout << c.value << "\n";  // 1
    // c++++;  // 编译错误：后置版本返回常量值，和内置类型 i++++ 非法一致
    return 0;
}
```

这样 `c++++` 会编译失败，和内置类型 `i++++` 非法一致。虽然很多标准库实现没有加这个 `const`，但在自己的数学类型/迭代器里加上，可以减少误用。

## 问题 3：对于自定义类型，前置和后置的性能差距到底有多大？

我们用一个会打印拷贝构造的类来直观感受：

```cpp
// flags: -O0 -g
#include <iostream>

struct Expensive {
    int value;

    Expensive(int v = 0) : value(v) {
        std::cout << "construct " << value << "\n";
    }
    Expensive(const Expensive& other) : value(other.value) {
        std::cout << "copy " << value << "\n";
    }

    Expensive& operator++() {
        ++value;
        return *this;
    }

    Expensive operator++(int) {
        Expensive old = *this;  // 拷贝
        ++value;
        return old;             // 再次拷贝
    }
};

int main() {
    Expensive e(0);
    std::cout << "--- prefix ---\n";
    ++e;
    std::cout << "--- postfix ---\n";
    e++;
    return 0;
}
```

输出大致是：

```text
construct 0
--- prefix ---
--- postfix ---
copy 0
copy 1
```

后置版本多出了两次拷贝：一次是保存旧值，一次是按值返回。如果类型更大（比如一个 64 字节的矩阵迭代器），这个开销在循环里会被放大很多次。

所以工程上有条简单规则：**能用前置就用前置**，除非你真的需要旧值。

---

## 问题 4：迭代器场景下，这条规则为什么特别重要？

自定义迭代器是自增运算符最常见的应用场景之一。来看一个会打印拷贝次数的简化容器：

```cpp
// flags: -O0 -g
#include <iostream>

struct Container {
    int data[3] = {10, 20, 30};

    struct Iterator {
        int* p;
        Iterator(int* ptr) : p(ptr) {}
        Iterator(const Iterator& other) : p(other.p) {
            std::cout << "iterator copied\n";
        }

        int& operator*() const { return *p; }
        Iterator& operator++() { ++p; return *this; }
        Iterator operator++(int) {
            Iterator old = *this;  // 后置版本必须先拷贝旧值
            ++p;
            return old;
        }
        bool operator!=(const Iterator& other) const { return p != other.p; }
    };

    Iterator begin() { return Iterator(data); }
    Iterator end()   { return Iterator(data + 3); }
};

int main() {
    Container c;
    std::cout << "--- postfix loop ---\n";
    for (auto it = c.begin(); it != c.end(); it++) {
        std::cout << *it << " ";
    }
    std::cout << "\n--- prefix loop ---\n";
    for (auto it = c.begin(); it != c.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << "\n";
    return 0;
}
```

在 `for` 循环的递增表达式里，`it++` 每次都会触发一次迭代器拷贝；改成 `++it` 就能避免这次无意义的拷贝。如果迭代器只包含一个指针，副本开销很小；但如果迭代器需要维护更多状态（比如范围迭代器、带边界检查的迭代器），这个成本就不可忽略。

现代编译器有时能优化掉这个副本，但**不要依赖编译器**。写代码时养成前置自增的习惯，既表达了"我只关心自增后的状态"的意图，也避免了对复杂迭代器的性能损耗。

---

## 总结

- 前置自增 `++i` 的签名是 `T& operator++()`，返回自增后的对象引用，**无额外拷贝**。
- 后置自增 `i++` 的签名是 `T operator++(int)`，返回自增前的旧值，**至少多一次拷贝**。
- 后置版本的实现通常复用前置版本：`T old = *this; ++(*this); return old;`。
- 遍历容器或操作自定义类型时，**优先使用前置自增**，除非需要保留旧值参与表达式。

---

## 引擎映射

| 引擎 | 应用场景 | 具体体现 |
|------|---------|---------|
| **SelfGameEngine** | 自定义容器迭代器 | `Array<T>::Iterator`、`HashMap<K,V>::Iterator` 等实现 `operator++()` 与 `operator++(int)`；遍历时统一使用 `++it` 避免拷贝开销 |
| **UE** | `TArray`、`TMap` 迭代器 | UE 容器迭代器同样区分前置/后置；在性能敏感的热路径（如每帧遍历组件数组）中，代码规范推荐前置自增 |

> [!note] 关键取舍
> 内置类型的 `i++` 和 `++i` 在现代编译器下通常没有性能差异，所以很多人不以为意。但引擎里大量使用自定义迭代器，这些迭代器可能包含指针、边界信息、调试校验数据，一次不必要的拷贝就会破坏缓存友好性。前置自增是零成本的表达习惯。

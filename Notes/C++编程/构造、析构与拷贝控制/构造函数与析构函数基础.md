# 构造函数与析构函数基础

> [!info] 一句话概括
> **构造函数**负责让一个对象「诞生」并进入合法状态；**析构函数**负责在对象死亡时清理它持有的资源。两者合起来，是 C++ 手动内存管理不至于失控的第一道防线。

---

## 问题 0：为什么需要构造函数？没有它会怎样？

想象你要写一个封装动态字符串的类，内部用 `char*` 指向堆上的字符数组：

```cpp
class String {
    char* data_;
    size_t len_;
};
```

如果你直接声明一个 `String s;`，C++ 会做什么？

答案是：**它会为 `String` 对象分配内存，但不会帮你初始化里面的 `data_` 和 `len_`**。也就是说，`data_` 是一个指向未知地址的野指针，`len_` 是一个随机整数。此时如果你直接访问 `data_`，行为是未定义的。

这就是构造函数要解决的问题：在对象被创建出来的那一瞬间，就把它的成员变量放进一个**有意义的、安全的初始状态**。

> [!abstract]
> **构造函数（Constructor）**是一种特殊的成员函数，函数名和类名相同，没有返回值。它在创建对象时由编译器自动调用，用来初始化对象的状态。

---

## 问题 1：构造函数到底在做什么？

### 它不只"初始化变量"

很多人误以为构造函数只是"给成员变量赋初值"。对 `String` 来说，它的真正任务还包括**向操作系统申请外部资源**——一块能装下字符串内容的堆内存。

```cpp
// flags: -O0 -g
#include <iostream>
#include <cstring>
#include <cstddef>

class String {
    char* data_;
    size_t len_;
public:
    // 构造函数：把 C 风格字符串拷贝到堆上
    String(const char* str) {
        len_ = std::strlen(str);
        data_ = new char[len_ + 1];  // 申请堆内存
        std::strcpy(data_, str);     // 复制内容
    }

    void print() const {
        std::cout << data_ << "\n";
    }
};

int main() {
    String s("hello");
    s.print();  // 输出 hello
    return 0;
}
```

这里发生了三件事：
1. 编译器为 `String` 对象分配内存（`data_` + `len_` 的空间）。
2. 进入 `String::String(const char*)` 构造函数体。
3. 构造函数内部调用 `new char[...]`，在堆上申请另一块内存，并把字符串内容拷贝进去。

注意：**对象本身的内存和 `data_` 指向的内存是两回事**。前者在栈上（局部对象）或堆上（`new String(...)`），后者永远在堆上。

### 带参构造 vs 默认构造

上面的 `String` 只能这样创建：

```cpp
String s("hello");  // ✅
String t;           // ❌ 编译错误：没有无参构造函数
```

如果你也想让 `String s;` 合法，可以提供一个**默认构造函数**（default constructor）：

```cpp
class String {
    char* data_;
    size_t len_;
public:
    String() {
        len_ = 0;
        data_ = new char[1]{ '\0' };  // 空字符串也分配 1 字节，保持不变式
    }

    String(const char* str) {
        len_ = std::strlen(str);
        data_ = new char[len_ + 1];
        std::strcpy(data_, str);
    }
};
```

> [!abstract]
> **默认构造函数**是不带参数的构造函数。只要一个类没有用户自定义的构造函数，编译器通常会生成一个默认构造函数；但一旦你写了任何构造函数，编译器就不再自动生成，需要时你得自己写。

## 问题 2：析构函数是干嘛的？为什么 `char*` 成员需要 `delete[]`？

构造函数申请了堆内存，那么这块内存什么时候释放？

如果靠程序员手动记住每个对象并调用 `delete[] s.data_;`，那 C++ 就会变成第二个 C：到处都是资源泄漏和重复释放的 bug。

所以 C++ 提供了**析构函数（Destructor）**：对象离开作用域或被显式销毁时，析构函数会自动调用，用来清理对象持有的资源。

```cpp
class String {
    char* data_;
    size_t len_;
public:
    String(const char* str) {
        len_ = std::strlen(str);
        data_ = new char[len_ + 1];
        std::strcpy(data_, str);
    }

    ~String() {
        delete[] data_;  // 释放构造函数里申请的数组内存
    }
};

int main() {
    {
        String s("hello");
        s.print();
    }  // ← s 在这里离开作用域，自动调用 ~String()
    return 0;
}
```

> [!warning]
> `data_` 是用 `new char[...]` 申请的数组，所以必须用 `delete[]` 释放。如果用错成 `delete data_;`，行为未定义，常见后果是只释放数组第一个元素、破坏堆元数据，程序随机崩溃。

### 析构函数的关键语义

- 析构函数名是 `~类名`，没有参数，没有返回值。
- 一个类只能有一个析构函数（不能重载）。
- 对象销毁时自动调用：局部对象在作用域结束时、动态对象在 `delete` 时、成员对象在包含它的对象析构时。
- 析构函数**不应该抛出异常**。如果析构函数抛出异常且未被捕获，程序会调用 `std::terminate()` 直接终止。

## 问题 3：`new` / `delete` 和构造 / 析构是什么关系？

很多人把 `new` 和"调用构造函数"混为一谈。其实 `new` 做了两件事：

1. **分配内存**：在堆上申请一块足够放下对象的未初始化内存。
2. **调用构造函数**：在这块内存上执行构造函数，把对象初始化。

```cpp
String* p = new String("hello");
// 等价于：
// 1. void* raw = operator new(sizeof(String));  // 分配原始内存
// 2. String* p = static_cast<String*>(raw);
// 3. p->String("hello");                         // 构造（概念上）
```

对应的 `delete` 也做两件事：

1. **调用析构函数**：清理对象持有的资源。
2. **释放内存**：把对象占用的堆内存还给系统。

```cpp
delete p;
// 等价于：
// 1. p->~String();    // 析构
// 2. operator delete(p);  // 释放内存
```

> [!tip]
> 构造函数和析构函数本身**不分配/释放对象自己的内存**，它们只负责**初始化/清理对象的状态**。对象内存的分配与释放由 `new` / `delete`（或栈帧机制）负责。
>
> 不过，构造函数内部可以再次调用 `new` 去申请对象**外部持有的资源**（比如 `char*` 指向的堆内存），析构函数则负责释放这些资源。

## 问题 4：Day 1 的练习只需要关心哪些点？

回到练习题目：手写一个只含 `char*` 和 `size_t` 的 `String` 类。

你需要确保以下不变式（invariant）成立：

| 时机 | 对象状态 |
|------|---------|
| 构造完成后 | `data_` 指向一块有效堆内存，`len_` 等于字符串长度（不含 `\0`） |
| 使用期间 | `data_` 始终指向一块大小至少为 `len_ + 1` 的可读可写内存 |
| 析构前 | `data_` 指向的内存必须被 `delete[]` 释放，且只释放一次 |

这意味着你的构造函数必须处理：
- 传入 `nullptr` 或空字符串怎么办？
- 字符串长度如何计算？
- 需要多申请一个字节放结尾的 `\0`。

你的析构函数必须处理：
- `delete[] data_` 而不是 `delete data_`。
- 不要重复释放（目前暂时不用考虑拷贝和赋值，那是 Day 2/Day 3 的内容）。

## 总结

- **构造函数**让对象诞生时就有合法状态，常常负责申请外部资源。
- **析构函数**在对象死亡时自动清理资源，是 RAII 思想的起点。
- `new` = 分配对象内存 + 调用构造函数；`delete` = 调用析构函数 + 释放对象内存。
- 用 `new[]` 申请的数组必须用 `delete[]` 释放，否则是未定义行为。
- 写一个含 `char*` 的类，核心不变式是：构造后指针指向有效内存，析构时只释放一次。

---

## 引擎映射

| 引擎 | 应用场景 | 具体体现 |
|------|---------|---------|
| **SelfGameEngine** | 自定义字符串、动态数组等容器 | `String`、`Array` 等类型的构造负责向分配器申请内存，析构负责归还；这是引擎禁用标准库容器后的基础构件 |
| **UE** | `FString`、`TArray` 等资源句柄 | 构造函数根据内容长度申请字符数组，析构时释放；UE 使用自定义分配器替代全局 `new`/`delete`，但构造/析构的职责划分不变 |

> [!note] 关键取舍
> 工业级引擎通常不会直接使用 `new`/`delete`，而是通过自定义分配器（如 FrameArena、BinnedAllocator）管理内存。但**构造函数申请资源、析构函数释放资源**的职责划分始终不变。理解了这一对基础机制，才能进一步学习智能指针、自定义分配器和异常安全。

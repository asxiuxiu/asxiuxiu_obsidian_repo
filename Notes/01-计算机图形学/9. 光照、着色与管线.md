# 光照、着色与渲染管线

> [!info] 前置知识
> 请先阅读 [[01-计算机图形学/6. 光栅化续 (Antialiasing and Z-Buffering)|光栅化与 Z-Buffer]] 了解光栅化基础。

## Why：为什么需要光照和着色？

光栅化告诉我们"哪个像素被三角形覆盖"，但没有告诉我们**像素应该是什么颜色**。

**着色 (Shading)** = 计算物体表面颜色的过程

**光照 (Illumination)** = 模拟光线如何与物体交互

没有光照，我们只能看到物体的轮廓；有了光照，才能呈现立体感、材质感和真实感。

## What：光照模型

### 1. 光的类型

在图形学中，我们考虑三种基本光：

| 类型 | 说明 |
|-----|------|
| **高光 (Specular)** | 光滑表面的镜面反射，形成亮斑 |
| **漫反射 (Diffuse)** | 粗糙表面的均匀散射，决定基本颜色 |
| **环境光 (Ambient)** | 间接光照的近似，避免纯黑区域 |

![[Assets/graphics/light-types.png]]

### 2. Blinn-Phong 反射模型

这是实时渲染中最经典的局部光照模型。

#### 漫反射 (Diffuse Reflection)

**原理**：粗糙表面向各个方向均匀反射光线。

**兰伯特余弦定律 (Lambert's Cosine Law)**：

接收到的光能量与入射角余弦成正比：

$$
L_d = k_d \cdot (I / r^2) \cdot \max(0, \mathbf{n} \cdot \mathbf{l})
$$

其中：
- $k_d$：漫反射系数（物体颜色）
- $I$：光源强度
- $r$：到光源的距离
- $\mathbf{n}$：表面法线（单位向量）
- $\mathbf{l}$：指向光源的方向（单位向量）

> 点积 $\mathbf{n} \cdot \mathbf{l} = \cos\theta$，当光线垂直表面时最亮，平行时为零。

![[Assets/graphics/diffuse-reflection.png]]

#### 高光 (Specular Reflection)

**原理**：光滑表面会像镜子一样反射光线。

**Blinn-Phong 改进**：用**半程向量 (Halfway Vector)** 代替反射方向，计算更简便。

半程向量：

$$
\mathbf{h} = \frac{\mathbf{v} + \mathbf{l}}{||\mathbf{v} + \mathbf{l}||}
$$

其中 $\mathbf{v}$ 是视线方向，$\mathbf{l}$ 是光源方向。

**高光公式**：

$$
L_s = k_s \cdot (I / r^2) \cdot \max(0, \mathbf{n} \cdot \mathbf{h})^p
$$

其中：
- $k_s$：高光系数
- $p$： shininess（光泽度），越大高光越集中

> $p$ 通常取 100-200 模拟金属，10-50 模拟塑料。

![[Assets/graphics/specular-reflection.png]]

#### 环境光 (Ambient Light)

**原理**：真实世界中光线会经过多次反射到达物体，我们不计算全局光照，用一个常数近似：

$$
L_a = k_a \cdot I_a
$$

其中：
- $k_a$：环境光系数
- $I_a$：环境光强度

#### 完整 Blinn-Phong 模型

$$
L = L_a + L_d + L_s = k_a I_a + k_d (I/r^2)\max(0, \mathbf{n}\cdot\mathbf{l}) + k_s (I/r^2)\max(0, \mathbf{n}\cdot\mathbf{h})^p
$$

![[Assets/graphics/blinn-phong-components.png]]

### 3. 着色频率 (Shading Frequency)

决定在哪计算光照公式：

| 频率 | 计算位置 | 特点 |
|-----|---------|------|
| **Flat Shading** | 每个三角形一次 | 最快，有明显棱角 |
| **Gouraud Shading** | 每个顶点一次，内部插值 | 平衡，适合平滑表面 |
| **Phong Shading** | 每个像素一次 | 最慢，效果最好 |

![[Assets/graphics/shading-frequency.png]]

#### Flat Shading（面着色）

- 对每个三角形计算一次颜色
- 使用三角形的法线
- **缺点**：棱角分明，不光滑

#### Gouraud Shading（顶点着色）

- 对每个顶点计算颜色
- 三角形内部用**重心坐标**插值颜色
- **缺点**：高光可能丢失（如果高光中心不在顶点上）

#### Phong Shading（像素着色）

- 对每个像素计算光照
- 先插值得到像素法线，再用法线计算光照
- **优点**：最精确，高光效果好

> [!note] 现代 GPU
> 现代 GPU 足够强大，几乎总是使用 Phong Shading（像素着色）。

### 4. 法线相关

#### 计算逐顶点法线

对于光滑曲面，顶点的法线是周围三角形法线的加权平均：

$$
\mathbf{n}_{vertex} = \frac{\sum_i \mathbf{n}_i}{||\sum_i \mathbf{n}_i||}
$$

#### 法线变换

当对物体进行**非均匀缩放**时，直接变换法线会得到错误结果！

正确做法：用法线矩阵 $(M^{-1})^T$

```cpp
// 对于均匀缩放，可以直接用模型矩阵
// 对于非均匀缩放，需要特殊处理
normal = normalize((modelMatrix.inverse().transpose() * vec4(normal, 0)).xyz);
```

## What：图形渲染管线

### 管线的整体流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          图形渲染管线 (Graphics Pipeline)                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  应用阶段        几何阶段                         光栅化阶段                  │
│     │               │                                │                     │
│     ▼               ▼                                ▼                     │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐      │
│  │ 顶点数据 │ → │ 顶点着色 │ → │ 图元装配 │ → │ 光栅化  │ → │ 片段着色 │      │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘      │
│                    │                              │           │          │
│                    ▼                              ▼           ▼          │
│               MVP变换输出                    片元数据      像素输出        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 各阶段详解

#### 1. 顶点处理阶段

**输入**：顶点数据（位置、法线、纹理坐标、颜色等）

**顶点着色器 (Vertex Shader)**：
- 对每个顶点执行
- 主要任务：MVP 变换（Model-View-Projection）
- 输出：裁剪空间坐标

```glsl
// GLSL 顶点着色器示例
#version 330 core
layout(location = 0) in vec3 aPos;
layout(location = 1) in vec3 aNormal;

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

out vec3 FragPos;
out vec3 Normal;

void main() {
    FragPos = vec3(model * vec4(aPos, 1.0));
    Normal = mat3(transpose(inverse(model))) * aNormal;
    gl_Position = projection * view * vec4(FragPos, 1.0);
}
```

#### 2. 图元装配 (Primitive Assembly)

- 将顶点组装成三角形
- 进行裁剪（去掉视锥体外的部分）
- 透视除法，转换到 NDC (Normalized Device Coordinates)

#### 3. 光栅化 (Rasterization)

- 将三角形离散化为片段 (Fragments)
- 进行深度测试（Z-Test）
- 属性插值（颜色、深度、纹理坐标等）

#### 4. 片段处理阶段

**片段着色器 (Fragment Shader)**：
- 对每个片段（像素候选）执行
- 主要任务：计算最终颜色
- 可以访问纹理、执行光照计算等

```glsl
// GLSL 片段着色器示例 (Blinn-Phong)
#version 330 core
in vec3 FragPos;
in vec3 Normal;

uniform vec3 lightPos;
uniform vec3 viewPos;
uniform vec3 lightColor;
uniform vec3 objectColor;

out vec4 FragColor;

void main() {
    // 环境光
    float ambientStrength = 0.1;
    vec3 ambient = ambientStrength * lightColor;
    
    // 漫反射
    vec3 norm = normalize(Normal);
    vec3 lightDir = normalize(lightPos - FragPos);
    float diff = max(dot(norm, lightDir), 0.0);
    vec3 diffuse = diff * lightColor;
    
    // 高光
    float specularStrength = 0.5;
    vec3 viewDir = normalize(viewPos - FragPos);
    vec3 halfwayDir = normalize(lightDir + viewDir);
    float spec = pow(max(dot(norm, halfwayDir), 0.0), 32.0);
    vec3 specular = specularStrength * spec * lightColor;
    
    vec3 result = (ambient + diffuse + specular) * objectColor;
    FragColor = vec4(result, 1.0);
}
```

#### 5. 输出合并 (Output Merger)

- 深度测试 (Z-Test)
- 模板测试 (Stencil Test)
- 混合 (Blending，处理透明)
- 写入帧缓冲

### 可编程 vs 固定管线

| 阶段 | 类型 | 说明 |
|-----|------|------|
| 顶点着色器 | 可编程 | 必须实现 |
| 几何着色器 | 可编程 | 可选，可生成/删除图元 |
| 光栅化 | 固定 | 硬件实现 |
| 片段着色器 | 可编程 | 必须实现 |

## How：实践要点

### 完整渲染流程代码框架

```cpp
class Renderer {
    Shader shader;
    Camera camera;
    std::vector<Mesh> meshes;
    
public:
    void render() {
        // 1. 清除缓冲
        glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
        
        // 2. 使用着色器
        shader.use();
        
        // 3. 设置 uniforms (MVP 矩阵、光照参数等)
        shader.setMat4("projection", camera.getProjectionMatrix());
        shader.setMat4("view", camera.getViewMatrix());
        shader.setVec3("lightPos", light.position);
        shader.setVec3("viewPos", camera.position);
        
        // 4. 绘制每个物体
        for (auto& mesh : meshes) {
            shader.setMat4("model", mesh.getModelMatrix());
            mesh.draw();
        }
    }
};
```

### 性能优化要点

| 优化策略 | 说明 |
|---------|------|
| **减少 Draw Call** | 合批处理，使用纹理图集 |
| **LOD (Level of Detail)** | 远距离使用低模 |
| **视锥剔除** | 不渲染视锥体外的物体 |
| **背面剔除** | 不渲染背向相机的三角形 |
| **Early-Z** | 尽早进行深度测试，避免无效着色计算 |

## 总结

| 概念 | 核心要点 |
|-----|---------|
| **Blinn-Phong** | 环境光 + 漫反射 + 高光，经典局部光照模型 |
| **漫反射** | 与入射角余弦成正比，$\max(0, \mathbf{n}\cdot\mathbf{l})$ |
| **高光** | 用半程向量，$\max(0, \mathbf{n}\cdot\mathbf{h})^p$ |
| **着色频率** | Flat(面) < Gouraud(顶点) < Phong(像素) |
| **渲染管线** | 顶点处理 → 图元装配 → 光栅化 → 片段处理 |
| **顶点着色器** | 执行 MVP 变换，输出裁剪空间坐标 |
| **片段着色器** | 计算最终颜色，光照、纹理在此处理 |

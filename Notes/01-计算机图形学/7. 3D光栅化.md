# 光栅化基础

> [!info] 前置知识
> 请先阅读 [[01-计算机图形学/4. 变换续 Transformation Cont.|MVP 变换]] 了解投影变换。

---

## Why：什么是光栅化？

经过 **MVP 变换**后，所有物体都被变换到了标准立方体 $[-1, 1]^3$ 中。但显示器是 2D 的像素阵列。

**光栅化 (Rasterization)** = 把连续的几何图形（主要是三角形）**离散化**为屏幕上的像素

> "Raster" 来自德语，意为"屏幕"。光栅化即"画在屏幕上"。

---

## What：从标准立方体到屏幕

### 1. 定义视锥体 (Viewing Frustum)

在进行投影之前，需要定义相机能看到的空间范围：

| 参数 | 符号 | 说明 |
|-----|------|------|
| **垂直视场角** | $fovY$ | 垂直方向上的开放角度 |
| **宽高比** | $aspect$ | 屏幕宽度与高度的比例 |
| **近裁剪面** | $n$ | 能看到的最近距离 |
| **远裁剪面** | $f$ | 能看到的最远距离 |

**计算视锥体边界**：

$$
\tan(\frac{fovY}{2}) = \frac{top}{n} \implies top = n \cdot \tan(\frac{fovY}{2})
$$

$$
bottom = -top, \quad right = top \cdot aspect, \quad left = -right
$$

有了 $l, r, b, t, n, f$，就可以构建**投影矩阵**，将视锥体内的物体变换到标准立方体 $[-1, 1]^3$。

### 2. 视口变换 (Viewport Transformation)

**屏幕空间定义**：
- 像素坐标范围：$(0, 0)$ 到 $(width-1, height-1)$
- 像素 $(x, y)$ 的**几何中心**：$(x+0.5, y+0.5)$

![[Assets/graphics/20240310102749.png]]

**视口变换矩阵**：将 $[-1, 1]^2$ 映射到 $[0, width] \times [0, height]$

$$
M_{viewport} = \begin{pmatrix}
\frac{width}{2} & 0 & 0 & \frac{width}{2} \\
0 & \frac{height}{2} & 0 & \frac{height}{2} \\
0 & 0 & 1 & 0 \\
0 & 0 & 0 & 1
\end{pmatrix}
$$

> 深度 $z$ 通常会映射到 $[0, 1]$ 范围用于深度测试。

## What：三角形光栅化

### 为什么选择三角形？

三角形是图形学的基础图元，原因有三：

| 特性 | 说明 |
|-----|------|
| **平面性** | 三角形三个顶点一定共面，简化计算 |
| **内外明确** | 三角形内部/外部定义清晰，便于判断 |
| **插值便利** | 内部任意属性可通过顶点属性平滑插值 |

![[Assets/graphics/20240310111940.png]]

### 光栅化的核心任务

> **对于屏幕上的每个像素，判断它是否被三角形覆盖，如果覆盖，则计算其颜色。**

这是一个**采样**过程：将连续的几何定义离散化为一个个像素。

#### 朴素方法：遍历所有像素

```cpp
for (int x = 0; x < xmax; ++x)
    for (int y = 0; y < ymax; ++y)
        // 在像素中心采样
        image[x][y] = inside(triangle, x + 0.5, y + 0.5);
```

其中 `inside` 函数判断点 $(x+0.5, y+0.5)$ 是否在三角形内部。

#### 判断点是否在三角形内

利用**叉乘**的性质判断（详见 [[01-计算机图形学/2. 线性代数 Linear Algebra|线性代数]]）：

假设三角形按逆时针顺序定义为 $A, B, C$，对于点 $P$：

```
AB × AP  (z分量)
BC × BP  (z分量)  三者同号 → P在三角形内
CA × CP  (z分量)
```

![[Assets/graphics/20231228211128.png]]

> [!note] 边界问题
> 当点恰好落在三角形边界上时，图形 API 通常有统一规则（如"上闭下开"）来避免像素被重复绘制或漏画。

#### 优化：包围盒 (Bounding Box)

遍历全屏效率太低，只遍历三角形周围的矩形区域：

```cpp
// 计算包围盒
int xmin = floor(min(A.x, B.x, C.x));
int xmax = ceil(max(A.x, B.x, C.x));
int ymin = floor(min(A.y, B.y, C.y));
int ymax = ceil(max(A.y, B.y, C.y));

for (int x = xmin; x <= xmax; ++x)
    for (int y = ymin; y <= ymax; ++y)
        if (inside(triangle, x + 0.5, y + 0.5))
            image[x][y] = color;
```

![[Assets/graphics/20240310114035.png]]

### 属性插值：重心坐标

知道像素在三角形内还不够，还需要知道该像素的颜色、深度等信息。

**重心坐标 (Barycentric Coordinates)** 是解决三角形插值的关键。

对于三角形 $ABC$ 内的点 $P$，可以表示为：

$$
P = \alpha A + \beta B + \gamma C
$$

其中 $\alpha + \beta + \gamma = 1$，且 $\alpha, \beta, \gamma \geq 0$。

**几何意义**：
- $\alpha$ = 子三角形 $PBC$ 的面积 / 大三角形 $ABC$ 的面积
- $\beta$ = 子三角形 $PAC$ 的面积 / 大三角形 $ABC$ 的面积  
- $\gamma$ = 子三角形 $PAB$ 的面积 / 大三角形 $ABC$ 的面积

**属性插值**：

$$
V_P = \alpha V_A + \beta V_B + \gamma V_C
$$

其中 $V$ 可以是颜色、深度、法线、纹理坐标等任意属性。

> [!warning] 透视校正
> 对于透视投影，这种线性插值在屏幕空间是**不正确**的。正确的做法需要进行**透视校正插值**，这将在后续着色部分详细讨论。

## What：采样瑕疵与反走样

### 锯齿现象 (Jaggies / Aliasing)

简单的光栅化方法（对像素中心点采样）会导致**锯齿**：

![[Assets/graphics/20240310115603.png]]

**产生原因**：
- 采样率（屏幕分辨率）低于信号变化的频率（几何边缘的剧烈变化）
- 本质上是信号处理中的**混叠 (Aliasing)** 现象

**采样瑕疵的类型**：

| 类型 | 描述 |
|-----|------|
| **锯齿** (Jaggies) | 边缘的阶梯状效果 |
| **摩尔纹** (Moiré) | 高频模式与采样频率相近时产生的干涉纹样 |
| **车轮效应** | 时间采样不足，快速运动的物体看起来反向运动 |

## How：反走样 (Antialiasing)

### 核心思想

**反走样的关键**：在采样之前先进行模糊处理（滤波）！

正确的顺序：
```
先滤波（去除高频）→ 再采样
```

错误的顺序：
```
先采样 → 再模糊（无法消除已产生的混叠）
```

### 从频域理解

**傅里叶变换告诉我们**：任何信号都可以分解为不同频率的组合。

**走样的本质**：用相同的采样频率去采样不同频率的信号，结果可能相同（无法区分）。

**解决方法**：
1. **提高采样率** — 最直接，但受限于硬件
2. **先滤波（模糊）** — 去除高频信息，这是实际主流方案

### 实际实现方法

#### 1. MSAA (Multi-Sample Anti-Aliasing)

**多重采样抗锯齿**：
- 将一个像素划分成多个子像素（如 2×2 或 4×4）
- 计算每个子像素是否在三角形内
- 根据覆盖比例确定最终像素颜色

![[Assets/graphics/22-msaa-division.png]]

**关键点**：MSAA 只在**覆盖率计算**阶段使用多采样点，最终输出分辨率不变。

**工业界优化**：
- 使用不规则分布的采样点（而非均匀网格）
- 相邻像素复用采样点，减少计算量

#### 2. FXAA (Fast Approximate Anti-Aliasing)

**快速近似抗锯齿**：
1. 先渲染出带有锯齿的图像
2. 通过图像处理识别锯齿边缘
3. 用平滑的边缘替换锯齿边缘

**优点**：速度极快，后期处理
**缺点**：可能引入图像模糊

#### 3. TAA (Temporal Anti-Aliasing)

**时间性抗锯齿**：
- **核心思想**：复用上一帧的结果
- 在时间上分布采样点（每帧在像素内不同位置采样）
- 相当于把 MSAA 分布在时间维度上

**优点**：效率高，效果好
**注意**：运动物体需要额外处理

## 总结

| 概念 | 核心要点 |
|-----|---------|
| **光栅化** | 将连续几何离散化为像素的过程 |
| **三角形** | 最基础的图元，共面、内外明确、易插值 |
| **判断内外** | 利用叉乘符号一致性判断点是否在三角形内 |
| **重心坐标** | 用于三角形内部的属性插值 |
| **走样** | 采样率不足导致的高频信息混叠 |
| **反走样** | 先滤波（去除高频）再采样 |
| **MSAA** | 增加子采样点计算覆盖程度 |
| **TAA** | 在时间维度分布采样，复用历史帧 |

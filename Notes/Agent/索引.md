## 模块一：Agent Loop —— 从对话框到自主执行

### 核心问题

为什么 AI 从一个"对话框"变成了能自主执行多轮任务、并行调用工具的 Agent？这个质变的核心不是模型变聪明了，而是**外部套了一个循环**。不理解这个循环，就无法把 AI 真正嵌入引擎。

### 关键概念

- **Query Loop**：Claude Code 的核心无限循环，`AsyncGenerator` 模式，每次 `yield` 一个事件
- **Turn/Step 事件模型**：Kimi CLI 的 Wire 协议把对话抽象为标准事件流
- **StreamingToolExecutor**：在 API 还在流式返回时，提前执行已完成的工具调用
- **ReAct**：Reasoning + Acting 的交错循环

## 模块三：Context Management —— 记忆与压缩

### 核心问题

Agent 没有原生记忆，上下文窗口有限且昂贵。如果不主动管理，Agent 会"失忆"、"烧钱"、"幻觉"。

### 关键概念

- **三层压缩**：Snip 压缩 → 自动压缩 → 紧急压缩
- **微压缩**：文件被修改后，自动删除消息历史中该文件旧的读取结果
- **缓存断点（Prompt Caching）**：稳定内容只付 10% Token 费用
- **四层记忆架构**：Session Memory → Memdir → Magic Docs → Team Memory + Dream

## 模块四：Multi-Agent —— 并行与协作

## 模块六：Permission Model —— 边界与信任

### 核心问题

AI 能自动执行命令、修改文件。如何防止它误删项目、泄露密钥、做出不可回滚的破坏？这是你把它集成进引擎前的必修课。

### 关键概念

- **多级规则来源**：`userSettings` → `projectSettings` → `localSettings` → `flagSettings` → `policySettings` → `cliArg` → `command` → `session`
- **三级决策树**：Always Allow → Always Deny → Ask User
- **Bash 25 项安全检查**：元字符检测、危险命令拦截、环境变量劫持、迭代固定点算法
- **Hook 系统**：`PreToolUse` / `PostToolUse` 扩展点

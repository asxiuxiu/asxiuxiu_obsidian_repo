# 3 层神经网络的实现

[[Notes/深度学习入门/神经网络结构与激活函数|上一篇笔记]]讲到，神经网络通过非线性激活函数可以表示复杂的函数。但到这里有一个很自然的问题：**表示函数有什么用？**

答案是：**神经网络学习的是一个从输入到输出的映射，用来解决一个具体问题**。比如输入一张图片，输出它是不是猫；输入房屋面积，输出房价。把网络组装起来、从输入一步步算到输出的过程，就叫做**前向传播**（forward propagation）。

---

## 为什么神经网络能解决问题

这个回答可能还是让你觉得抽象。真正的问题是：**一个只会加权求和加激活函数的模型，凭什么能解决房价预测、图片分类这些复杂任务？**

### 机器学习问题 = 找一个输入到输出的函数

先看这些问题的本质：

- **房价预测**：想找函数 $f(\text{面积}, \text{卧室数}, \text{地段}) = \text{价格}$
- **手写数字识别**：想找函数 $f(\text{像素}_1, \dots, \text{像素}_{784}) = \text{数字}$

这些函数确实存在——对于每个输入，理论上都有一个正确答案。但我们不知道它们的精确公式。

### 我们有什么

我们不知道函数公式，但我们有**大量例子**：

- 这套房子 100㎡、3 室、在朝阳区，卖了 500 万
- 那张图片的像素是这样，标签是数字 7

机器学习做的事情，就是**从这些例子中自动推断出背后的函数**。

### 神经网络为什么适合

神经网络特别适合这件事，有两个原因：

1. **它非常灵活**。多层神经元叠加非线性激活函数，可以拟合出很复杂的弯曲关系，不只是直线。
2. **它可以从错误中学习**。一开始权重是随机的，预测很糟糕；但通过损失函数衡量误差，再用梯度下降调整权重，它会慢慢逼近正确答案。

所以「神经网络能逼近任意函数」真正的含义是：**只要现实问题可以被表示成输入到输出的连续映射，并且你有足够多的例子，神经网络就有可能学会这个映射。**

---

## 万事万物怎么变成数字

到这里你可能又会问：房子、图片、文字这些东西，神经网络怎么能处理呢？神经网络不是只接受数字吗？

没错。**神经网络看到的从来都不是「图片」「文字」「房子」，而是一个数字向量**。把现实世界的东西翻译成数字向量的过程，叫做**特征表示**（feature representation）。

### 房价：本来就是数字

房子的特征很容易量化：

```python
x = np.array([100.0, 3.0, 8.5, 10.0])
#            面积    卧室数  地段评分  房龄
```

### MNIST 图片：把像素展平

MNIST 图像是 28×28 的灰度图，每个像素是一个 0~255 的数。把它按行展平，就得到一个 784 维的向量：

```python
image = np.random.randint(0, 256, (28, 28))
x = image.flatten()   # 形状 (784,)
x = x / 255.0         # 归一化到 0~1
```

神经网络看到的不是「一个手写 7」，而是 784 个 0~1 之间的数字。它通过训练发现：某些像素组合亮起来时，答案更可能是 7。

### 文本：先变成 token，再变成向量

文字本身不是数字，需要先做转换。最简单的方法是 **one-hot 编码**：

假设词汇表只有 3 个词：「猫」「狗」「喜欢」。每个词对应一个 3 维向量：

- 猫 → `[1, 0, 0]`
- 狗 → `[0, 1, 0]`
- 喜欢 → `[0, 0, 1]`

实际应用中会用更复杂的 **词嵌入**（word embedding），但核心思想是一样的：把离散的符号变成连续的向量。

### 关键区别

不同问题有不同的数字化方式，这就是**特征工程**。传统机器学习中人类需要精心设计特征；深度学习的优势在于，对于图像、文本这类复杂数据，网络可以从原始数字中自己学到有用的特征表示。

> [!tip] 一句话总结
> 神经网络只认数字。人类先把现实世界的事物编码成向量，网络再在这些向量上找规律。

> [!info] 从 MNIST 到语言模型
> 这篇笔记只讲到最基础的 3 层全连接网络。现代 AI 的语言理解、代码生成能力，本质上也是「输入 → 输出」的函数学习，但用的是更复杂的网络结构（如 RNN、Transformer）和海量的训练数据。这些超出了本书的入门范围，但核心思想完全一样：把事物变成数字向量，让网络从数据中学出映射关系。

## 先明确要解决什么问题

在谈输出层之前，必须先说清楚神经网络通常解决哪两类问题。因为**输出层的结构完全取决于你要解决什么问题**。

### 回归问题：预测一个连续数值

如果输出是一个可以连续变化的数值，就是**回归问题**（regression）。

- 输入房屋信息 → 输出房价
- 输入历史气温 → 输出明天的温度
- 输入用户年龄、收入 → 输出预计消费金额

这类问题的输出层通常只有 **1 个节点**，激活函数用**恒等函数**（直接输出原始值）。

### 分类问题：预测一个类别

如果输出是从几个离散类别中选择一个，就是**分类问题**（classification）。

- 输入手写数字图片 → 输出是 0~9 中的哪一个
- 输入邮件文本 → 输出是垃圾邮件还是正常邮件
- 输入图片 → 输出是猫、狗还是鸟

这类问题的输出层有 **N 个节点**（N 是类别数），激活函数用 **softmax**，把输出变成概率分布。

> [!note] 关键区别
> 回归输出的是一个**数值**，分类输出的是一个**概率分布**。输出层的设计就是根据这个区别来选择的。

---

## 前向传播就是一层一层算

不管解决什么问题，网络内部的计算方式都是一样的。每一层可以分成两步：

1. 用输入和权重做矩阵乘法，加上偏置，得到加权输入：
   $$
   \mathbf{A}^{(l)} = \mathbf{X} \mathbf{W}^{(l)} + \mathbf{B}^{(l)}
   $$
2. 把加权输入传给激活函数，得到这一层的输出：
   $$
   \mathbf{Z}^{(l)} = h(\mathbf{A}^{(l)})
   $$

下一层再把 $\mathbf{Z}^{(l)}$ 当作输入继续算，直到输出层。

## 用 NumPy 实现 3 层网络

假设输入是 2 个节点，第 1 隐藏层是 3 个节点，第 2 隐藏层是 2 个节点，输出层是 2 个节点：

```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def identity_function(x):
    return x

def init_network():
    network = {}
    network['W1'] = np.array([[0.1, 0.3, 0.5], [0.2, 0.4, 0.6]])
    network['b1'] = np.array([0.1, 0.2, 0.3])
    network['W2'] = np.array([[0.1, 0.4], [0.2, 0.5], [0.3, 0.6]])
    network['b2'] = np.array([0.1, 0.2])
    network['W3'] = np.array([[0.1, 0.3], [0.2, 0.4]])
    network['b3'] = np.array([0.1, 0.2])
    return network

def forward(network, x):
    W1, W2, W3 = network['W1'], network['W2'], network['W3']
    b1, b2, b3 = network['b1'], network['b2'], network['b3']

    a1 = np.dot(x, W1) + b1
    z1 = sigmoid(a1)
    a2 = np.dot(z1, W2) + b2
    z2 = sigmoid(a2)
    a3 = np.dot(z2, W3) + b3
    y = identity_function(a3)

    return y

network = init_network()
x = np.array([1.0, 0.5])
y = forward(network, x)
print(y)  # 输出两个数值
```

这里的权重和偏置是人为设定的，只是为了演示前向传播的计算流程。后面的笔记会讲这些数字是怎么通过训练自动学到的。

`identity_function` 是**恒等函数**，直接返回输入。它常用于回归问题的输出层。

---

## 输出层的设计：由问题类型决定

前面代码的输出层用了恒等函数，对应的是回归问题。如果换成分类问题，就要用 softmax。

### 回归问题：恒等函数

恒等函数什么都不做，把加权输入直接输出：

```python
def identity_function(x):
    return x
```

适合输出一个连续值，比如房价、温度。

### 分类问题：softmax 函数

softmax 把输出层的任意实数变成概率，所有输出之和为 1：

$$
y_k = \frac{\exp(a_k)}{\sum_{i=1}^{n} \exp(a_i)}
$$

```python
import numpy as np

def softmax(a):
    c = np.max(a)  # 防止指数爆炸
    exp_a = np.exp(a - c)
    sum_exp_a = np.sum(exp_a)
    return exp_a / sum_exp_a

a = np.array([0.3, 2.9, 4.0])
print(softmax(a))  # [0.018 0.245 0.737]
print(np.sum(softmax(a)))  # 1.0
```

代码里先减去 `np.max(a)` 是为了防止 $\exp$ 对大的数溢出。这只是一个数值稳定性技巧，不影响最终概率。

> [!tip] 一句话总结
> 输出层没有固定答案：想预测数值就用恒等函数，想做分类就用 softmax。

## 批处理：一次推理多张图片

实际应用时，我们不会一张一张地输入图片，而是把多张图片打包成一个矩阵一起算。这种打包好的数据叫一个 **batch**（批次）。

比如 MNIST 数据集中，一张图片是 784 维向量。如果一次处理 100 张，输入就是一个 `(100, 784)` 的矩阵：

```python
import numpy as np

# 假设 network 已经初始化，输入 x 是 (100, 784) 的 batch
x = np.random.rand(100, 784)

W1 = network['W1']  # (784, 50)
b1 = network['b1']  # (50,)
W2 = network['W2']  # (50, 100)
b2 = network['b2']  # (100,)
W3 = network['W3']  # (100, 10)
b3 = network['b3']  # (10,)

a1 = np.dot(x, W1) + b1      # (100, 50)
z1 = sigmoid(a1)
a2 = np.dot(z1, W2) + b2     # (100, 100)
z2 = sigmoid(a2)
a3 = np.dot(z2, W3) + b3     # (100, 10)
y = softmax(a3)              # 每行是一个样本的概率分布
```

通过广播，`b1` 会自动加到 `a1` 的每一行上。输出 `y` 的形状是 `(100, 10)`，每一行对应一张图片属于 0~9 的概率。

## 对 MNIST 进行推理

到这里你可能还会问：前面一直在讲数学公式和代码，怎么突然就能识别手写数字了？这里还差一个关键概念：**训练 vs 推理**。

- **训练（Training）**：给网络看很多「输入 + 正确答案」，让它自动调整权重，使得输出越来越接近正确答案。
- **推理（Inference）**：训练完成后，用调好的权重对新的输入做预测。

本书这一章为了先讲清楚「网络是怎么把输入算成输出的」，暂时把权重写死了。真正的权重是怎么学出来的，会在[[Notes/深度学习入门/损失函数与数值微分|下一篇笔记]]里讲。

把以上代码组合起来，就可以对 MNIST 手写数字进行推理：

1. 加载 MNIST 数据集和预训练权重（通常是 `sample_weight.pkl`）。
2. 把每张图片的像素值归一化到 0~1 之间。
3. 用 3 层网络做前向传播，输出 softmax 概率。
4. 取概率最大的类别作为预测结果。

```python
import numpy as np

def predict(network, x):
    W1, W2, W3 = network['W1'], network['W2'], network['W3']
    b1, b2, b3 = network['b1'], network['b2'], network['b3']

    a1 = np.dot(x, W1) + b1
    z1 = sigmoid(a1)
    a2 = np.dot(z1, W2) + b2
    z2 = sigmoid(a2)
    a3 = np.dot(z2, W3) + b3
    y = softmax(a3)
    return y
```

> 注意：上面的代码需要配合 MNIST 数据集和预训练权重才能实际运行。这里的关键是理解前向传播的流程。

## 小结

- 现实世界的问题往往可以表示成「输入 → 输出」的函数，但我们通常不知道这个函数的精确公式。
- 神经网络的价值在于：**只要问题能被表示成连续映射，并且有足够多的例子，它就有可能自动学会这个映射**。
- 神经网络只接受数字向量，所以现实事物（图片、文字、房屋信息）必须先通过**特征表示**转换成向量。
- 神经网络学习的是一个从输入到输出的映射，用来解决**回归**或**分类**问题。
- **前向传播**就是从输入层逐层计算到输出层的过程。
- 每一层都是 $\mathbf{A} = \mathbf{XW} + \mathbf{B}$ 加上激活函数。
- 输出层根据任务选择函数：**回归用恒等函数，分类用 softmax**。
- **批处理**能一次性推理多个样本，利用矩阵运算提高效率。
- **训练**是学权重的过程，**推理**是用学好的权重做预测。
- 3 层神经网络的前向传播代码是后面实现训练循环的基础。

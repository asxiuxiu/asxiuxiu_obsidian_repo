# 反向传播的实现

数值微分虽然能算出梯度，但神经网络稍微大一点，它就要对每个参数做两次前向计算，速度非常慢。上一篇笔记用计算图解释了链式法则：如果把网络拆成一层一层的小模块，每一层只关心自己的局部导数，就能从输出端一路把梯度传回输入端。这篇笔记就来把这些小模块实现出来，拼成一个完整的、用反向传播训练的神经网络。

---

## 从计算图到“层”

计算图里的每一个节点，本质上都是一个函数。反向传播时，我们需要知道两件事：

1. **这个节点做了什么变换**（前向传播）。
2. **它的输出对输入的局部导数是什么**（反向传播）。

把这两点封装成一个 Python 类，就是书中所说的“层”。每个层都有 `forward` 和 `backward` 两个方法：

- `forward(x)` 接收输入，返回输出，同时把反向传播需要的中间结果保存下来。
- `backward(dout)` 接收上游传来的梯度，返回传给下游的梯度。

```python
import numpy as np


class Relu:
    def __init__(self):
        self.mask = None

    def forward(self, x):
        self.mask = (x <= 0)
        out = x.copy()
        out[self.mask] = 0
        return out

    def backward(self, dout):
        dout[self.mask] = 0
        return dout
```

`Relu` 层的前向传播很简单：大于 0 的原样输出，小于等于 0 的输出 0。反向传播时，正数部分的梯度原样传下去，负数部分的梯度被截断为 0。`self.mask` 就是用来记住“哪些位置被截断了”的布尔掩码。

## Sigmoid 层

Sigmoid 函数是：

$$
y = \frac{1}{1 + e^{-x}}
$$

它的导数有一个很漂亮的性质：

$$
\frac{\partial y}{\partial x} = y(1 - y)
$$

所以 Sigmoid 层在反向传播时，只要保存前向传播的输出 `y`，就能直接算出梯度，不需要再重新计算指数函数。

```python
class Sigmoid:
    def __init__(self):
        self.out = None

    def forward(self, x):
        out = 1 / (1 + np.exp(-x))
        self.out = out
        return out

    def backward(self, dout):
        return dout * self.out * (1.0 - self.out)
```

## Affine 层：矩阵乘法的反向传播

神经网络里真正的“权重层”通常是 $Y = X W + b$。这就是 `Affine` 层（仿射变换层）。

前向传播很直观：输入 `x` 与权重 `W` 做矩阵乘法，再加上偏置 `b`。反向传播需要分别求出传给输入端的梯度 `dx`、传给权重的梯度 `dW`、传给偏置的梯度 `db`。

如果设 $Y = X W + b$，损失对 $Y$ 的梯度是 $dout$，那么：

$$
dX = dout \cdot W^{\mathsf{T}}
$$

$$
dW = X^{\mathsf{T}} \cdot dout
$$

$$
db = \sum_i dout_i
$$

```python
class Affine:
    def __init__(self, W, b):
        self.W = W
        self.b = b
        self.x = None
        self.dW = None
        self.db = None

    def forward(self, x):
        self.x = x
        return np.dot(x, self.W) + self.b

    def backward(self, dout):
        dx = np.dot(dout, self.W.T)
        self.dW = np.dot(self.x.T, dout)
        self.db = np.sum(dout, axis=0)
        return dx
```

注意 `dW` 和 `db` 保存在层对象内部，因为训练时要用它们去更新 `self.W` 和 `self.b`。

## Softmax-with-Loss 层

最后一层比较特殊：它把 softmax 和交叉熵误差合并在一起。为什么要合并？因为 softmax 的导数和交叉熵误差的导数刚好能互相抵消，结果变得非常简单。

Softmax 输出概率：

$$
y_k = \frac{e^{a_k}}{\sum_i e^{a_i}}
$$

交叉熵误差：

$$
L = -\sum_k t_k \log y_k
$$

当标签 $t$ 是 one-hot 形式时，损失对 softmax 输入 $a$ 的梯度就是：

$$
\frac{\partial L}{\partial a} = y - t
$$

这意味着：正确类别的输出比 1 小多少，梯度就往那个方向拉多少。这个形式非常优雅，也是把 softmax 和 loss 放在一起实现的主要原因。

```python
def softmax(x):
    if x.ndim == 2:
        x = x - x.max(axis=1, keepdims=True)
        x = np.exp(x)
        x /= x.sum(axis=1, keepdims=True)
    elif x.ndim == 1:
        x = x - np.max(x)
        x = np.exp(x) / np.sum(np.exp(x))
    return x


def cross_entropy_error(y, t):
    if y.ndim == 1:
        t = t.reshape(1, t.size)
        y = y.reshape(1, y.size)
    if t.size == y.size:
        t = t.argmax(axis=1)
    batch_size = y.shape[0]
    return -np.sum(np.log(y[np.arange(batch_size), t] + 1e-7)) / batch_size


class SoftmaxWithLoss:
    def __init__(self):
        self.loss = None
        self.y = None
        self.t = None

    def forward(self, x, t):
        self.t = t
        self.y = softmax(x)
        self.loss = cross_entropy_error(self.y, self.t)
        return self.loss

    def backward(self, dout=1):
        batch_size = self.t.shape[0]
        if self.t.size == self.y.size:  # one-hot 标签
            dx = (self.y - self.t) / batch_size
        else:  # 索引标签
            dx = self.y.copy()
            dx[np.arange(batch_size), self.t] -= 1
            dx /= batch_size
        return dx
```

除以 `batch_size` 是因为损失是 mini-batch 的平均，梯度也要对应平均。

## 组装成 TwoLayerNet

有了这些层，就可以像搭积木一样拼出两层的神经网络。和上一版用数值微分的 `TwoLayerNet` 相比，这次只是把内部结构换成了层对象。

```python
class TwoLayerNet:
    def __init__(self, input_size, hidden_size, output_size,
                 weight_init_std=0.01):
        self.params = {}
        self.params['W1'] = weight_init_std * np.random.randn(input_size, hidden_size)
        self.params['b1'] = np.zeros(hidden_size)
        self.params['W2'] = weight_init_std * np.random.randn(hidden_size, output_size)
        self.params['b2'] = np.zeros(output_size)

        self.layers = {}
        self.layers['Affine1'] = Affine(self.params['W1'], self.params['b1'])
        self.layers['Relu1'] = Relu()
        self.layers['Affine2'] = Affine(self.params['W2'], self.params['b2'])
        self.lastLayer = SoftmaxWithLoss()

    def predict(self, x):
        for layer in self.layers.values():
            x = layer.forward(x)
        return x

    def loss(self, x, t):
        y = self.predict(x)
        return self.lastLayer.forward(y, t)

    def accuracy(self, x, t):
        y = self.predict(x)
        y = np.argmax(y, axis=1)
        if t.ndim != 1:
            t = np.argmax(t, axis=1)
        return np.sum(y == t) / float(x.shape[0])
```

`predict` 和 `loss` 只做前向传播，`gradient` 才做反向传播。

```python
    def gradient(self, x, t):
        # forward
        self.loss(x, t)

        # backward
        dout = 1
        dout = self.lastLayer.backward(dout)
        layers = list(self.layers.values())
        layers.reverse()
        for layer in layers:
            dout = layer.backward(dout)

        grads = {}
        grads['W1'] = self.layers['Affine1'].dW
        grads['b1'] = self.layers['Affine1'].db
        grads['W2'] = self.layers['Affine2'].dW
        grads['b2'] = self.layers['Affine2'].db
        return grads

    def numerical_gradient(self, x, t):
        loss_W = lambda W: self.loss(x, t)
        grads = {}
        grads['W1'] = numerical_gradient(loss_W, self.params['W1'])
        grads['b1'] = numerical_gradient(loss_W, self.params['b1'])
        grads['W2'] = numerical_gradient(loss_W, self.params['W2'])
        grads['b2'] = numerical_gradient(loss_W, self.params['b2'])
        return grads
```

反向传播的流程是从后往前：先调用 `SoftmaxWithLoss.backward`，再把梯度依次传给 `Affine2`、`Relu1`、`Affine1`。每层的 `backward` 会自动把自己需要的 `dW`、`db` 算好并保存下来。

## 梯度确认

反向传播的实现容易出错，所以书中推荐做一个“梯度确认”：把反向传播算出来的梯度和数值微分算出来的梯度做对比，如果两者非常接近，就说明实现大概率是对的。

```python
def numerical_gradient(f, x):
    h = 1e-4
    grad = np.zeros_like(x)
    it = np.nditer(x, flags=['multi_index'], op_flags=['readwrite'])
    while not it.finished:
        idx = it.multi_index
        tmp_val = x[idx]
        x[idx] = tmp_val + h
        fxh1 = f(x)
        x[idx] = tmp_val - h
        fxh2 = f(x)
        grad[idx] = (fxh1 - fxh2) / (2 * h)
        x[idx] = tmp_val
        it.iternext()
    return grad


np.random.seed(42)
net = TwoLayerNet(input_size=4, hidden_size=5, output_size=3)
x = np.random.randn(3, 4)
t = np.array([0, 1, 2])

grad_backprop = net.gradient(x, t)
grad_numerical = net.numerical_gradient(x, t)

for key in grad_backprop.keys():
    diff = np.average(np.abs(grad_backprop[key] - grad_numerical[key]))
    print(f"{key}: diff = {diff:.2e}")
```

通常差值在 $10^{-6}$ 或更小就可以认为是通过的。

## 用反向传播训练

一旦确认梯度正确，训练循环就可以换成更快的反向传播版本。和数值微分版相比，唯一变化就是 `net.gradient(x_batch, t_batch)` 不再是逐个参数算差分，而是走一遍反向传播。

```python
# 一个自包含的玩具数据集：2 维输入，3 个类别
np.random.seed(0)
n_samples = 120
x = np.random.randn(n_samples, 2)
# 根据角度把样本分成 3 类
theta = np.arctan2(x[:, 1], x[:, 0])
t = np.zeros(n_samples, dtype=int)
t[(theta > np.pi / 3) & (theta <= np.pi)] = 1
t[(theta > -np.pi) & (theta <= -np.pi / 3)] = 2

# 划分训练集 / 测试集
split = int(n_samples * 0.8)
x_train, x_test = x[:split], x[split:]
t_train, t_test = t[:split], t[split:]

net = TwoLayerNet(input_size=2, hidden_size=6, output_size=3)
iters_num = 1000
learning_rate = 0.1
batch_size = 20

for i in range(iters_num):
    batch_mask = np.random.choice(x_train.shape[0], batch_size)
    x_batch = x_train[batch_mask]
    t_batch = t_train[batch_mask]

    grad = net.gradient(x_batch, t_batch)
    for key in ('W1', 'b1', 'W2', 'b2'):
        net.params[key] -= learning_rate * grad[key]

    if i % 100 == 0:
        loss = net.loss(x_batch, t_batch)
        train_acc = net.accuracy(x_train, t_train)
        test_acc = net.accuracy(x_test, t_test)
        print(f"iter {i}: loss={loss:.4f}, train_acc={train_acc:.4f}, test_acc={test_acc:.4f}")
```

> 注意：这个例子只是为了演示训练循环可以跑通，不需要额外下载 MNIST 数据集。如果想复现书中完整的 MNIST 实验，只需要把 `x_train`、`t_train` 等换成 MNIST 数据即可。

## 为什么反向传播快

数值微分对一个有 $n$ 个参数的网络，每轮要执行约 $2n$ 次前向传播。反向传播每轮只需要一次前向加一次反向，时间复杂度从 $O(n)$ 降到 $O(\text{网络深度})$。

更直观地说：数值微分是在“ perturb 每个参数看损失怎么变”，而反向传播是在“让损失自己告诉我们每个参数该负多大责任”。链式法则让这条信息可以从后往前高效传递。

---

## 小结

- 每个层只要实现 `forward` 和 `backward`，就能拼成完整网络。
- ReLU 的反向传播用掩码把负数位置置 0；Sigmoid 利用 $y(1-y)$ 简化导数。
- Affine 层要同时计算 `dx`、`dW`、`db`，其中 `dW` 和 `db` 保存在层里供参数更新使用。
- Softmax 与交叉熵合并后，输出层梯度简化为 $y - t$。
- 实现完反向传播后，要用数值微分做梯度确认，确保两者差异足够小。
- 确认无误后，训练循环里的梯度计算可以从数值微分切换到反向传播，速度大幅提升。

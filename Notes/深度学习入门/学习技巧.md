# 学习技巧

训练神经网络不只是“算梯度、更新参数”那么简单。现实中，损失函数的表面往往像一个起伏很大的山谷：有的地方很平缓，梯度几乎为零；有的地方很陡峭，步子稍大就跳过头；还有的地方像个马鞍，某些方向往上、某些方向往下。为了让训练更稳定、更快收敛，人们总结出一套常用的学习技巧。这篇笔记按书中的顺序，介绍优化器、权重初始化、Batch Normalization、正则化和超参数验证。

---

## 优化器：参数怎么更新

优化器（optimizer）决定的是：拿到梯度之后，以什么样的规则去更新参数。最简单的规则是 **SGD（随机梯度下降）**：每次沿着梯度的反方向走一小步。

$$
W \leftarrow W - \eta \frac{\partial L}{\partial W}
$$

```python
class SGD:
    def __init__(self, lr=0.01):
        self.lr = lr

    def update(self, params, grads):
        for key in params.keys():
            params[key] -= self.lr * grads[key]
```

SGD 简单，但问题也很明显：如果损失表面在某个方向很平缓、另一个方向很陡峭，它会在陡峭方向来回震荡，在平缓方向几乎不动。这就像一个人下山时，沿着陡坡反复横跳，却迟迟走不到谷底。

---

## Momentum：给优化加上惯性

Momentum 的想法是给参数更新加上“速度”：像小球滚下山一样，累积之前的运动方向。即使当前梯度很小，之前的速度仍然能推着参数继续前进。

$$
v \leftarrow \alpha v - \eta \frac{\partial L}{\partial W}
$$

$$
W \leftarrow W + v
$$

```python
class Momentum:
    def __init__(self, lr=0.01, momentum=0.9):
        self.lr = lr
        self.momentum = momentum
        self.v = None

    def update(self, params, grads):
        if self.v is None:
            self.v = {}
            for key, val in params.items():
                self.v[key] = np.zeros_like(val)
        for key in params.keys():
            self.v[key] = self.momentum * self.v[key] - self.lr * grads[key]
            params[key] += self.v[key]
```

`momentum=0.9` 表示保留 90% 的旧速度。惯性让优化在一致的方向上加速，在震荡的方向上相互抵消。

---

## AdaGrad：按参数调整学习率

不同参数更新的频率和量级往往不同。AdaGrad 的思路是：给那些 historically 梯度很大的参数更小的学习率，给梯度很小的参数更大的学习率。每个参数“自定义”学习步长。

$$
h \leftarrow h + \left(\frac{\partial L}{\partial W}\right)^2
$$

$$
W \leftarrow W - \eta \frac{1}{\sqrt{h}} \frac{\partial L}{\partial W}
$$

```python
class AdaGrad:
    def __init__(self, lr=0.01):
        self.lr = lr
        self.h = None

    def update(self, params, grads):
        if self.h is None:
            self.h = {}
            for key, val in params.items():
                self.h[key] = np.zeros_like(val)
        for key in params.keys():
            self.h[key] += grads[key] * grads[key]
            params[key] -= self.lr * grads[key] / (np.sqrt(self.h[key]) + 1e-7)
```

`h` 会不断累积梯度平方，所以学习率会越变越小。这适合凸问题，但在深度网络中可能过早收敛。后续很多改进（如 RMSprop、Adam）都是从这个思想出发的。

---

## Adam：动量 + 自适应学习率

Adam 把 Momentum 和 AdaGrad 结合起来：既保留速度，又按参数调整学习率。它是深度学习中最常用的默认优化器之一。

```python
class Adam:
    def __init__(self, lr=0.001, beta1=0.9, beta2=0.999):
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.iter = 0
        self.m = None
        self.v = None

    def update(self, params, grads):
        if self.m is None:
            self.m, self.v = {}, {}
            for key, val in params.items():
                self.m[key] = np.zeros_like(val)
                self.v[key] = np.zeros_like(val)
        self.iter += 1
        lr_t = self.lr * np.sqrt(1.0 - self.beta2 ** self.iter) \
               / (1.0 - self.beta1 ** self.iter)
        for key in params.keys():
            self.m[key] += (1 - self.beta1) * (grads[key] - self.m[key])
            self.v[key] += (1 - self.beta2) * (grads[key] ** 2 - self.v[key])
            params[key] -= lr_t * self.m[key] / (np.sqrt(self.v[key]) + 1e-7)
```

`m` 是一阶矩估计（动量），`v` 是二阶矩估计（梯度平方的滑动平均）。开头的 `lr_t` 是偏差修正，防止训练初期估计值偏向 0。

## 权重初始化：起点很重要

神经网络训练之前，权重必须被初始化。如果初始化得不好，激活值可能在深层网络中逐层爆炸或消失，导致训练根本跑不动。

**Xavier 初始化**（Glorot 初始化）假设激活函数关于 0 对称，推荐标准差为：

$$
\sigma = \frac{1}{\sqrt{n_{in}}}
$$

**He 初始化**则针对 ReLU 这种非对称激活函数，标准差为：

$$
\sigma = \sqrt{\frac{2}{n_{in}}}
$$

```python
def xavier_init(input_size, output_size):
    return np.random.randn(input_size, output_size) / np.sqrt(input_size)


def he_init(input_size, output_size):
    return np.random.randn(input_size, output_size) * np.sqrt(2.0 / input_size)
```

用 Xavier 配 sigmoid/tanh，用 He 配 ReLU，这是实践中比较常见的搭配。

## Batch Normalization：让中间层更稳定

Batch Normalization（批归一化）不是直接调学习率或初始化，而是对每一层的输出做归一化：让每个 mini-batch 的激活值均值为 0、方差为 1。这样无论前面的权重怎么变，后面层接收到的输入分布都不会剧烈漂移。

书中提到的核心效果：

- 允许使用更大的学习率，训练更快。
- 减少对初始化的敏感程度。
- 起到轻微的正则化效果。

实现时，Batch Normalization 层在训练阶段用当前 batch 的均值和方差，在测试阶段用训练时保存的移动平均。书中把这一层当成一个黑盒技巧来介绍，重点是知道它解决的是“内部协变量偏移”问题。

---

## 权值衰减：给大权重打 penalty

权值衰减（weight decay）也叫 L2 正则化。它在损失函数里加上权重的平方和：

$$
L' = L + \frac{1}{2} \lambda W^2
$$

更新时等价于：

$$
W \leftarrow W - \eta \frac{\partial L}{\partial W} - \eta \lambda W
$$

这样大的权重会被持续拉向 0，防止模型对训练数据“死记硬背”，从而降低过拟合风险。偏置通常不加权值衰减。

---

## Dropout：随机关掉一部分神经元

Dropout 是另一种防止过拟合的技巧。训练时，以一定概率（比如 0.5）随机把某些神经元的输出置 0。这样网络不能依赖某几个神经元，被迫学习更鲁棒的特征。

测试时则把所有神经元都打开，但要按丢弃比例缩放输出，保证期望值一致。

```python
class Dropout:
    def __init__(self, dropout_ratio=0.5):
        self.dropout_ratio = dropout_ratio
        self.mask = None

    def forward(self, x, train_flg=True):
        if train_flg:
            self.mask = np.random.rand(*x.shape) > self.dropout_ratio
            return x * self.mask
        else:
            return x * (1.0 - self.dropout_ratio)

    def backward(self, dout):
        return dout * self.mask
```

Dropout 只在训练时用，测试时不用。它和权值衰减都是让模型不要过度自信地拟合训练集。

---

## 超参数验证：别在测试集上调参

神经网络里需要手动选择的量很多：学习率、隐藏层大小、权重衰减强度、dropout 比例、优化器参数等，这些都叫**超参数**（hyperparameter）。

调参时容易犯的一个错误是：在测试集上反复试，然后挑表现最好的。这其实等于把测试集也当成了训练集，最终评估会虚高。

正确的做法是：

1. 把数据分成训练集、验证集、测试集。
2. 在训练集上训练，在验证集上挑超参数。
3. 最后只在测试集上评估一次，作为最终成绩。

如果数据不多，也可以用交叉验证：把训练集分成 k 份，轮流拿一份当验证集，取平均表现。

---

## 小结

- 优化器决定“怎么更新参数”：SGD 最基础；Momentum 加惯性；AdaGrad 按参数调学习率；Adam 两者结合，是最常用的默认选择。
- 权重初始化决定“起点好不好”：Xavier 适合 sigmoid/tanh，He 适合 ReLU。
- Batch Normalization 让每层输入分布稳定，允许更大学习率。
- 权值衰减和 Dropout 都是防止过拟合：一个惩罚大权重，一个随机禁用神经元。
- 超参数要在验证集上挑选，测试集只用于最终评估。

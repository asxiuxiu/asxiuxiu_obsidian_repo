# SelfGameEngine 构建手册

> 本目录存放从源码分析中提炼出的**通用游戏引擎知识**。所有内容完全脱敏，可直接用于个人项目或 vibe coding。
>
> **核心设计理念**：这是一个为 **AI 协作开发** 而设计的 ECS 引擎知识库。每篇笔记在讨论具体模块时，默认回扣两个问题：**AI 如何观察它？** **AI 如何操作它？**

---

## 引擎构建总览

游戏引擎体量庞大，本手册将其拆分为 **6 个阶段、15 个核心模块、约 40 个可执行子任务**。每个阶段结束都有一个**可编译运行的里程碑**。建议你按顺序逐个击破，不要在没有完成前置模块时直接跳入深层专题。

```
阶段 0：设计原则确立
    │
    ▼
阶段 1：最小引擎骨架可编译运行
    │
    ├──► 阶段 2：基础工具就位（可并行补充）
    │
    ▼
阶段 3：核心运行时闭环
    │
    ├──► 阶段 4：第一帧画面渲染
    │
    ▼
阶段 5：工具链与 AI 桥接
    │
    ▼
阶段 6：工业级专题增强
```

---

## 阶段 0：设计原则确立

**里程碑**：确定了 ECS + AI 友好 + 确定性的元规则。所有后续设计决策有统一判据。

**验收标准**：
- [x] 能回答"为什么这个引擎选择 ECS 而非 OOP"
- [x] 能列出 AI 友好架构的 5 条设计红线
- [x] 所有后续模块的设计决策都能回扣到本阶段的原则

### 0.1 AI 友好的引擎架构

| 子任务 | 状态 | 产出物 |
|--------|------|--------|
| 0.1.1 为什么传统引擎对 AI 不友好 | ⏳ | 问题场景分析 |
| 0.1.2 ECS + 反射 + 确定性的三元组 | ⏳ | 核心设计公式 |
| 0.1.3 AI 可观测性的实现标准 | ⏳ | 状态平铺、自描述、工具边界 checklist |
| 0.1.4 多 Agent 编排的架构预设 | ⏳ | Orchestrator 角色与沙箱原则 |

> **前置**：无
> **下一步**：[[从零开始的引擎骨架]] — 在原则确立后，立刻让代码跑起来验证思路。

## 阶段 1：最小引擎骨架可编译运行

**里程碑**：`main()` 能启动，拥有最简陋的 World、Entity、Component、System。这是后续一切的基础。

**验收标准**：
- [x] 有一个 `World` 类管理实体生命周期
- [x] 有平铺的组件存储（哪怕是 `std::vector`）
- [x] 有至少一个 `System` 在 Tick 中批量修改组件
- [x] 有自描述的 `TypeRegistry`，让外部无需读头文件即可知道数据结构
- [x] `main()` 能编译运行并输出 3 帧以上的状态变化

### 1.1 从零开始的引擎骨架

| 1.1.2 自描述 TypeRegistry | ✅ | 运行时组件元数据 |
| 1.1.3 确定性 System 调度 | ✅ | 固定顺序 Tick |
| 1.1.4 引入 AI 后的骨架增强 | ✅ | AgentBridge 概念、MCP 适配层、权限控制 |

> **前置**：[[AI友好的引擎架构]]（原则理解）
> **下一步**：[[组件系统架构]] — 骨架跑通后，立刻把裸数组升级为真正的 ECS 存储模型。

## 阶段 2：基础工具就位

**里程碑**：数学库、字符串、容器/分配器已集成。引擎能处理基本的数值计算和内存分配。

**验收标准**：
- [ ] 向量/矩阵/四元数运算可编译验证
- [ ] 有小型字符串优化或固定缓冲字符串的实现
- [ ] 有至少一种自定义分配器（FrameArena 或 ObjectPool）
- [ ] 随机数生成器在相同种子下输出确定

> **阶段说明**：基础层不依赖于上层，但上层都依赖于它。你可以在阶段 1~5 的任何间隙补充阅读，但**必须在进入渲染管线前完成数学基础**。

## 阶段 3：核心运行时闭环

**里程碑**：ECS 查询/存储、场景图、反射、Tick 调度全部连通。引擎能"描述一个世界并让它运转"。

**验收标准**：
- [x] ECS 存储模型支持多组件联合查询（Query）
- [x] 实体能在空间中建立层级父子关系，且变换能正确传播
- [x] 反射系统能自动/半自动生成组件的字段描述
- [x] System 能按依赖图排序执行，支持命令缓冲延迟提交
- [x] 能保存/恢复世界快照，同一输入序列产生同一输出

## 阶段 4：第一帧画面渲染

**里程碑**：RHI 抽象、资源管理、渲染生命周期已接通。屏幕上有可见输出（三角形或立方体）。

**验收标准**：
- [x] 有跨平台的 RHI 抽象层（至少支持一个后端，如 D3D12 或 Vulkan）
- [x] 能创建窗口、交换链、命令缓冲区
- [x] 有资源加载器，能管理纹理/网格的生命周期
- [x] 渲染一帧的完整生命周期清晰可追踪（Culling → Draw Call → Present）
- [x] 屏幕上至少能渲染一个三角形或带纹理的立方体

- [ ] AI 能通过工具查询实体、修改组件、推进帧、保存快照

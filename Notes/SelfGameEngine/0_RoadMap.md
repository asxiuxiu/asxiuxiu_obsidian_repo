# SelfGameEngine 构建手册

> 本手册是一本**可执行的引擎构建指南**。每一步都必须是**可落地、可看见、可验证**的。我们不追求先读完所有理论再动手，而是**每完成一个阶段，屏幕上就要多出一些东西**。
>
> **核心设计理念**：为 **AI 协作开发** 而设计的 ECS 引擎。每篇笔记默认回扣两个问题：**AI 如何观察它？** **AI 如何操作它？**

## 构建路线图总览

```
阶段 0：元规则确立
    │
    ▼
阶段 1：Hello Engine Window（窗口 + 日志 + 主循环）
    │
    ▼
阶段 2：最小 ECS 骨架（在窗口里可视化调试 ECS）
    │
    ▼
阶段 3：基础工具层（数学 / 字符串 / 容器 / 文件IO / 线程池）
    │
    ▼
阶段 4：核心运行时闭环（场景图 / 反射 / 调度 / 事件 / Prefab）
    │
    ▼
阶段 5：渲染管线与第一帧（RHI / 资源 / 画三角形）
    │
    ▼
阶段 6：动画与 Gameplay 运行时（动画 / Motor / 物理 / 导航）
    │
    ▼
阶段 7：工具链与 AI 桥接（编辑器 / 脚本 / MCP / 多 Agent）
    │
    ▼
阶段 8：工业级专题增强（网络 / 内存 / 服务器 / CI-CD）
```

**每个阶段的验收标准是"能看见的"**：
- **阶段 1**：一个能打开的窗口，左上角有实时滚动的日志和 FPS 计数器
- **阶段 2**：同一个窗口里，ImGui 面板能列出 ECS 实体和组件数值
- **阶段 3**：通过日志/Inspector 验证数学运算和文件加载，不再硬编码 `printf`
- **阶段 4**：窗口里有会动的几何体，ImGui 可以实时修改 Transform 数值并看到变化
- **阶段 5**：屏幕中央出现带纹理的三角形或立方体，由 ECS 组件驱动渲染
- **阶段 6**：窗口里有一个能行走、跳跃、播放动画的角色，能碰撞和寻路
- **阶段 7**：外部 AI 能发送命令修改场景中的组件，编辑器支持 Undo/Redo
- **阶段 8**：多人客户端联机、状态同步、CI/CD 自动打包部署

## 阶段 1：Hello Engine Window

**里程碑**：`main()` 启动后弹出一个可交互窗口，有实时日志流和 FPS 显示。从此不再依赖 `printf` 硬编码验证。

**验收标准（能看见的）**：
- [x] 运行后弹出一个窗口（GLFW / SDL / Win32 / 任意后端）
- [x] 窗口左上角有实时滚动的日志面板（至少 Debug / Info / Error 三级过滤）
- [x] 日志能输出到控制台和文件双端
- [x] 窗口标题栏或面板显示当前 FPS 和帧时间
- [x] 按 ESC 键可以干净退出，无内存泄漏

> **下一步**：先让"能看见的日志"就位，再 build 上层。

### 1.2 可视化日志系统

- [x] [[可视化日志系统]] — 异步日志核心、文件滚动、屏幕面板、调用点追踪

## 阶段 4：核心运行时闭环

- [ ] 窗口里有 3~5 个彩色立方体在旋转或平移

- [ ] 立方体之间有父子层级关系，修改父节点时子节点跟随
- [ ] 按空格键可以触发一个事件，让所有立方体变颜色

## 阶段 8：工业级专题增强

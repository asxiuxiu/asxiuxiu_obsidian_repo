
## 2W2H 笔记：C++ 软件渲染器

### 🤔 Why：与我何干？
现代 GPU 驱动和图形 API 隐藏了太多细节。用 C++ 从零写渲染器，你能亲手触摸到每一个像素是如何诞生的——这让你真正理解 GPU 在做什么，也为日后驾驭 Vulkan、写计算着色器、甚至设计光线追踪器打下根基。

---

### 📦 What：核心组件
| 模块 | 职责 | 关键概念 |
|:---|:---|:---|
| **🖼️ 帧缓冲（Framebuffer）** | 内存中的二维像素数组 | 宽度、高度、颜色缓冲、深度缓冲 |
| **📐 几何表示** | 顶点 + 索引定义三角形网格 | 顶点属性：位置、法线、UV |
| **🔄 坐标变换管线** | 3D 世界 → 2D 屏幕 | **MVP 矩阵**（Model-View-Projection） |
| **🔺 光栅化（Rasterization）** | 三角形 → 像素覆盖 | 重心坐标、边缘函数、插值 |
| **💡 着色计算** | 每个像素最终颜色 | 顶点着色器、片段着色器（软件模拟） |

---

#### 第二阶段：画一个三角形（第 3-7 天）

这是整个渲染器的核心难点。三种方法，由简到精：

| 方法                     | 思路                  | 学习价值         |
| :--------------------- | :------------------ | :----------- |
| 1. 包围盒+重心坐标            | 遍历三角形 AABB，判断点是否在内部 | 理解插值原理       |
| 2. 边缘函数（Edge Function） | 用叉积符号判断左右，并行友好      | 现代 GPU 的标准算法 |
| 3. 扫描线（Scanline）       | 逐行计算左右边界，填充         | 经典算法，理解历史    |

**推荐从方法 1 开始**，代码最直观：

```cpp
// 重心坐标：判断点 P 在三角形 ABC 内，并计算插值权重
bool barycentric(const Vec3& A, const Vec3& B, const Vec3& C, 
                 const Vec3& P, float& u, float& v, float& w);
```

**关键细节**：
- 顶点坐标经过 **🔄 视口变换（Viewport Transform）** 从 `[-1,1]` 映射到屏幕像素
- 用 **🎯 Z-Buffer** 解决深度冲突：每个像素只保留最近的三角形

#### 第三阶段：3D 管线贯通（第 2-4 周）

**坐标变换链（左手记忆：M-V-P-S）**：

```
模型空间 (Model) ──[Model矩阵]──► 世界空间 (World)
                              ──[View矩阵]──► 相机空间 (Camera/View)
                                            ──[Projection矩阵]──► 裁剪空间 (Clip)
                                                              ──[透视除法]──► NDC
                                                                            ──[Viewport]──► 屏幕空间
```

**矩阵库选择**：
| 方案 | 建议 |
|:---|:---|
| 自己写 4×4 矩阵类 | 推荐！理解旋转、平移、透视投影的数学 |
| 用 GLM | 生产常用，但初学建议先手写 |

**顶点着色器（软件模拟）**：

```cpp
struct VertexOut {
    Vec4 position;  // 裁剪空间
    Vec3 normal;    // 透视校正插值
    Vec2 uv;
};

VertexOut vertex_shader(const VertexIn& in, const Uniforms& u) {
    VertexOut out;
    out.position = u.MVP * Vec4(in.position, 1.0);
    out.normal = (u.normal_matrix * Vec4(in.normal, 0.0)).xyz();
    out.uv = in.uv;
    return out;
}
```

**片段着色器**：

```cpp
Vec4 fragment_shader(const VertexOut& in, const Uniforms& u, const Texture* tex) {
    Vec3 light_dir = normalize(u.light_pos - in.world_pos);
    float diff = max(dot(normalize(in.normal), light_dir), 0.0);
    Vec3 color = tex ? texture_sample(tex, in.uv) : u.base_color;
    return Vec4(color * diff, 1.0);
}
```

#### 第四阶段：加载模型与纹理（第 4-6 周）

| 任务 | 工具/格式 | 注意点 |
|:---|:---|:---|
| 加载模型 | **📦 OBJ 格式**（文本，易解析） | 只读顶点、法线、UV、面索引 |
| 加载纹理 | **🖼️ TGA 格式**（无压缩，易解析） | 或写 BMP/PNM 读取器 |
| 透视校正插值 | 1/w 在屏幕空间线性插值 | 纹理映射不扭曲的关键 |

**OBJ 解析核心循环**：
```cpp
if (line[0] == 'v' && line[1] == ' ') parse_position();
if (line[0] == 'v' && line[1] == 'n') parse_normal();
if (line[0] == 'v' && line[1] == 't') parse_uv();
if (line[0] == 'f') parse_face();  // 处理 v/vt/vn 索引
```

### ✨ How Good：你能获得什么？
- **彻底理解 GPU 管线**：顶点处理 → 图元装配 → 光栅化 → 片段处理

- **无缝衔接现代 API**：之后学 Vulkan，只是同样的概念换了一套接口
- **可展示的作品**：一个从零构建的渲染器，比调库更有说服力

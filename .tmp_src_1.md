# client-源码解析：渲染窗口与 SwapChain

## Why：为什么要分析渲染窗口与 SwapChain？

- **问题背景**：游戏引擎最终要把画面呈现到屏幕上，而屏幕上的第一个像素始于窗口创建和 SwapChain 绑定。理解这一层有助于搞清楚编辑器是如何把渲染画面嵌入 WPF 面板的，也能解释为什么 PIE 模式下"编辑器里长啥样，游戏里就长啥样"。
- **不用自研窗口抽象的后果**：若直接依赖 GLFW/SDL，则难以在编辑器中把同一个渲染器无缝嵌入到第三方托管窗口（WPF）内。
- **应用场景**：
  - 游戏运行时的全屏/窗口模式切换、分辨率动态调整
  - 编辑器 Scene/Simulator 面板的渲染视口桥接
  - 多窗口（如多显示器）场景下的 SwapChain 管理

## What：client 层对外提供了什么？

### 核心类一览

| 类 | 文件 | 职责 |
|----|------|------|
| `RenderWindow` | `client/render/render_window/chaos_render_window.h` | 跨平台窗口抽象基类，定义窗口尺寸、模式、标题等通用接口 |
| `WindowsRenderWindow` | `client/render/render_window/windows/chaos_windows_render_window.h/.cpp` | Win32 实现，直接操作 `HWND` 和 Win32 API |
| `RenderSystemManager` | `client/render/chaos_render_system_manager.h/.cpp` | 渲染系统总控，管理 `RenderWindow` 数组、SwapChain 生命周期、分辨率切换 |
| `RHISystem` | `client/graphics/graphics_rhi/chaos_rhi.h + chaos_swapchain.cpp` | RHI 层入口，负责 `createSwapChain`、`resizeSwapChain`、`present` |
| `SwapChainBase` | 平台相关子目录（如 `graphics_d3d12/`、`graphics_vulkan/`） | 各图形 API 的 SwapChain 具体实现 |

## How：源码级深度解析

### 第 1 层：接口层

#### 1.1 RenderWindow — 跨平台窗口抽象

> 文件：`chaos/_source/_engine/source/client/public/chaos/client/render/render_window/chaos_render_window.h`，第 71~167 行

```cpp
class CLIENT_API RenderWindow
{
public:
    virtual void SetWindowHandle(void* native_handle) {}
    virtual void CreateRenderWindow(
        const String& title, Int monitor_index,
        RenderSettings::WindowMode window_mode,
        Int final_window_width, Int final_window_height) {}
    virtual void* GetOSWindowHandle() const { return nullptr; }
    virtual void ReshapeWindow(Int x, Int y, Int width, Int height) {}
    virtual void SetWindowMode(EWindowMode window_mode) {}
    // ... 最大化、最小化、焦点、DPI 等接口
protected:
    Int m_width = 1280, m_height = 720;
    Int m_render_width = 0, m_render_height = 0;
    Bool m_is_extern_hwnd {k_false}; // 标记是否为外部传入的窗口句柄
    WindowDefinition m_window_definition;
    EWindowMode m_window_mode;
    RenderWindowID m_render_window_id;
    Float m_dpi_scale_factor{ 1.f };
};
```

**关键接口语义**：
- `SetWindowHandle`：绑定一个**外部传入**的原生窗口句柄（编辑器模式下由 WPF 提供 `Panel` 的 `HWND`）。
- `CreateRenderWindow`：由引擎**自己创建**一个新的原生窗口（游戏运行时模式）。
- `m_is_extern_hwnd`：区分"引擎自管窗口"与"托管窗口"，影响析构时是否调用 `DestroyWindow`。

#### 1.2 WindowsRenderWindow — Win32 实现

> 文件：`chaos/_source/_engine/source/client/public/chaos/client/render/render_window/windows/chaos_windows_render_window.h`，第 25~153 行

```cpp
class CLIENT_API WindowsRenderWindow : public RenderWindow
{
public:
    WindowsRenderWindow(HINSTANCE instance);
    virtual void CreateRenderWindow(...) final override;
    virtual void SetWindowHandle(void* native_handle) override;
    virtual void* GetOSWindowHandle() const override { return m_hwnd; }
    HWND GetHWnd() const { return m_hwnd; }
    // ...
protected:
    HINSTANCE m_process_instance;
    HWND      m_hwnd;

    DWORD m_windowed_style;
    DWORD m_borderless_fullscreen_style;
    DWORD m_fullscreen_style;

```

**设计要点**：
- 直接使用 `WNDCLASSEXW` 注册窗口类 `"Chaos_Window"`，`lpfnWndProc` 指向 `WindowEventUtilities::_WndProc`。
- 全屏/无边框全屏/窗口化三种模式通过切换 `GWL_STYLE`（`WS_POPUP` vs `WS_OVERLAPPED | WS_CAPTION`）实现。
- `SetWindowHandle` 会将外部 `HWND` 的宽高同步到 `m_width/m_height`，并设置 `m_is_extern_hwnd = true`，避免引擎析构时销毁编辑器面板。

#### 3.2 编辑器模式：WPF HWND 桥接

编辑器 C# 前端通过 `ModuleEntryPoint` 创建 `EditorCore` 时，会把 `WindowsFormsHost` 中 `Panel` 的 `HWND` 作为 `native_wnd` 传入：

> 文件：`chaos/_source/_tools/source/modules/common/engine_common_module/ModuleEntryPoint.cs`，第 36~46 行

```csharp
public ModuleEntryPoint(IntPtr native_wnd, IntPtr main_wnd)
{
    EditorCore = ChaosEditorModule.CreatEditorCore(native_wnd, main_wnd);
}
```

C++ 侧 `EngineEditorCore` 初始化时调用 `RenderSystemManager::initialize(native_wnd)`，最终进入 `WindowsRenderWindow::SetWindowHandle()`：

> 文件：`chaos/_source/_engine/source/client/private/chaos/client/render/render_window/windows/chaos_windows_render_window.cpp`，第 44~95 行

```cpp
void WindowsRenderWindow::SetWindowHandle(void* native_wnd)
{
    Destroy();
    m_hwnd = static_cast<HWND>(native_wnd);
    m_is_extern_hwnd = true;
    // 同步外部窗口尺寸
    RECT rc;
    ::GetClientRect(m_hwnd, &rc);
    m_width = rc.right - rc.left;
    m_height = rc.bottom - rc.top;
    // 注册到事件系统、IME、输入设备
    WindowEventUtilities::_addRenderWindow(this);
    g_client_global_context.m_ime_manager->setDefaultHWND(m_hwnd);
    g_client_global_context.m_input_device_manager->setDefaultHWND(m_hwnd);
}
```

**核心约束**：编辑器中的 Scene 面板和 Simulator 面板共用一个 `HWND`（单 `Panel` 句柄）。当用户在两个面板间切换时，Avalondock 事件触发，WPF 将该 `Panel` 的父容器迁移，但**渲染目标句柄不变**，C++ 渲染器持续向同一 `HWND` 提交帧。

#### 3.3 SwapChain 创建流程

> 文件：`chaos/_source/_engine/source/client/private/chaos/client/graphics/graphics_rhi/chaos_swapchain.cpp`，第 76~197 行

```cpp
void RHISystem::createSwapChain(RenderWindowID render_window_id, const SwapChainDesc& swapchain_info)
{
    clearSwapChain(render_window_id); // 先清理旧链
    m_swap_chain_desc.m_buffer_count = 2; // PC 默认双缓冲
    m_swap_chain_desc.m_color_buffer_format = TEX_FORMAT_RGBA8_UNORM;
    m_swap_chain_desc.m_depth_buffer_format = TEX_FORMAT_D24_UNORM_S8_UINT;
    m_swap_chain_desc.m_width  = swapchain_info.m_width;
    m_swap_chain_desc.m_height = swapchain_info.m_height;

    void* window_handle = g_client_global_context.m_render_system_manager
        ->getWindowByID(render_window_id)->GetOSWindowHandle();

    SwapChainBase* swap_chain_ptr = nullptr;
    m_engine_factory->CreateSwapChain(
        m_render_device_ptr, m_immediate_context_ptr,
        m_texture_factory_delegate, m_swap_chain_desc,
        full_screen_mode_desc, window_handle, &swap_chain_ptr);

    m_swap_chain_map.insert(render_window_id, swap_chain_ptr);

    // 提取 BackBuffer RTV/DSV，设置到 Immediate Context
    TextureViewBase* pRTV = (TextureViewBase*)(swap_chain_ptr->GetCurrentBackBufferRTV());
    TextureViewBase* pDSV = (TextureViewBase*)(swap_chain_ptr->GetDepthBufferDSV());
    m_immediate_context_ptr->SetRenderTargets(1, &pRTV, ..., pDSV, ...);

    // 包装为 CommonRHITexture2D 供上层使用
    CommonRHITexture2D* NewTexture = CHAOS_NEW_T(CommonRHITexture2D)(...);
    m_back_buffer_map.insert(render_window_id, NewTexture);
}
```

**关键观察**：
- `CreateSwapChain` 的最后一个参数 `window_handle` 在 Win32 下就是 `HWND`，在 Vulkan 下可能是 `HWND` / `xcb_window_t` / `wl_surface`，在 Metal 下是 `CALayer`。
- 创建完成后立即绑定 RTV/DSV 到 Immediate Context，并清空深度模板缓冲，确保后续渲染可以直接开始。

RHI 层的 `resizeSwapChain`（以 D3D12 为例）：

> 文件：`chaos/_source/_engine/source/client/private/chaos/client/graphics/graphics_rhi/chaos_swapchain.cpp`，第 242~294 行

```cpp
void RHISystem::resizeSwapChain(RenderWindowID render_window_id, SwapChainResizeInfo resize_info)
{
    // 1. 清理旧 BackBuffer
    clearSwapChain(render_window_id);
    // 2. 重置 Immediate Context 的 RenderTarget
    m_immediate_rhi_context->clearAllRenderTargets();
    m_immediate_context_ptr->ResetRenderTargets();
    m_immediate_context_ptr->InvalidateState();
    m_immediate_context_ptr->Flush();
    // 3. 调用底层 SwapChain.Resize
    ChaosD3D12SwapChain* swap_chain = ...;
    swap_chain->Resize(resize_info.m_width, resize_info.m_height, ...);
    // 4. 重建 BackBuffer 的 RHI 包装
    CommonRHITexture2D* NewTexture = CHAOS_NEW_T(CommonRHITexture2D)(...);
    m_back_buffer_map.insert(render_window_id, NewTexture);
}
```

**Resize 时的线程安全策略**：
- `updateWindowSize()` 只在逻辑线程中把变更信息推入 `m_render_window_update_infos`。
- 真正的 Resize 被延迟到渲染线程的 `rhiPreRender()` 中执行，确保 GPU 命令列表已 Flush，避免在 Resize 时仍有未完成的绘制命令引用旧 BackBuffer。

## 与上下层的关系

```
┌─────────────────────────────────────────────────────────────┐
│  C# 前端 (WPF) / 游戏运行时启动器                              │
│  ├─ 编辑器：WindowsFormsHost Panel → HWND                    │
│  └─ 游戏：无（由引擎自创建窗口）                               │
├─────────────────────────────────────────────────────────────┤
│  client/render/                                              │
│  ├─ RenderWindow (跨平台抽象)                                │
│  ├─ WindowsRenderWindow (Win32 HWND)                         │
│  └─ RenderSystemManager (窗口数组 + SwapChain 生命周期)       │
├─────────────────────────────────────────────────────────────┤
│  client/graphics/graphics_rhi/                                │
│  ├─ RHISystem::createSwapChain / resizeSwapChain / present   │
│  ├─ SwapChainBase (D3D12/Vulkan/Metal 派生)                  │
│  └─ CommonRHITexture2D (BackBuffer 包装)                     │
├─────────────────────────────────────────────────────────────┤
│  底层图形 API                                                 │
│  ├─ D3D12: IDXGISwapChain3                                   │

└─────────────────────────────────────────────────────────────┘
```

**引用关系**：
- `RenderSystemManager` 被 `ClientTickManager` 每帧在 `tickRender` 前调用 `preRender()`，驱动 SwapChain BeginFrame。
- `RenderPipeline` / `RenderSceneManager` 在渲染阶段从 `RHISystem` 获取 `m_back_buffer_map` 中的 BackBuffer 作为最终输出目标。
- 编辑器 `EngineEditorPIEManager::resizePIEViewPort()` 直接调用 `m_client_viewport_accessor->resizeClientViewport()`，同时影响 `RenderWindow` 的视口尺寸。

## 设计亮点

1. **外部 HWND 兼容**：`SetWindowHandle` + `m_is_extern_hwnd` 设计让引擎渲染器可以无缝嵌入到 WPF、Qt、WinForms 等第三方框架中，无需为编辑器重写一套渲染后端。
2. **延迟创建机制**：`asyncCreateWindow` 将窗口创建延迟到 `rhiPreRender()`，避免在 UI 事件线程中直接操作 GPU 资源，消除多线程竞态。
3. **全屏状态本地保存**：`WindowsRenderWindow` 使用 `WINDOWPLACEMENT` 记录进入全屏前的窗口位置和尺寸，退出全屏时一键还原。
4. **Resize 安全窗口**：SwapChain Resize 前执行 `Flush` + `InvalidateState` + `ResetRenderTargets`，确保旧 BackBuffer 的引用已全部释放，杜绝 D3D12 `DXGI_ERROR_DEVICE_REMOVED` 类崩溃。
5. **同一渲染目标保证视觉一致**：编辑器 Scene 面板和 PIE 模式共用同一个 `HWND`/SwapChain，只是切换了 `ClientWorld`（`g_current_world`），画面输出路径完全一致。

# C++ 练习动态计划调整

## 定位

本 Skill 是练习计划的**策略顾问**。它不替用户做决定，而是：
1. 读取 tracker 数据，识别模式和趋势
2. 根据规则生成调整建议
3. 用户确认后，修改计划笔记

## 触发条件

- 用户说"调整计划" / "后面怎么安排" / "帮我看看计划"
- 每天练习自然结束后由 Orchestrator 触发
- 用户主动要求"合并 Day X 和 Day Y" / "跳过这道题"

## 前置步骤

1. 读取 `workspace/cpp-recovery/.practice-tracker/state.json`
2. 读取 `Notes/学习计划/C++基础恢复70天计划.md`
3. 分析最近 3-7 天的 dailyRecords 数据

## 调整策略

### 自动触发条件

| 信号 | 触发条件 | 建议动作 |
|------|---------|---------|
| **加速** | 连续 2 天 30 分钟内独立完成 + 评审无大问题 | 建议合并下一组题目，或提前进入复盘 |
| **放缓** | 当天用到 L3 提示仍未完成，或校验 ≤1/4 | 建议拆分当天任务，或插入同类巩固题 |
| **巩固** | 某维度连续 3 天增长 ≤0.2 | 在计划中插入该维度的专项练习 |
| **跳过** | 用户明确表示"这题太简单了" | 标记为已掌握，跳到下一题 |
| **合并** | 用户要求合并相邻题目 | 生成组合任务描述，保留两者考察点 |

### 调整规则详情

#### 1. 合并任务

当用户要求合并或系统建议合并时：
- 保留**所有被合并题目的核心考察点**
- 新任务描述包含多个子目标
- 难度取最高值，业务映射保留全部

**示例**：合并 Day 1（String 构造+析构）和 Day 2（拷贝构造+拷贝赋值）
```
新任务：手写完整 String 类（构造、析构、深拷贝）
考察点：构造函数、析构函数、深拷贝、拷贝语义、自赋值检查
难度：⭐⭐
```

#### 2. 插入巩固题

当某维度薄弱时，从题库中抽取同维度、同难度或更低难度的题目插入：
- 语法巩固：简单的类封装练习
- 指针巩固：手动管理数组、链表节点
- 内存巩固：RAII 小练习、异常安全边界条件

#### 3. 拆分任务

当任务太难时，拆成多个子任务：
- 原任务：手写 SharedPtr（简化版）
- 拆分后：
  - 子任务 A：只写控制块（ControlBlock）
  - 子任务 B：写 SharedPtr 构造和析构
  - 子任务 C：写拷贝构造和拷贝赋值

#### 4. 跳过标记

用户确认跳过某题后：
- 在计划笔记中标记为 ⏭️ 跳过
- 在 state.json 中记录跳过原因
- 不推进能力评分（因为没实际练习）

## 输出格式

```
📋 计划调整建议
━━━━━━━━━━━━━━━━━━━━━━

📊 当前状态：
  已完成：Day 1 ~ Day N
  当前能力：syntax=X pointer=Y ...
  最近趋势：<加速/平稳/放缓>

🔧 建议调整（基于最近 X 天数据）：

  1. <具体建议>
     理由：<为什么这样建议>
     影响：<计划会如何变化>

  2. <如果有第二条建议>
     ...

📅 调整后的计划预览：
  Day N+1: <任务>
  Day N+2: <任务>
  ...

❓ 是否采纳？
  说"采纳"确认修改计划笔记
  说"调整"告诉我具体想怎么改
  说"保持"继续原计划
```

## 计划笔记修改规范

用户确认采纳后，修改 `Notes/学习计划/C++基础恢复70天计划.md`：

1. **合并**：在被合并的表格行下方添加注释说明，创建新的合并行
2. **插入**：在合适位置插入新行，保持整体顺序
3. **跳过**：在对应行状态列添加 ⏭️ 标记
4. **调整难度**：修改难度星级，添加调整原因注释

## 用户主动调整

用户可以直接说：
- "把 Day 2 和 Day 3 合并" → 执行合并逻辑
- "跳过 Day 5" → 执行跳过逻辑
- "在 Day 4 后面加一道链表题" → 执行插入逻辑
- "减慢速度，每道题给我两天" → 调整整体节奏

## 注意事项

1. **建议不是命令**：所有调整必须经用户确认后才执行
2. **保持骨架完整**：即使合并/跳过，也要确保 10 周的主题骨架不变
3. **记录调整历史**：所有调整原因记录到 state.json 的 planAdjustments 中
4. **不复盘不调整**：复盘日（Day 7/14/...）不引入新调整，只做总结

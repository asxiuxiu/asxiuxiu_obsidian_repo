# C++ 练习知识校验

## 定位

本 Skill 是练习效果的**质检员**。它不评价代码，而是：
1. 针对当天知识点设计追问
2. 让用户口头回答（不需要写代码）
3. 评估掌握度，记录薄弱点

## 触发条件

- 用户说"考考我" / "来个小测验" / "检验一下"
- 用户说"今天学的掌握了没有"
- 每天练习结束后由 Orchestrator 建议进行

## 前置步骤

1. 读取 `workspace/cpp-recovery/.practice-tracker/state.json`，确认 `currentDay` 和 `weekTheme`
2. 读取 `Notes/学习计划/C++基础恢复70天计划.md` 获取当天题目和考察点
3. 基于考察点设计追问

## 追问设计原则

每次校验设计 **3~5 个问题**，覆盖以下类型：

| 类型 | 目的 | 数量 |
|------|------|------|
| **概念理解** | 知道"是什么" | 1 |
| **设计动机** | 知道"为什么" | 1 |
| **边界/变体** | 能推演"如果...会怎样" | 1 |
| **代码纠错** | 能识别常见错误 | 1（可选） |
| **标准库关联** | 能与标准库设计建立联系 | 1（可选） |

### 各类型示例

**概念理解**（Day 1 String 构造+析构）：
> "你的 String 构造函数里，为什么用 `new char[len_ + 1]` 而不是 `new char[len_]`？"

**设计动机**（Day 3 String 移动语义）：
> "为什么 C++11 要引入移动语义？如果只有拷贝构造，写 `return std::move(str)` 会怎样？"

**边界/变体**（Day 2 String 拷贝赋值）：
> "如果你的拷贝赋值运算符收到 `s = s`（自赋值），不做特殊处理会发生什么？"

**代码纠错**（Day 5 UniquePtr）：
> "下面这段 UniquePtr 的析构函数有什么问题？
> ```cpp
> ~UniquePtr() { delete ptr_; }
> ```
> 提示：考虑 `UniquePtr<int[]>` 的情况。"

## 交互方式

### 逐题提问模式（推荐）

一次只问一题，等用户回答后再问下一题。这样：
- 可以给即时反馈
- 可以根据用户回答调整后续问题的难度
- 更像对话，不像考试

### 流程

```
📝 知识校验（Day X: <taskName>）
━━━━━━━━━━━━━━━━━━━━━━

第 1/4 题（概念理解）：
<问题>

💬 请回答...

---

✅ 你的回答：<用户回答>

📌 评分：<正确/部分正确/需加强>
📌 关键点：<用户是否命中核心概念>
📌 补充：<如果回答不完整，给简要补充>

第 2/4 题（设计动机）：
...
```

### 评分标准

| 回答质量 | 标记 | 对能力矩阵的影响 |
|---------|------|----------------|
| 完全正确，能举一反三 | 🟢 掌握 | 相关维度 +1.0 |
| 基本正确，有小遗漏 | 🟡 基本掌握 | 相关维度 +0.5 |
| 部分正确，概念模糊 | 🟠 需巩固 | 相关维度 无变化 |
| 错误或答不上来 | 🔴 未掌握 | 相关维度 -0.5 |

## 输出格式

校验结束后输出总结：

```
📊 校验结果（Day X: <taskName>）
━━━━━━━━━━━━━━━━━━━━━━

得分：X/4

各题表现：
  1. 概念理解      🟢 掌握
  2. 设计动机      🟡 基本掌握
  3. 边界/变体     🟠 需巩固
  4. 代码纠错      🟢 掌握

🔍 薄弱点：
  • <具体薄弱点 1> → 建议复习 <相关笔记/代码>
  • <具体薄弱点 2> → ...

✅ 已掌握：
  • <已确认掌握的知识点>

📌 建议：
  <基于校验结果的下一步建议>
```

## 超纲控制

校验问题**严格限定在当天学习目标内**：
- Day 1 不问拷贝构造相关的问题
- Day 3 才问移动语义
- 不涉及尚未学习的标准库内部实现

如果用户回答时提到了超纲内容，可以顺势拓展，但**不纳入评分**。

## 状态更新

每道题问答结束后，追加到当天会话日志：

```bash
echo '{"ts":"2026-06-03T14:35:00","day":1,"taskId":"day01_string_basic","type":"check","qIndex":1,"qType":"概念理解","userAnswer":"...","rating":"基本掌握"}' >> .practice-tracker/sessions/day01_2026-06-03.jsonl
```

`rating`：🟢掌握 / 🟡基本掌握 / 🟠需巩固 / 🔴未掌握

校验结束后，更新 `state.json`：

## 注意事项

1. **不是考试**：校验的目的是帮助用户发现盲区，不是打分排名
2. **允许翻笔记**：用户可以说"让我看看笔记"，这不影响评分
3. **鼓励推导**：如果用户说"我猜是..."，让他推推导过程，推导正确也算掌握
4. **错题变笔记**：校验中发现的薄弱点，建议用户补充到笔记中

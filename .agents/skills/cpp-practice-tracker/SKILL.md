# C++ 练习水平追踪

## 定位

本 Skill 是练习数据的**记录员和分析师**。它不直接参与代码评审或提示，而是：
1. 收集每次交互的信号（评审结果、提示使用次数、校验得分）
2. 更新 6 维能力矩阵
3. 输出能力变化趋势和弱项预警

## 触发条件

- 用户说"记录今天的练习" / "记录一下" / "存档"
- 用户说"我的水平怎么样" / "能力报告" / "进度如何"
- 每次 Review / Hint / Check 执行后由 Orchestrator 自动触发
- 每个 Phase 的复盘练习完成后自动输出阶段总结

## 6 维能力矩阵

| 维度 | 英文名 | 衡量标准 | 评分 0-10 |
|------|--------|---------|----------|
| 语法熟练度 | `syntax` | 编译通过率、代码整洁度、命名规范 | 0 = 频繁编译错误；10 = 一次通过 |
| 指针操作 | `pointer` | raw pointer 管理、内存布局理解、nullptr 检查 | 0 = 害怕指针；10 = 能画内存图 |
| 内存管理 | `memoryMgmt` | RAII、深拷贝/移动、异常安全、无泄漏 | 0 = 忘记 delete；10 = 异常安全 |
| 模板理解 | `template` | 模板参数使用、实例化理解、简单元编程 | 0 = 看不懂报错；10 = 能写 traits |
| 设计意识 | `design` | 接口设计、封装、最小暴露、职责分离 | 0 = 全 public；10 = 合理抽象 |
| 并发基础 | `concurrency` | 锁使用、线程安全、数据竞争识别 | 0 = 未接触；10 = 能写线程池 |

## 评分规则

### 从评审结果中提取信号

**基础信号**（直接从单轮评审提取）：

| 评审信号 | 对应维度变化 |
|---------|-------------|
| 编译一次通过 | syntax +0.5 |
| 评审发现内存泄漏 bug | memoryMgmt -0.5 |
| 评审表扬接口设计好 | design +0.5 |
| 评审建议用初始化列表（没想起来）| syntax -0.3 |
| 正确处理了边界条件 | pointer +0.5 |
| 遗漏了自赋值检查 | memoryMgmt -0.5 |

**迭代修正信号**（从完整交互日志提取，用于更准确的评估）：

| 日志信号 | 说明 | 维度变化 |
|---------|------|---------|
| Round 1 通过（一次写对）| 独立完成，无 hint | 相关维度额外 +0.5 |
| Round 2 通过 | 评审后自行修正 | 相关维度 无额外变化 |
| Round ≥3 通过 | 多次迭代才通过 | 相关维度 -0.3（不熟练） |
| 使用了 Hint | 日志中存在 `type:hint` | 根据 hint 级别按规则评分（见下） |
| 重复犯同类错误 | 多轮评审 issues 有重叠 | memoryMgmt/pointer -0.5（概念未真正掌握） |

### 从提示使用中提取信号

从交互日志中读取所有 `type:hint` 记录，统计使用次数和级别：

| 提示使用（以日志为准） | 信号 |
|---------|------|
| 未使用提示独立完成 | 当前练习相关维度 +1.0 |
| 用了 L1 后完成 | 当前练习相关维度 +0.5 |
| 用到 L2 才完成 | 当前练习相关维度 +0.2 |
| 用到 L3 才完成 | 当前练习相关维度 无变化（已掌握但不熟练）|
| L3 后仍未完成 | 当前练习相关维度 -0.5（需巩固） |

**重要**：如果用户换窗口导致 AI 不知道之前用过 hint，以**日志文件**为准。AI 每次执行 Tracker 前必须先读取日志。

### 从校验结果中提取信号

| 校验得分 | 信号 |
|---------|------|
| 4/4 全对 | 相关维度 +1.0 |
| 3/4 | 相关维度 +0.5 |
| 2/4 | 相关维度 无变化 |
| ≤1/4 | 相关维度 -0.5（需复习） |

## 状态文件更新

每次 Tracker 执行后，更新 `state.json`：

```json
{
  "meta": { "lastActiveDate": "2026-06-17" },
  "abilityMatrix": {
    "syntax": 2.5,
    "pointer": 1.0,
    "memoryMgmt": 1.5,
    "template": 0,
    "design": 0.5,
    "concurrency": 0
  },
  "exerciseRecords": [
    {
      "phase": "01",
      "exercise": "1.1",
      "abilityDelta": {
        "syntax": 0.5,
        "memoryMgmt": 0.5
      }
    }
  ]
}
```

> 兼容旧数据：如果 `state.json` 中只有 `dailyRecords`，保留 `dailyRecords` 并追加新记录（同时写入 `phase`/`exercise` 字段）。新记录优先写入 `exerciseRecords`。

## 注意事项

> **🔴 第一优先级铁律：Tracker 虽不直接参与代码评审或提示，但若在输出总结或弱项分析时涉及具体代码问题，严禁直接给出答案或正确实现。只能指出薄弱概念方向，引导用户回顾相关笔记或重新尝试。**

1. **评分是相对的**：10 分不是"精通 C++"，而是"当前练习阶段的满分"
2. **允许波动**：某练习评分下降是正常的，连续 3 个练习下降才触发弱项预警
3. **不与他人比较**：只与用户自己的历史数据比较
4. **数据是辅助**：能力矩阵只是参考，最终判断权在用户自己

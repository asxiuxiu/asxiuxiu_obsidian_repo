## 核心问题

> 向量是**列表 of 数字**，还是**空间中的箭头**？
> > **答：取决于你的视角，但几何视角更强大。**

## 三种看待向量的方式

### 1. 物理学视角 — 箭头

向量是空间中的**有向线段**：
- 有**方向**（指向哪里）
- 有**长度**（大小）
- **与位置无关** — 相同方向和长度的向量相等

```
        ↗
       /      ↗
      /      /
     ●─────→●
    
    这些向量是相等的！
```

### 2. 计算机科学视角 — 有序列表

向量是**数字的有序列表**：
```
v = [3, 4]  // 二维向量
v = [1, 2, 3]  // 三维向量
```
- 适合存储和计算
- 抽象了具体含义

### 3. 数学视角 — 满足公理的对象

向量是**满足加法与数乘公理**的任何东西：
- 可以是箭头
- 可以是函数
- 可以是矩阵
- ...只要满足8条公理

## 向量坐标：几何与代数的桥梁

### 建立坐标系

```
     y
     ↑
  4 ─┼────→ v = [3, 4]
     │   ↗
  3 ─┼  / 
     │ /  
  2 ─┼/   
     ●────┬────┬──→ x
     1    2    3
```

### 坐标的几何意义

向量 **v = [3, 4]** 表示：
- 沿 x 轴走 3 个单位
- 沿 y 轴走 4 个单位

基向量（Basis Vectors）：
- **î** = [1, 0] （x方向的单位向量）
- **ĵ** = [1, 0] （y方向的单位向量）

则：**v = 3î + 4ĵ**

## 向量加法：几何直观

### 代数定义
```
[1]   [3]   [4]
[2] + [1] = [3]
```

### 几何解释 — 首尾相接

```
         ↗ v+w
        / 
       /  
   w  /   
     ↗    
    /     
   ●─────→
        v

平移 w，使其起点与 v 的终点重合
```

**物理意义**：先沿 v 走，再沿 w 走，最终位移 = v + w

## 向量数乘：缩放

### 代数定义
```
    [2]   [6]
3 × [1] = [3]
```

### 几何解释

```
          ↑ 2v
          │
    v →   │
          │
   ───────┼──────→
          │
          │
          ↓ -v
          
数乘 = 拉伸/压缩 (+ 可能的反向)
```

| 乘数 | 效果 |
|------|------|
| k > 1 | 伸长 k 倍 |
| 0 < k < 1 | 压缩 |
| -1 | 反向 |
| k < 0 | 反向并缩放 |

## 关键洞察

> **"坐标依赖于基的选择"**

同一个向量，在不同坐标系下有不同的坐标表示。

```
标准基：    旋转后的基：
v = [3, 2]  v' = [?, ?]

坐标变了，但向量本身（箭头）没变！
```

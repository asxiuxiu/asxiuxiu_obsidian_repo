>
> 但每次都去控制每个像素的颜色，未免太累——画一条斜线要算几百个像素坐标，稍微移动一下还得全部重算。

## Part 1：直线光栅化

### Why：为什么要改进画法？

| 画法           | 代码量 | 移动线段     | 适用场景   |
| ------------ | --- | -------- | ------ |
| 逐像素设置        | 数百行 | 重写所有像素   | 基本不会用  |
| **定义端点自动连线** | 几行  | 只改 2 个坐标 | 所有线条绘制 |

核心思路：**用几何描述代替逐像素操作。**

这种"根据两个端点自动填充中间像素"的技术，就是 **直线光栅化（Line Rasterization）**。

### What：直线光栅化要做什么？

#### 问题定义

```
输入：线段的 2 个端点 (x1,y1), (x2,y2)
         ↓
步骤 1：计算线段经过的所有像素
         ↓
步骤 2：把这些像素涂成指定颜色
         ↓
输出：一条画好的线段
```

#### 直观思路：基于方程的采样

数学上，直线方程是 `y = kx + b`。我们可以这样画：

```cpp
// 简单但低效的方法：基于 x 采样
void drawLineNaive(Image& img, int x1, int y1, int x2, int y2, Color c) {
    float k = (float)(y2 - y1) / (x2 - x1);  // 斜率
    float b = y1 - k * x1;                    // 截距
    
    for (int x = x1; x <= x2; x++) {
        int y = (int)(k * x + b);   // 浮点运算 + 类型转换
        img.setPixel(x, y, c);
    }
}
```

**问题很明显**：
1. **浮点运算慢**：每个像素都要做乘法和加法
2. **不适用于陡线**：当 `|k| > 1` 时，y 方向跳跃，会断开
> [!question] 为什么浮点运算会有"精度问题"？
> 
> **三个层面的误差来源：**
> 
> **1. 浮点数本身的存储误差**
> 
> `float` 只有 7 位有效数字。例如画一条从 `(0,0)` 到 `(100000, 1)` 的线：
> - 真实斜率 `k = 0.00001`
> - 但 float 存储时可能变成 `0.0000100000008...` 存在微小偏差
> 
> **2. 误差随迭代累积放大**
> 
> 每一步都用 `y = k * x + b` 重新计算，误差像滚雪球：
> 
> | x | 理想 y | 实际计算的 y | 累积误差 |
> |---|--------|-------------|----------|
> | 0 | 0 | 0.0000001 | 可忽略 |
> | 1000 | 10 | 10.0001 | 开始明显 |
> | 10000 | 100 | 100.001 | 可能错 1 个像素 |
> 
> 线段越长，末端偏离越严重。
> 
> **3. 整数截断是"硬截断"**
> 
> `(int)` 直接舍弃小数部分——0.3、0.5、0.8 都被丢掉，**不会补偿到下一轮**。
> 
> **对比 Bresenham 算法**
> 
> Bresenham 用整数误差跟踪：
> ```cpp
> int e = 0;           // 单位是"像素"，精确
> e += 2 * dy;         // 整数加法，无精度损失
> if (e >= dx) {
>     y++;             // 误差够了才调整
>     e -= 2 * dx;     // 调整时把误差"还回去"
> }
> ```
> 
> 全程整数运算，误差不会累积失控。
> 
> **一句话总结**：精度问题 = 浮点存储误差 × 迭代次数 + 硬截断不补偿

> [!question] 为什么斜率大的线会"断开"？
> 
> **根本原因：采样方向错误**
> 
> 简单画线法的核心是 `x` 每增加 1，算一个 `y`：
> ```cpp
> for (int x = x1; x <= x2; x++) {
>     int y = (int)(k * x + b);  // 根据 x 算 y
>     img.setPixel(x, y, c);      // 只画一个像素
> }
> ```
> 它隐含假设：**x 每走 1 步，y 的变化不超过 1 个像素** —— 这只对平缓直线成立。
> 
> **举个具体例子**：画一条从 `(0,0)` 到 `(3,12)` 的陡线（k = 4）：
> 
> | x | y = 4x | 整数 y | 实际画出的点 |
> |---|--------|--------|-------------|
> | 0 | 0 | 0 | (0, 0) ✓ |
> | 1 | 4 | 4 | (1, 4) ✓ |
> | 2 | 8 | 8 | (2, 8) ✓ |
> | 3 | 12 | 12 | (3, 12) ✓ |
> 
> **发现问题了吗？**
> 
> y 从 0 直接跳到 4，中间 **y = 1,2,3** 这些行完全没有像素！视觉上就是一条虚线：
> ```
> 行12:      ●
> 行11:      
> 行10:      
> 行9:       
> 行8:    ●  
> 行7:       
> 行6:       
> 行5:       
> 行4:  ●    
> 行3:       
> 行2:       
> 行1:       
> 行0:●      ← 中间大片空白！
> ```
> 
> **对比平缓的线** `(0,0)` 到 `(12,3)`，k = 0.25：
> - x 每增加 1，y 增加 0.25
> - 整数 y 每次最多变化 1（或不变）
> - **不会跳过任何行**，线是连续的
> 
> **解决方案**：陡线应该沿着 y 方向采样（把 y 当横坐标，算 x），这样每一步都画一个像素，不会漏掉任何一行。

### How：Bresenham 直线算法

1962 年，IBM 的 Jack Bresenham 提出了一个天才方案：

> **只用整数加法，就能决定下一个像素该点哪里。**

#### 核心思想：增量计算

假设我们在画一条**平缓的直线**（`0 < 斜率 < 1`），每次 x 增加 1，y 要么不变，要么增加 1：

```
当前像素在 (x, y)
下一个像素只能是：
    候选 A: (x+1, y)     ← 保持 y
    候选 B: (x+1, y+1)   ← y 增加 1

选哪个？看谁离真实直线更近！
```

#### 关键洞察：误差积累

我们用 **决策参数 `e`** 来记录"当前 y 与理想直线的垂直偏差"：

```
初始化：e = 0

每一步：
    e = e + 斜率   // 累积误差
    
    if (e >= 0.5) {
        y = y + 1;   // 偏离太多，向上走一步
        e = e - 1;   // 重置误差基准
    }
    
    画点 (x, y)
```

#### 优化：消除浮点

上面的 `0.5` 和斜率 `dy/dx` 都是浮点数。两边同乘 `2*dx`：

```
令 e' = e * 2*dx  （变成整数！）

判断条件 e >= 0.5  变成  e' >= dx
更新 e = e + dy/dx 变成 e' = e' + 2*dy
重置 e = e - 1    变成 e' = e' - 2*dx
```

**现在所有运算都是整数了！**

#### 代码实现

```cpp
// Bresenham 直线算法（适用于 0 < 斜率 < 1 的情况）
void drawLineBresenham(Image& img, int x1, int y1, int x2, int y2, Color c) {
    int dx = x2 - x1;
    int dy = y2 - y1;
    
    int y = y1;
    int e = 0;  // 决策参数（误差积累）
    
    for (int x = x1; x <= x2; x++) {
        img.setPixel(x, y, c);
        
        e += 2 * dy;        // 累积误差
        if (e >= dx) {      // 偏离超过半个像素
            y++;            // 向上走一步
            e -= 2 * dx;    // 重置误差
        }
    }
}
```

#### 扩展到所有象限

上面的代码只处理了**第一象限的平缓直线**。完整的实现需要处理 8 种情况：

```cpp
void drawLine(Image& img, int x1, int y1, int x2, int y2, Color c) {
    int dx = abs(x2 - x1);
    int dy = abs(y2 - y1);
    
    // 判断是"陡"还是"平"
    bool steep = dy > dx;
    
    if (steep) {
        // 陡线：交换 x 和 y，变成平缓情况处理
        swap(x1, y1);
        swap(x2, y2);
        swap(dx, dy);
    }
    
    // 确保从左到右画
    if (x1 > x2) {
        swap(x1, x2);
        swap(y1, y2);
    }
    
    // 判断 y 方向
    int yStep = (y2 > y1) ? 1 : -1;
    
    int y = y1;
    int e = 0;
    
    for (int x = x1; x <= x2; x++) {
        if (steep) {
            img.setPixel(y, x, c);  // 注意坐标交换回来
        } else {
            img.setPixel(x, y, c);
        }
        
        e += 2 * dy;
        if (e >= dx) {
            y += yStep;
            e -= 2 * dx;
        }
    }
}
```

> [!tip] 核心技巧
> - **陡线转平线**：通过交换坐标轴
> - **反向转正**：通过交换端点
> - 最终只写一套核心逻辑

## Part 2：三角形光栅化

### 从线框到填充：新的挑战

直线算法可以画出三角形的**边框**：

```cpp
// 用三条线画三角形（线框模式）
void drawTriangleWireframe(Image& img, 
                           int x1, int y1, 
                           int x2, int y2, 
                           int x3, int y3, 
                           Color c) {
    drawLine(img, x1, y1, x2, y2, c);
    drawLine(img, x2, y2, x3, y3, c);
    drawLine(img, x3, y3, x1, y1, c);
}
```

但 3D 渲染需要的是**实心填充**的三角形表面。怎么判断一个像素在三角形**内部**？

### What：三角形光栅化要做什么？

```
输入：三角形的 3 个顶点 (x1,y1), (x2,y2), (x3,y3)
         ↓
步骤 1：找出三角形覆盖的所有像素
         ↓
步骤 2：把这些像素涂成指定颜色
         ↓
输出：一张画好三角形的图片
```

#### 关键问题：如何判断一个像素在三角形内？

假设三角形的三个顶点是 `P1`, `P2`, `P3`，要判断点 `O` 是否在三角形内：

```
           P2
          /\
         /  \
        /  O \    ← O 在内部
       /______\
      P1        P3

判断方法：
从 P1 走到 P2，O 在左边 → OK
从 P2 走到 P3，O 在左边 → OK  
从 P3 走到 P1，O 在左边 → OK
三个条件都满足，O 就在三角形内部
```

**数学工具**：用**[[Generated/Math/线性代数的本质/08-叉积|叉乘（Cross Product）]]**判断点在边的哪一侧。

> [!info] 为什么叉乘能判断左右？
> 
> 详细原理见 [[Generated/Math/线性代数的本质/08-叉积#判断点是否在边的内侧（光栅化核心）|线性代数：叉积的方向判定]] —— 核心在于**二维叉积计算的是有向面积，其符号反映向量的相对旋转方向**。

### How：三角形填充代码实现

#### 步骤 1：定义基础数据结构

```cpp
// 二维点/向量
struct Vec2 {
    float x, y;
    Vec2(float x = 0, float y = 0) : x(x), y(y) {}
    
    Vec2 operator-(const Vec2& other) const {
        return Vec2(x - other.x, y - other.y);
    }
};

// 三角形 = 三个顶点
struct Triangle {
    Vec2 p1, p2, p3;
    Triangle(const Vec2& a, const Vec2& b, const Vec2& c) 
        : p1(a), p2(b), p3(c) {}
};

// 颜色（RGBA）
struct Color {
    uint8_t r, g, b, a;
    Color(uint8_t r, uint8_t g, uint8_t b, uint8_t a = 255) 
        : r(r), g(g), b(b), a(a) {}
};
```

#### 步骤 2：[[Generated/Math/线性代数的本质/08-叉积|叉乘]] —— 判断方向的工具

```cpp
// 2D 叉乘：a × b = ax * by - ay * bx
// 结果 > 0：b 在 a 的左侧
// 结果 < 0：b 在 a 的右侧
// 结果 = 0：b 与 a 共线
float cross(const Vec2& a, const Vec2& b) {
    return a.x * b.y - a.y * b.x;
}
```

#### 步骤 3：判断点是否在三角形内

```cpp
bool pointInTriangle(const Vec2& p, const Triangle& tri) {
    // 三角形的三条边
    Vec2 e1 = tri.p2 - tri.p1;  // 边 P1→P2
    Vec2 e2 = tri.p3 - tri.p2;  // 边 P2→P3
    Vec2 e3 = tri.p1 - tri.p3;  // 边 P3→P1
    
    // 从顶点指向待判断点的向量
    Vec2 v1 = p - tri.p1;
    Vec2 v2 = p - tri.p2;
    Vec2 v3 = p - tri.p3;
    
    // 三次叉乘
    float c1 = cross(e1, v1);
    float c2 = cross(e2, v2);
    float c3 = cross(e3, v3);
    
    // 符号相同则在内部（全正或全负）
    bool hasNeg = (c1 < 0) || (c2 < 0) || (c3 < 0);
    bool hasPos = (c1 > 0) || (c2 > 0) || (c3 > 0);
    
    return !(hasNeg && hasPos);
}
```

#### 步骤 4：光栅化主函数

```cpp
void rasterizeTriangle(
    Image& image,                    // 输出图像
    const Triangle& tri,             // 要画的三角形
    const Color& color               // 填充颜色
) {
    int width = image.getWidth();
    int height = image.getHeight();
    
    // 遍历所有像素
    for (int y = 0; y < height; y++) {
        for (int x = 0; x < width; x++) {
            
            // 取像素中心点坐标
            Vec2 pixelPos(x + 0.5f, y + 0.5f);
            
            // 判断是否在三角形内
            if (pointInTriangle(pixelPos, tri)) {
                image.setPixel(x, y, color);
            }
        }
    }
}
```

#### 步骤 5：完整使用示例

```cpp
int main() {
    // 创建 100x100 的图像
    Image canvas(100, 100);
    
    // 定义一个红色三角形（只需三个顶点！）
    Triangle tri(
        Vec2(50, 10),   // 顶点1：顶部
        Vec2(10, 90),   // 顶点2：左下
        Vec2(90, 90)    // 顶点3：右下
    );
    
    // 光栅化 —— 自动填充三角形内的所有像素
    rasterizeTriangle(canvas, tri, Color(255, 0, 0));
    
    // 保存结果
    canvas.save("red_triangle.png");
    
    return 0;
}
```

### 优化：包围盒（Bounding Box）

上面的代码会遍历整张图片，即使三角形只占一小部分。**大部分像素根本不用判断。**

优化思路：先算出三角形覆盖的最小矩形区域，只在这个区域内遍历。

```cpp
void rasterizeTriangleOptimized(Image& image, const Triangle& tri, const Color& color) {
    // 计算包围盒
    int minX = std::max(0, (int)std::min({tri.p1.x, tri.p2.x, tri.p3.x}));
    int maxX = std::min(image.getWidth() - 1, (int)std::max({tri.p1.x, tri.p2.x, tri.p3.x}) + 1);
    int minY = std::max(0, (int)std::min({tri.p1.y, tri.p2.y, tri.p3.y}));
    int maxY = std::min(image.getHeight() - 1, (int)std::max({tri.p1.y, tri.p2.y, tri.p3.y}) + 1);
    
    // 只遍历包围盒内的像素
    for (int y = minY; y <= maxY; y++) {
        for (int x = minX; x <= maxX; x++) {
            if (pointInTriangle(Vec2(x + 0.5f, y + 0.5f), tri)) {
                image.setPixel(x, y, color);
            }
        }
    }
}
```

> [!tip] 效果
> 对于只占画面 10% 的三角形，优化后只需判断约 10% 的像素，**速度提升近 10 倍**。

## Part 3：重心坐标法（进阶）

前面我们用[[Generated/Math/线性代数的本质/08-叉积#判断点是否在边的内侧（光栅化核心）|叉乘法]]判断点是否在三角形内。这种方法直观、高效，但有一个局限：**只能回答"在不在"，无法直接得到"在哪里"**。

如果你要给三角形做**渐变色填充**或**贴图**，就需要知道每个像素相对于三个顶点的"位置关系"——这正是**重心坐标（Barycentric Coordinates）**能提供的。

### Why：为什么需要重心坐标？

| 需求 | [[Generated/Math/线性代数的本质/08-叉积#判断点是否在边的内侧（光栅化核心）|叉乘法]] | 重心坐标法 |
|------|--------|-----------|
| 纯色填充 | ✅ 够用 | ✅ 也能做 |
| **渐变色（顶点颜色插值）** | ❌ 需额外计算 | ✅ 权重直接就是插值系数 |
| **纹理映射（UV插值）** | ❌ 需额外计算 | ✅ 权重用于插值UV坐标 |
| **深度缓冲（Z值插值）** | ❌ 需额外计算 | ✅ 权重可直接插值Z值 |

**一句话**：重心坐标法在判断点是否在三角形内的同时，还能顺便算出**插值所需的权重**。

### What：重心坐标是什么？

#### 核心思想：用三个顶点的加权平均表示任意点

三角形 ABC 所在平面内的任意一点 P，都可以表示为三个顶点的加权平均：

$$
P = \alpha A + \beta B + \gamma C
$$

其中权重满足：
$$
\alpha + \beta + \gamma = 1
$$

**几何意义**：
- 当 P 在三角形**内部**时：$\alpha, \beta, \gamma \geq 0$
- 当 P 靠近顶点 A 时：$\alpha$ 接近 1，$\beta, \gamma$ 接近 0
- 当 P 在边 BC 上时：$\alpha = 0$

#### 与面积的直观关系

重心坐标还有另一个直观的理解——**面积比**：

```
        A
       /\
      /  \
     / P  \      α = area(PBC) / area(ABC)
    /______\     β = area(APC) / area(ABC)
   B        C    γ = area(ABP) / area(ABC)
```

> [!question] 为什么重心坐标定理成立？
> 
> **定理**：三角形 ABC 所在平面内的任意一点 P，都可以唯一表示为 $P = \alpha A + \beta B + \gamma C$，其中 $\alpha + \beta + \gamma = 1$。
> 
> **证明思路**：
> 
> **1. 存在性**：将条件 $\alpha = 1 - \beta - \gamma$ 代入：
> $$
> P = A + \beta(B-A) + \gamma(C-A)
> $$
> 即 $\vec{AP} = \beta \vec{AB} + \gamma \vec{AC}$。由于 $\vec{AB}, \vec{AC}$ 不共线（构成三角形的两边），它们构成平面的一组基，因此任意向量 $\vec{AP}$ 都能被唯一线性表示。
> 
> **2. 面积比公式**：利用二维叉乘表示有向面积：
> $$
> \alpha = \frac{\vec{PB} \times \vec{PC}}{\vec{AB} \times \vec{AC}} = \frac{S_{\triangle PBC}}{S_{\triangle ABC}}
> $$
> 同理可得 $\beta, \gamma$ 的面积比表达式。
> 
> **3. 内部判定**：
> - P 在内部 ⟺ 三个子三角形与母三角形同向 ⟺ $\alpha, \beta, \gamma \geq 0$
> - P 在边 BC 上 ⟺ $\triangle PBC$ 退化 ⟺ $\alpha = 0$
> 
> **4. 归一化**：三个子三角形面积之和等于母三角形面积，故 $\alpha + \beta + \gamma = 1$。

### How：重心坐标的计算与应用

#### 步骤 1：计算重心坐标

利用叉乘（有向面积）来计算权重：

```cpp
// 计算重心坐标 (alpha, beta, gamma)
// 返回 true 表示点在三角形内
bool barycentric(
    const Vec2& p, 
    const Triangle& tri,
    float& alpha, float& beta, float& gamma
) {
    // 三角形的有向面积（2倍）
    float denom = cross(tri.p2 - tri.p1, tri.p3 - tri.p1);
    
    // 三个子三角形的有向面积比
    alpha = cross(tri.p2 - p, tri.p3 - p) / denom;  // 对应顶点 p1
    beta  = cross(tri.p3 - p, tri.p1 - p) / denom;  // 对应顶点 p2
    gamma = 1.0f - alpha - beta;                     // 对应顶点 p3
    
    // 判断是否在内：所有权重非负
    return (alpha >= 0) && (beta >= 0) && (gamma >= 0);
}
```

> [!info] 注意 denom 的符号
> 如果三角形的顶点顺序是顺时针，denom 会是负数。但只要 alpha/beta/gamma 都用同一个 denom 计算，最终的正负判断不受影响。

#### 步骤 2：用重心坐标进行属性插值

```cpp
void rasterizeTriangleWithColor(
    Image& image,
    const Triangle& tri,
    const Color& c1, const Color& c2, const Color& c3  // 三个顶点的颜色
) {
    // 计算包围盒（复用之前的优化）
    int minX = /* ... */, maxX = /* ... */;
    int minY = /* ... */, maxY = /* ... */;
    
    for (int y = minY; y <= maxY; y++) {
        for (int x = minX; x <= maxX; x++) {
            Vec2 pixelPos(x + 0.5f, y + 0.5f);

                );
                image.setPixel(x, y, finalColor);
            }
        }
    }
}
```

```

## 从直线到三角形：知识的递进

| | 直线光栅化 | 三角形光栅化 |
|:--|:--|:--|
| **输入** | 2 个端点 | 3 个顶点 |
| **核心问题** | 下一个像素走 y 还是 y+1 | 像素是否在三角形内部 |
| **核心算法** | Bresenham 增量计算 | [[Generated/Math/线性代数的本质/08-叉积|叉乘]]方向判断 |
| **优化思想** | 只遍历 x 范围 | 包围盒裁剪 |
| **输出** | 一维的像素序列 | 二维的填充区域 |

**关键联系**：
- Bresenham 告诉我们：**用整数运算避免浮点**（三角形判断也可用整数版本）
- 三角形告诉我们：**用几何判定替代逐点计算**，从"怎么画"升级到"画哪里"

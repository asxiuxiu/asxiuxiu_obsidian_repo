> [!abstract] 核心观点
> 所有数字图像最终都分解为像素。3D 渲染、UI 设计的终点都是**控制像素颜色**。

图形学可能是每个游戏开发者的意难平，我也不例外。

图形工程师，总是被称为在**像素**背后的男人。

那么第一个问题：==像素是什么？==

---

## Why：为什么要理解像素？

像素是图形学的**原子**。

| 不理解像素的后果           | 理解像素后的收益            |
| :----------------- | :------------------ |
| 搞不懂[[走样]]、分辨率、像素密度 | 能优化渲染性能和图像质量        |
| 无法解释边缘锯齿和模糊        | 能设计抗锯齿策略            |
| 难以处理不同 DPI 屏幕的适配   | 理解 Retina/HiDPI 的原理 |

> [!tip] 一句话总结
> 不理解像素，就无法真正控制屏幕上的每一个颜色点。

---

## What：像素是什么？

> **像素（Pixel = Picture Element）** 是数字图像的==最小单位==。

### 物理层面：RGB 子像素

屏幕上的每个像素由 **RGB 三个子像素** 组成：

```
┌─────────────────┐
│  [R] [G] [B]   │  ← 一个像素 = 红 + 绿 + 蓝
└─────────────────┘
      ↓
   通过调节亮度混合出各种颜色
```

### 内存层面：颜色值存储

像素在内存中通常用 RGB/RGBA 值存储：

| 颜色 | RGB 值 | 说明 |
|:--|:--|:--|
| 纯红 | `(255, 0, 0)` | R 通道最大，GB 为 0 |
| 纯绿 | `(0, 255, 0)` | G 通道最大，RB 为 0 |
| 纯蓝 | `(0, 0, 255)` | B 通道最大，RG 为 0 |
| 白色 | `(255, 255, 255)` | 三通道全满 |
| 黑色 | `(0, 0, 0)` | 三通道全空 |

### 分辨率 = 像素矩阵

`1920×1080` 表示约 ==200 万个==像素排列成的矩阵。

## How：如何与像素打交道？

### 1. 定义像素结构

```cpp
#include <vector>
#include <cstdint>

// 一个像素 = 4 个字节 (RGBA8888)
struct Pixel { 
    uint8_t r, g, b, a;  // 红、绿、蓝、透明度
};

// 创建 100×100 的红色图像
std::vector<Pixel> image(100 * 100, {255, 0, 0, 255});

// 修改中心像素为绿色
image[50 * 100 + 50] = {0, 255, 0, 255};
```

> [!question] 为什么用 `uint8_t` 而不是 `uint` 或 `ushort`？
> 
> **三个原因：精度够用、内存高效、硬件原生支持**
> 
> | 类型 | 单通道 | RGBA 像素 | 1920×1080 图像 |
> |:--|:--:|:--:|:--:|
> | `uint8_t` | 1字节 | **4字节** | ~8 MB |
> | `ushort` | 2字节 | 8字节 | ~16 MB |
> | `uint` | 4字节 | 16字节 | ~32 MB |
> 
> - `uint8_t` (0-255) 的 256 级亮度已满足人眼分辨能力
> - 4字节对齐 32 位机器字长，GPU 原生支持 **RGBA8888** 格式
> - HDR 场景才需要 `half` 或 `float`（16/32 位浮点）

### 2. 保存像素为图像文件

内存里的像素数据是看不见的，需要保存为**图像文件**才能查看。

#### PPM 格式：最简单的图像格式

**PPM**（Portable Pixmap Format）是一种极简格式：

| 特性 | 优势 |
|:--|:--|
| **纯文本** | 可用记事本直接打开查看内容 |
| **无压缩** | 直接存储 RGB 值，零编解码开销 |
| **零依赖** | 不需要第三方库，几行代码就能读写 |
| **通用兼容** | 系统照片应用、浏览器都能打开 |

**PPM 文件结构：**
```
P3              # 格式标识：P3=ASCII文本，P6=二进制
100 100         # 图像宽度和高度（像素）
255             # 每个通道最大值（通常是255）
255 0 0         # 像素1的 R G B
0 255 0         # 像素2的 R G B
...             # 共 宽×高 个像素
```

#### 代码：保存 PPM

```cpp
#include <fstream>
#include <vector>
#include <cstdint>

struct Pixel { uint8_t r, g, b; };  // PPM 通常不需要 Alpha

void save_ppm(const char* filename, 
              const std::vector<Pixel>& pixels, 
              int width, int height) {
    std::ofstream file(filename);
    
    // 写入 PPM 头部
    file << "P3\n";              // P3 = ASCII 格式
    file << width << " " << height << "\n";
    file << "255\n";             // 最大颜色值
    
    // 写入像素数据
    for (const auto& p : pixels) {
        file << (int)p.r << " " 
             << (int)p.g << " " 
             << (int)p.b << " ";
    }
}

// 使用示例：生成红蓝渐变
int main() {
    const int width = 256, height = 256;
    std::vector<Pixel> image(width * height);
    
    for (int y = 0; y < height; y++) {
        for (int x = 0; x < width; x++) {
            image[y * width + x] = {
                static_cast<uint8_t>(x),   // R: 横向渐变
                0,                          // G: 无
                static_cast<uint8_t>(y)    // B: 纵向渐变
            };
        }
    }
    
    save_ppm("gradient.ppm", image, width, height);
    return 0;
}
```

#### 运行结果预览

运行后会得到一个从黑到红蓝的渐变：

```
左下角(0,0):      黑色  (0, 0, 0)
右下角(255,0):    纯红  (255, 0, 0)
左上角(0,255):    纯蓝  (0, 0, 255)  
右上角(255,255):  紫红  (255, 0, 255)
```

#### 查看 PPM 文件

| 方式 | 操作 |
|:--|:--|
| **Windows 照片** | 双击 `.ppm` 文件 |
| **浏览器** | 把 PPM 拖到浏览器窗口 |
| **VS Code** | 安装「Image Preview」插件 |
| **命令行** | `start gradient.ppm` |

---

> [!info] 为什么不用 BMP/PNG/JPEG？
> 
> - **BMP**：有文件头结构，需要处理字节对齐
> - **PNG/JPEG**：需要压缩编码，依赖第三方库
> - **PPM**：学习阶段的最佳选择，把关注点放在「像素操作」本身

## 小结

| 概念 | 要点 |
|:--|:--|
| **像素是什么** | 数字图像的最小单位，由 RGB 子像素组成 |
| **内存存储** | 用 `uint8_t` 表示 0-255 的颜色强度 |
| **如何验证** | 用 PPM 格式保存，直接观察像素操作结果 |
